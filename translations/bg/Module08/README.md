# Модул 08: Практическа работа с Microsoft Foundry Local - Пълен комплект за разработчици

## Преглед

Microsoft Foundry Local представлява следващото поколение на edge AI разработката, предоставяйки на разработчиците мощни инструменти за създаване, внедряване и мащабиране на AI приложения локално, като същевременно се осигурява безпроблемна интеграция с Azure AI Foundry. Този модул предлага изчерпателно покритие на Foundry Local - от инсталация до разработка на усъвършенствани агенти.

**Основни технологии:**
- Microsoft Foundry Local CLI и SDK
- Интеграция с Azure AI Foundry
- Локално извеждане на модели
- Кеширане и оптимизация на модели на устройството
- Архитектури, базирани на агенти

## Цели на обучението в модула

След завършване на този модул, ще можете:

- **Овладяване на настройката на Foundry Local**: Инсталиране, конфигуриране и оптимизиране на Foundry Local за разработка на Windows 11
- **Внедряване на разнообразни модели**: Стартиране на phi, qwen, deepseek и GPT-OSS-20B модели локално чрез CLI команди
- **Създаване на производствени решения**: Разработване на AI приложения с усъвършенствано инженерство на подсказки и интеграция на данни
- **Използване на екосистемата с отворен код**: Интеграция на модели от Hugging Face и допълнения от общността
- **Сравнение на AI архитектури**: Разбиране на компромисите между LLMs и SLMs и стратегии за внедряване
- **Разработка на AI агенти**: Създаване на интелигентни агенти с архитектурата и възможностите за grounding на Foundry Local
- **Прилагане на модели като инструменти**: Създаване на модулни, персонализируеми AI решения за корпоративни приложения

## Предварителни изисквания

### Системни изисквания
- **Операционна система**: Windows 11 (22H2 или по-нова версия)
- **Памет**: 16GB RAM (32GB препоръчително за по-големи модели)
- **Съхранение**: 50GB свободно пространство за кеширане на модели
- **Хардуер**: Препоръчително устройство с NPU (Copilot+ PC), GPU по избор

- Visual Studio Code с разширение AI Toolkit
- Python 3.10+ и pip
- Git за контрол на версиите
- PowerShell или Command Prompt
- Azure CLI (по избор за облачна интеграция)

- Основи на програмирането с Python
- Концепции за REST API
- Основни познания за подсказки и извеждане на модели

## Бърз старт за примери

### 1) Настройка на среда (Windows cmd.exe)
```cmd
cd Module08
py -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Стартиране на локален модел (нов терминал)
```cmd
foundry model list
foundry model run phi-4-mini
```

### 3) Стартиране на Chainlit демонстрация (Сесия 4)
```cmd
cd Module08
.\.venv\Scripts\activate
chainlit run samples\04\app.py -w
```

### 4) Стартиране на координатор за множество агенти (Сесия 5)
```cmd
cd Module08
.\.venv\Scripts\activate
python -m samples.05.agents.coordinator
```

Ако видите грешки при свързване, валидирайте Foundry Local:
```cmd
curl http://localhost:8000/v1/models
```

### Конфигурация на рутер (Сесия 6)
Рутерът извършва бърза проверка на състоянието и поддържа конфигурация, базирана на среда:
```cmd
set BASE_URL=http://localhost:8000
set GENERAL_MODEL=phi-4-mini
set REASONING_MODEL=deepseek-r1-distill-qwen-7b
set CODE_MODEL=qwen2.5-7b-instruct
REM Or provide a full JSON registry
set TOOL_REGISTRY={"general":{"model":"phi-4-mini"}}
python samples\06\router.py "Pick the best model for code refactoring"
```

Този модул представлява авангарда на edge AI разработката, комбинирайки корпоративните инструменти на Microsoft с гъвкавостта и иновациите на екосистемата с отворен код. Овладявайки Foundry Local, ще бъдете позиционирани на предната линия на разработката на AI приложения.

За Azure OpenAI (Сесия 2), вижте README файла за необходимите променливи на средата и настройки на API версията.
