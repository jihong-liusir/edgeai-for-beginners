# Session 4: Najmodernejšie modely – LLM, SLM a inferencia na zariadení

## Prehľad

Porovnajte LLM a SLM, zhodnoťte kompromisy medzi lokálnou a cloudovou inferenciou a implementujte ukážky, ktoré demonštrujú scenáre EdgeAI pomocou Phi a ONNX Runtime. Zároveň sa zameriame na Chainlit RAG, možnosti inferencie WebGPU a integráciu Open WebUI.

Referencie:
- Dokumentácia Foundry Local: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Návod na Open WebUI (chat aplikácia s Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Ciele učenia
- Pochopiť kompromisy medzi LLM a SLM z hľadiska nákladov, latencie a presnosti
- Vybrať medzi lokálnou a cloudovou inferenciou podľa konkrétnych obchodných potrieb
- Implementovať malú RAG ukážku s Chainlit
- Preskúmať WebGPU pre akceleráciu na strane prehliadača
- Pripojiť Open WebUI k Foundry Local

## Časť 1: LLM vs SLM – Rozhodovacia matica

Zvážte:
- Latencia: SLM na zariadení často poskytujú odpovede do jednej sekundy
- Náklady: lokálna inferencia znižuje náklady na cloud
- Súkromie: citlivé údaje zostávajú na zariadení
- Schopnosti: LLM môžu prekonať SLM pri zložitých úlohách
- Spoľahlivosť: hybridné stratégie znižujú riziko výpadkov

## Časť 2: Lokálne vs Cloud – Hybridné vzory

- Lokálne ako prvé s cloudovým záložným riešením pre veľké/zložité požiadavky
- Cloud ako prvý s lokálnym riešením pre citlivé na súkromie alebo offline scenáre
- Smerovanie podľa typu úlohy (generovanie kódu na DeepSeek, všeobecný chat na Phi/Qwen)

## Časť 3: RAG Chat aplikácia s Chainlit (Minimalistická)

Inštalujte závislosti:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Spustite:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Rozšírenie: pridajte jednoduchý vyhľadávač (lokálne súbory) a predpíšte získaný kontext k používateľskej požiadavke.

## Časť 4: WebGPU Inferencia (Upozornenie)

Spustite malé modely priamo v prehliadači pomocou WebGPU. Ideálne pre ukážky zamerané na súkromie a skúsenosti bez inštalácie. Nižšie je minimalistický, krok za krokom príklad s ONNX Runtime Web a poskytovateľom vykonávania WebGPU.

1) Skontrolujte podporu WebGPU
- Prehliadače Chromium: chrome://gpu → potvrďte, že “WebGPU” je povolené
- Programatická kontrola (skontrolujeme aj v kóde): `if (!('gpu' in navigator)) { /* no WebGPU */ }`

2) Vytvorte minimalistický projekt
Vytvorte priečinok a dva súbory: `index.html` a `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Spustite lokálne (Windows cmd.exe)
Použite jednoduchý statický server, aby prehliadač mohol načítať model.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Otvorte http://localhost:5173 vo vašom prehliadači. Mali by ste vidieť inicializačné logy, vytvorenie relácie s WebGPU a predikciu argmax.

4) Riešenie problémov
- Ak WebGPU nie je dostupné: aktualizujte Chrome/Edge a uistite sa, že ovládače GPU sú aktuálne, potom skontrolujte chrome://flags pre “Enable WebGPU”.
- Ak sa vyskytnú chyby CORS alebo fetch: uistite sa, že súbory sú poskytované cez http:// (nie file://) a URL modelu umožňuje požiadavky z iných domén.
- Záložné riešenie na CPU: zmeňte `executionProviders: ['wasm']` na overenie základného správania.

5) Ďalšie kroky
- Vymeniť za doménovo špecifický ONNX model (napr. klasifikácia obrázkov alebo malý textový model).
- Pridať logiku predspracovania/postspracovania pre reálne vstupy.
- Pre väčšie modely alebo produkčnú latenciu uprednostnite Foundry Local alebo ONNX Runtime Server.

## Časť 5: Open WebUI + Foundry Local (Krok za krokom)

Toto pripojí Open WebUI k OpenAI-kompatibilnému endpointu Foundry Local pre lokálne chatovacie rozhranie.

1) Predpoklady
- Foundry Local nainštalovaný a funkčný (`foundry --version`)
- Jeden model pripravený na lokálne spustenie (napr. `phi-4-mini`)
- Nainštalovaný Docker Desktop (odporúča sa pre Open WebUI)

2) Spustite model s Foundry Local
```powershell
foundry model run phi-4-mini
```
Tým sa vystaví OpenAI-kompatibilné API na `http://localhost:8000`.

3) Spustite Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Poznámky:
- Na Windows, `host.docker.internal` umožňuje kontajneru dosiahnuť váš hostiteľ na `localhost`.
- Nastavíme `OPENAI_API_BASE_URL` na endpoint Foundry Local a fiktívny `OPENAI_API_KEY`.

4) Konfigurácia z rozhrania Open WebUI (alternatíva)
- Prejdite na http://localhost:3000
- Dokončite počiatočné nastavenie (admin používateľ)
- Choďte na Nastavenia → Modely/Poskytovatelia
- Nastavte základnú URL: `http://host.docker.internal:8000/v1`
- Nastavte API kľúč: `local-key` (zástupný znak)
- Uložte

5) Spustite testovaciu požiadavku
- V Open WebUI chate vyberte alebo zadajte názov modelu `phi-4-mini`
- Požiadavka: “Vymenujte päť výhod inferencie na zariadení.”
- Mali by ste vidieť odpoveď streamovanú z vášho lokálneho modelu

6) Riešenie problémov
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Voliteľné: Uloženie dát Open WebUI
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
