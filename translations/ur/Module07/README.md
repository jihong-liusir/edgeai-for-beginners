# باب 07: ایج اے آئی کے نمونے

ایج اے آئی مصنوعی ذہانت اور ایج کمپیوٹنگ کے امتزاج کو ظاہر کرتا ہے، جو کہ براہ راست آلات پر ذہین پروسیسنگ کو ممکن بناتا ہے، بغیر کلاؤڈ کنیکٹیویٹی پر انحصار کیے۔ یہ باب مختلف پلیٹ فارمز اور فریم ورکز پر پانچ منفرد ایج اے آئی کے نفاذ کو دریافت کرتا ہے، جو ایج پر اے آئی ماڈلز چلانے کی طاقت اور لچک کو ظاہر کرتا ہے۔

## 1. NVIDIA Jetson Orin Nano میں ایج اے آئی

NVIDIA Jetson Orin Nano ایج اے آئی کمپیوٹنگ میں ایک انقلابی قدم ہے، جو کریڈٹ کارڈ کے سائز کے کمپیکٹ ڈیزائن میں 67 TOPS تک اے آئی کی کارکردگی فراہم کرتا ہے۔ یہ طاقتور ایج اے آئی پلیٹ فارم شوقین افراد، طلباء، اور پیشہ ور ڈویلپرز کے لیے جنریٹو اے آئی کی ترقی کو آسان بناتا ہے۔

### اہم خصوصیات
- 67 TOPS تک اے آئی کی کارکردگی فراہم کرتا ہے—اپنے پیشرو سے 1.7X بہتر
- 1024 CUDA کورز اور 32 تک ٹینسر کورز اے آئی پروسیسنگ کے لیے
- 6-کور Arm Cortex-A78AE v8.2 64-bit CPU، زیادہ سے زیادہ 1.5 GHz کی فریکوئنسی کے ساتھ
- صرف $249 کی قیمت پر، یہ پلیٹ فارم ڈویلپرز، طلباء، اور میکرز کے لیے سب سے زیادہ قابل رسائی اور سستا ہے

### ایپلیکیشنز
Jetson Orin Nano جدید جنریٹو اے آئی ماڈلز جیسے ویژن ٹرانسفارمرز، بڑے زبان کے ماڈلز، اور ویژن-لینگویج ماڈلز چلانے میں مہارت رکھتا ہے۔ یہ خاص طور پر GenAI کے استعمال کے لیے ڈیزائن کیا گیا ہے، اور اب آپ کئی LLMs کو ایک ہتھیلی کے سائز کے آلے پر چلا سکتے ہیں۔ مقبول استعمال میں شامل ہیں: اے آئی سے چلنے والے روبوٹکس، اسمارٹ ڈرونز، ذہین کیمرے، اور خود مختار ایج ڈیوائسز۔

**مزید جانیں**: [NVIDIA's Jetson Orin Nano SuperComputer: The Next Big Thing in EdgeAI](https://medium.com/data-science-in-your-pocket/nvidias-jetson-orin-nano-supercomputer-the-next-big-thing-in-edgeai-e9eff687ae62)

## 2. موبائل ایپلیکیشنز میں ایج اے آئی .NET MAUI اور ONNX Runtime GenAI کے ساتھ

یہ حل جنریٹو اے آئی اور بڑے زبان کے ماڈلز (LLMs) کو .NET MAUI (ملٹی پلیٹ فارم ایپ UI) اور ONNX Runtime GenAI کا استعمال کرتے ہوئے کراس پلیٹ فارم موبائل ایپلیکیشنز میں ضم کرنے کا مظاہرہ کرتا ہے۔ یہ طریقہ .NET ڈویلپرز کو اینڈرائیڈ اور iOS ڈیوائسز پر مقامی طور پر چلنے والی جدید اے آئی سے چلنے والی موبائل ایپلیکیشنز بنانے کے قابل بناتا ہے۔

### اہم خصوصیات
- .NET MAUI فریم ورک پر بنایا گیا، جو اینڈرائیڈ اور iOS ایپلیکیشنز کے لیے ایک ہی کوڈ بیس فراہم کرتا ہے
- ONNX Runtime GenAI انضمام جنریٹو اے آئی ماڈلز کو براہ راست موبائل ڈیوائسز پر چلانے کے قابل بناتا ہے
- موبائل ڈیوائسز کے لیے مخصوص ہارڈویئر ایکسیلیریٹرز کی حمایت کرتا ہے، بشمول CPU، GPU، اور خصوصی موبائل اے آئی پروسیسرز
- ONNX Runtime کے ذریعے iOS کے لیے CoreML اور اینڈرائیڈ کے لیے NNAPI جیسی پلیٹ فارم مخصوص اصلاحات
- جنریٹو اے آئی لوپ کو مکمل طور پر نافذ کرتا ہے، بشمول پری اور پوسٹ پروسیسنگ، انفرنس، لاجٹس پروسیسنگ، سرچ اور سیمپلنگ، اور KV کیش مینجمنٹ

### ترقی کے فوائد
.NET MAUI کا طریقہ ڈویلپرز کو ان کے موجودہ C# اور .NET مہارتوں کو استعمال کرنے کی اجازت دیتا ہے جبکہ کراس پلیٹ فارم اے آئی ایپلیکیشنز بناتے ہیں۔ ONNX Runtime GenAI فریم ورک متعدد ماڈل آرکیٹیکچرز کی حمایت کرتا ہے، بشمول Llama، Mistral، Phi، Gemma، اور دیگر۔ ARM64 کے لیے بہتر کردہ کرنلز INT4 کوانٹائزڈ میٹرکس ملٹیپلیکیشن کو تیز کرتے ہیں، موبائل ہارڈویئر پر موثر کارکردگی کو یقینی بناتے ہیں جبکہ .NET ترقی کے تجربے کو برقرار رکھتے ہیں۔

### استعمال کے کیسز
یہ حل ان ڈویلپرز کے لیے مثالی ہے جو .NET ٹیکنالوجیز کا استعمال کرتے ہوئے اے آئی سے چلنے والی موبائل ایپلیکیشنز بنانا چاہتے ہیں، بشمول ذہین چیٹ بوٹس، امیج ریکگنیشن ایپس، زبان کے ترجمے کے ٹولز، اور ذاتی نوعیت کے سفارشاتی نظام جو مکمل طور پر ڈیوائس پر چلتے ہیں، بہتر پرائیویسی اور آف لائن صلاحیت کے لیے۔

**مزید جانیں**: [.NET MAUI ONNX Runtime GenAI Example](https://github.com/microsoft/onnxruntime-genai/tree/jialli/genny-maui/examples/csharp/GennyMaui)

## 3. Azure میں ایج اے آئی چھوٹے زبان کے ماڈلز انجن کے ساتھ

Microsoft کا Azure پر مبنی ایج اے آئی حل چھوٹے زبان کے ماڈلز (SLMs) کو کلاؤڈ-ایج ہائبرڈ ماحول میں مؤثر طریقے سے تعینات کرنے پر مرکوز ہے۔ یہ طریقہ کلاؤڈ اسکیل اے آئی سروسز اور ایج تعیناتی کی ضروریات کے درمیان خلا کو پُر کرتا ہے۔

### آرکیٹیکچر کے فوائد
- Azure AI سروسز کے ساتھ ہموار انضمام
- SLMs/LLMs اور ملٹی موڈل ماڈلز کو ONNX Runtime کے ساتھ ڈیوائس اور کلاؤڈ پر چلائیں
- انٹرپرائز اسکیل تعیناتی کے لیے بہتر
- مسلسل ماڈل اپڈیٹس اور مینجمنٹ کی حمایت

### استعمال کے کیسز
Azure ایج اے آئی نفاذ ان منظرناموں میں بہترین ہے جن میں انٹرپرائز گریڈ اے آئی تعیناتی کی ضرورت ہوتی ہے، کلاؤڈ مینجمنٹ کی صلاحیتوں کے ساتھ۔ اس میں ذہین دستاویز پروسیسنگ، ریئل ٹائم اینالیٹکس، اور ہائبرڈ اے آئی ورک فلو شامل ہیں جو کلاؤڈ اور ایج کمپیوٹنگ وسائل دونوں کا فائدہ اٹھاتے ہیں۔

**مزید جانیں**: [Azure EdgeAI SLM Engine](https://github.com/microsoft/onnxruntime-genai/tree/main/examples/slm_engine)

## 4. Windows ML کے ساتھ ایج اے آئی

Windows ML Microsoft کا جدید ترین رن ٹائم ہے جو ڈیوائس پر ماڈل انفرنس کے لیے بہتر بنایا گیا ہے اور تعیناتی کو آسان بناتا ہے، جو Windows AI Foundry کی بنیاد کے طور پر کام کرتا ہے۔ یہ پلیٹ فارم ڈویلپرز کو اے آئی سے چلنے والی Windows ایپلیکیشنز بنانے کے قابل بناتا ہے جو PC ہارڈویئر کی مکمل صلاحیتوں کا فائدہ اٹھاتی ہیں۔

### پلیٹ فارم کی صلاحیتیں
- تمام Windows 11 PCs پر کام کرتا ہے جو ورژن 24H2 (بلڈ 26100) یا اس سے زیادہ پر چل رہے ہیں
- تمام x64 اور ARM64 PC ہارڈویئر پر کام کرتا ہے، یہاں تک کہ وہ PCs جن میں NPUs یا GPUs نہیں ہیں
- ڈویلپرز کو اپنے ماڈلز لانے اور انہیں AMD، Intel، NVIDIA، اور Qualcomm کے CPU، GPU، NPU سمیت سلیکون پارٹنر ایکو سسٹم میں مؤثر طریقے سے تعینات کرنے کی اجازت دیتا ہے
- انفراسٹرکچر APIs کا فائدہ اٹھاتے ہوئے، ڈویلپرز کو مختلف سلیکون کو ہدف بنانے کے لیے اپنی ایپ کے متعدد بلڈز بنانے کی ضرورت نہیں ہے

### ڈویلپر فوائد
Windows ML ہارڈویئر اور ایکزیکیوشن پرووائیڈرز کو خلاصہ کرتا ہے، تاکہ آپ اپنے کوڈ پر توجہ مرکوز کر سکیں۔ اس کے علاوہ، Windows ML خود بخود تازہ ترین NPUs، GPUs، اور CPUs کی حمایت کے لیے اپ ڈیٹ ہوتا ہے جیسے ہی وہ جاری ہوتے ہیں۔ یہ پلیٹ فارم Windows ہارڈویئر ایکو سسٹم میں اے آئی کی ترقی کے لیے ایک متحد فریم ورک فراہم کرتا ہے۔

**مزید جانیں**: 
- [Windows ML Overview](https://learn.microsoft.com/en-us/windows/ai/new-windows-ml/overview)
- [Windows EdgeAI Development Guide](../windowdeveloper.md) - Windows Edge AI ترقی کے لیے جامع گائیڈ

## 5. Foundry Local ایپلیکیشنز کے ساتھ ایج اے آئی

Foundry Local ڈویلپرز کو .NET میں مقامی وسائل کا استعمال کرتے ہوئے Retrieval Augmented Generation (RAG) ایپلیکیشنز بنانے کے قابل بناتا ہے، جو مقامی زبان کے ماڈلز کو سیمینٹک سرچ صلاحیتوں کے ساتھ جوڑتا ہے۔ یہ طریقہ پرائیویسی پر مبنی اے آئی حل فراہم کرتا ہے جو مکمل طور پر مقامی انفراسٹرکچر پر کام کرتے ہیں۔

### تکنیکی آرکیٹیکچر
- Phi زبان کے ماڈل، مقامی ایمبیڈنگز، اور سیمینٹک کرنل کو RAG منظرنامہ بنانے کے لیے جوڑتا ہے
- ایمبیڈنگز کو فلوٹنگ پوائنٹ ویلیوز کے ویکٹرز (ایرے) کے طور پر استعمال کرتا ہے جو مواد اور اس کے سیمینٹک معنی کی نمائندگی کرتے ہیں
- سیمینٹک کرنل مرکزی آرکیسٹریٹر کے طور پر کام کرتا ہے، Phi اور اسمارٹ کمپوننٹس کو مربوط کرتا ہے تاکہ ایک ہموار RAG پائپ لائن بنائی جا سکے
- مقامی ویکٹر ڈیٹا بیسز کی حمایت کرتا ہے، بشمول SQLite اور Qdrant

### نفاذ کے فوائد
RAG، یا Retrieval Augmented Generation، بنیادی طور پر "کچھ معلومات تلاش کریں اور اسے پرامپٹ میں ڈالیں" کہنے کا ایک جدید طریقہ ہے۔ یہ مقامی نفاذ ڈیٹا پرائیویسی کو یقینی بناتا ہے جبکہ اپنی مرضی کے علم کے ذخائر پر مبنی ذہین جوابات فراہم کرتا ہے۔ یہ طریقہ خاص طور پر انٹرپرائز منظرناموں کے لیے قیمتی ہے جن میں ڈیٹا خودمختاری اور آف لائن آپریشن کی صلاحیتیں درکار ہوتی ہیں۔

**مزید جانیں**: [Foundry Local RAG Samples](https://github.com/microsoft/Foundry-Local/tree/main/samples/dotNET/rag)

### Windows Foundry Local

Microsoft Foundry Local ایک OpenAI‑مطابقت پذیر REST سرور فراہم کرتا ہے جو ONNX Runtime کے ذریعے Windows پر ماڈلز کو مقامی طور پر چلانے کے لیے طاقت دیتا ہے۔ نیچے ایک فوری، تصدیق شدہ خلاصہ دیا گیا ہے؛ مکمل تفصیلات کے لیے آفیشل ڈاکس دیکھیں۔

- شروعات کریں: https://learn.microsoft.com/azure/ai-foundry/foundry-local/get-started
- آرکیٹیکچر: https://learn.microsoft.com/azure/ai-foundry/foundry-local/concepts/foundry-local-architecture
- CLI حوالہ: https://learn.microsoft.com/azure/ai-foundry/foundry-local/reference/reference-cli
- اس ریپو میں مکمل Windows گائیڈ: [foundrylocal.md](./foundrylocal.md)

Windows پر انسٹال یا اپ گریڈ کریں (cmd.exe):
```cmd
winget install Microsoft.FoundryLocal
winget upgrade --id Microsoft.FoundryLocal
foundry --version
```

CLI کیٹیگریز کو دریافت کریں:
```cmd
foundry model --help
foundry service --help
foundry cache --help
```

ماڈل چلائیں اور متحرک اینڈ پوائنٹ دریافت کریں:
```cmd
foundry model run gpt-oss-20b
foundry service status
```

ماڈلز کی فہرست کے لیے فوری REST چیک (PORT کو اسٹیٹس سے تبدیل کریں):
```cmd
curl -s http://localhost:PORT/v1/models
```

ٹپس:
- SDK انضمام: https://learn.microsoft.com/azure/ai-foundry/foundry-local/how-to/how-to-integrate-with-inference-sdks
- اپنا ماڈل لائیں (کمپائل کریں): https://learn.microsoft.com/azure/ai-foundry/foundry-local/how-to/how-to-compile-hugging-face-models

## Windows EdgeAI ترقی کے وسائل

خاص طور پر Windows پلیٹ فارم کو ہدف بنانے والے ڈویلپرز کے لیے، ہم نے ایک جامع گائیڈ تیار کی ہے جو مکمل Windows EdgeAI ایکو سسٹم کا احاطہ کرتی ہے۔ یہ وسیلہ Windows AI Foundry کے بارے میں تفصیلی معلومات فراہم کرتا ہے، بشمول APIs، ٹولز، اور Windows پر EdgeAI ترقی کے لیے بہترین طریقے۔

### Windows AI Foundry پلیٹ فارم
Windows AI Foundry پلیٹ فارم Edge AI ترقی کے لیے خاص طور پر ڈیزائن کردہ ٹولز اور APIs کا جامع سوٹ فراہم کرتا ہے۔ اس میں NPU-تیز کردہ ہارڈویئر، Windows ML انضمام، اور پلیٹ فارم مخصوص اصلاحی تکنیکوں کے لیے خصوصی حمایت شامل ہے۔

**جامع گائیڈ**: [Windows EdgeAI Development Guide](../windowdeveloper.md)

یہ گائیڈ شامل کرتی ہے:
- Windows AI Foundry پلیٹ فارم کا جائزہ اور اجزاء
- NPU ہارڈویئر پر مؤثر انفرنس کے لیے Phi Silica API
- امیج پروسیسنگ اور OCR کے لیے کمپیوٹر ویژن APIs
- Windows ML رن ٹائم انضمام اور اصلاح
- مقامی ترقی اور جانچ کے لیے Foundry Local CLI
- Windows ڈیوائسز کے لیے ہارڈویئر اصلاحی حکمت عملی
- عملی نفاذ کی مثالیں اور بہترین طریقے

### Edge AI ترقی کے لیے AI ٹول کٹ
Visual Studio Code استعمال کرنے والے ڈویلپرز کے لیے، AI ٹول کٹ ایکسٹینشن Edge AI ایپلیکیشنز بنانے، جانچنے، اور تعینات کرنے کے لیے ایک جامع ترقیاتی ماحول فراہم کرتا ہے۔ یہ ٹول کٹ VS Code کے اندر Edge AI ترقی کے ورک فلو کو آسان بناتا ہے۔

**ترقیاتی گائیڈ**: [AI Toolkit for Edge AI Development](../aitoolkit.md)

AI ٹول کٹ گائیڈ شامل کرتی ہے:
- ایج تعیناتی کے لیے ماڈل دریافت اور انتخاب
- مقامی جانچ اور اصلاحی ورک فلو
- ایج ماڈلز کے لیے ONNX اور Ollama انضمام
- ماڈل کنورژن اور کوانٹائزیشن تکنیک
- ایج منظرناموں کے لیے ایجنٹ ترقی
- کارکردگی کی تشخیص اور نگرانی
- تعیناتی کی تیاری اور بہترین طریقے

## نتیجہ

یہ پانچ ایج اے آئی کے نفاذ آج دستیاب ایج اے آئی حلوں کی پختگی اور تنوع کو ظاہر کرتے ہیں۔ ہارڈویئر-تیز کردہ ایج ڈیوائسز جیسے Jetson Orin Nano سے لے کر سافٹ ویئر فریم ورکز جیسے ONNX Runtime GenAI اور Windows ML تک، ڈویلپرز کے پاس ایج پر ذہین ایپلیکیشنز تعینات کرنے کے لیے بے مثال اختیارات ہیں۔

ان تمام پلیٹ فارمز میں مشترک دھاگہ اے آئی صلاحیتوں کی جمہوریت ہے، جو مختلف مہارت کی سطحوں اور استعمال کے کیسز کے ڈویلپرز کے لیے جدید مشین لرننگ کو قابل رسائی بناتا ہے۔ چاہے موبائل ایپلیکیشنز، ڈیسک ٹاپ سافٹ ویئر، یا ایمبیڈڈ سسٹمز بنانا ہو، یہ ایج اے آئی حل اگلی نسل کی ذہین ایپلیکیشنز کے لیے بنیاد فراہم کرتے ہیں جو ایج پر مؤثر اور نجی طور پر کام کرتی ہیں۔

ہر پلیٹ فارم منفرد فوائد پیش کرتا ہے: Jetson Orin Nano ہارڈویئر-تیز کردہ ایج کمپیوٹنگ کے لیے، ONNX Runtime GenAI کراس پلیٹ فارم موبائل ترقی کے لیے، Azure EdgeAI انٹرپرائز کلاؤڈ-ایج انضمام کے لیے، Windows ML Windows-مقامی ایپلیکیشنز کے لیے، اور Foundry Local پرائیویسی پر مبنی RAG نفاذ کے لیے۔ یہ سب مل کر ایج اے آئی ترقی کے لیے ایک جامع ایکو سسٹم کی نمائندگی کرتے ہیں۔
