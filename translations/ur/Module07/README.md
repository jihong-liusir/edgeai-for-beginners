# باب 07: ایج اے آئی کے نمونے

ایج اے آئی مصنوعی ذہانت اور ایج کمپیوٹنگ کے امتزاج کو ظاہر کرتا ہے، جو براہ راست آلات پر ذہین پروسیسنگ کو ممکن بناتا ہے بغیر کلاؤڈ کنیکٹیویٹی پر انحصار کیے۔ یہ باب مختلف پلیٹ فارمز اور فریم ورکز پر پانچ منفرد ایج اے آئی کے نفاذ کو دریافت کرتا ہے، جو ایج پر اے آئی ماڈلز چلانے کی طاقت اور لچک کو ظاہر کرتا ہے۔

## 1. NVIDIA Jetson Orin Nano میں ایج اے آئی

NVIDIA Jetson Orin Nano ایج اے آئی کمپیوٹنگ میں ایک انقلابی پیش رفت ہے، جو کریڈٹ کارڈ کے سائز کے کمپیکٹ ڈیزائن میں 67 TOPS تک اے آئی پرفارمنس فراہم کرتا ہے۔ یہ طاقتور ایج اے آئی پلیٹ فارم شوقین افراد، طلباء، اور پیشہ ور ڈویلپرز کے لیے جنریٹو اے آئی کی ترقی کو آسان بناتا ہے۔

### اہم خصوصیات
- 67 TOPS تک اے آئی پرفارمنس فراہم کرتا ہے—اپنے پیشرو سے 1.7X بہتر
- 1024 CUDA کورز اور 32 تک ٹینسر کورز اے آئی پروسیسنگ کے لیے
- 6-کور Arm Cortex-A78AE v8.2 64-bit CPU، زیادہ سے زیادہ 1.5 GHz فریکوئنسی کے ساتھ
- صرف $249 کی قیمت پر، یہ پلیٹ فارم ڈویلپرز، طلباء، اور میکرز کے لیے سب سے زیادہ قابل رسائی اور سستا ہے

### ایپلیکیشنز
Jetson Orin Nano جدید جنریٹو اے آئی ماڈلز جیسے وژن ٹرانسفارمرز، بڑے زبان ماڈلز، اور وژن-لینگویج ماڈلز چلانے میں مہارت رکھتا ہے۔ یہ خاص طور پر GenAI کے استعمال کے لیے ڈیزائن کیا گیا ہے، اور اب آپ کئی LLMs کو ایک ہتھیلی کے سائز کے آلے پر چلا سکتے ہیں۔ مقبول استعمالات میں اے آئی سے چلنے والے روبوٹکس، اسمارٹ ڈرونز، ذہین کیمرے، اور خود مختار ایج ڈیوائسز شامل ہیں۔

**مزید جانیں**: [NVIDIA's Jetson Orin Nano SuperComputer: The Next Big Thing in EdgeAI](https://medium.com/data-science-in-your-pocket/nvidias-jetson-orin-nano-supercomputer-the-next-big-thing-in-edgeai-e9eff687ae62)

## 2. موبائل ایپلیکیشنز میں ایج اے آئی .NET MAUI اور ONNX Runtime GenAI کے ساتھ

یہ حل جنریٹو اے آئی اور بڑے زبان ماڈلز (LLMs) کو .NET MAUI (ملٹی پلیٹ فارم ایپ UI) اور ONNX Runtime GenAI کا استعمال کرتے ہوئے کراس پلیٹ فارم موبائل ایپلیکیشنز میں ضم کرنے کا مظاہرہ کرتا ہے۔ یہ طریقہ .NET ڈویلپرز کو اینڈرائیڈ اور iOS ڈیوائسز پر مقامی طور پر چلنے والی جدید اے آئی سے چلنے والی موبائل ایپلیکیشنز بنانے کے قابل بناتا ہے۔

### اہم خصوصیات
- .NET MAUI فریم ورک پر بنایا گیا، جو اینڈرائیڈ اور iOS ایپلیکیشنز کے لیے ایک واحد کوڈ بیس فراہم کرتا ہے
- ONNX Runtime GenAI انضمام جنریٹو اے آئی ماڈلز کو براہ راست موبائل ڈیوائسز پر چلانے کے قابل بناتا ہے
- موبائل ڈیوائسز کے لیے مخصوص ہارڈویئر ایکسیلیریٹرز کی حمایت کرتا ہے، بشمول CPU، GPU، اور خصوصی موبائل اے آئی پروسیسرز
- ONNX Runtime کے ذریعے iOS کے لیے CoreML اور اینڈرائیڈ کے لیے NNAPI جیسی پلیٹ فارم مخصوص اصلاحات
- جنریٹو اے آئی لوپ کو مکمل طور پر نافذ کرتا ہے، بشمول پری اور پوسٹ پروسیسنگ، انفرنس، لاجٹس پروسیسنگ، سرچ اور سیمپلنگ، اور KV کیش مینجمنٹ

### ترقی کے فوائد
.NET MAUI کا طریقہ ڈویلپرز کو ان کے موجودہ C# اور .NET مہارتوں کو استعمال کرنے کی اجازت دیتا ہے جبکہ کراس پلیٹ فارم اے آئی ایپلیکیشنز بناتے ہیں۔ ONNX Runtime GenAI فریم ورک متعدد ماڈل آرکیٹیکچرز کی حمایت کرتا ہے، بشمول Llama، Mistral، Phi، Gemma، اور دیگر۔ ARM64 کرنلز کو بہتر بنایا گیا ہے تاکہ INT4 کوانٹائزڈ میٹرکس ضرب کو تیز کیا جا سکے، موبائل ہارڈویئر پر موثر کارکردگی کو یقینی بناتے ہوئے .NET ترقی کے تجربے کو برقرار رکھا جا سکے۔

### استعمال کے کیسز
یہ حل ان ڈویلپرز کے لیے مثالی ہے جو .NET ٹیکنالوجیز کا استعمال کرتے ہوئے اے آئی سے چلنے والی موبائل ایپلیکیشنز بنانا چاہتے ہیں، بشمول ذہین چیٹ بوٹس، امیج ریکگنیشن ایپس، زبان کے ترجمے کے ٹولز، اور ذاتی نوعیت کے سفارشاتی نظام جو مکمل طور پر ڈیوائس پر چلتے ہیں، بہتر پرائیویسی اور آف لائن صلاحیت کے لیے۔

**مزید جانیں**: [.NET MAUI ONNX Runtime GenAI Example](https://github.com/microsoft/onnxruntime-genai/tree/jialli/genny-maui/examples/csharp/GennyMaui)

## 3. Azure میں ایج اے آئی چھوٹے زبان ماڈلز انجن کے ساتھ

Microsoft کا Azure پر مبنی ایج اے آئی حل کلاؤڈ-ایج ہائبرڈ ماحول میں چھوٹے زبان ماڈلز (SLMs) کو مؤثر طریقے سے تعینات کرنے پر مرکوز ہے۔ یہ طریقہ کلاؤڈ اسکیل اے آئی سروسز اور ایج تعیناتی کی ضروریات کے درمیان خلا کو پُر کرتا ہے۔

### آرکیٹیکچر کے فوائد
- Azure AI سروسز کے ساتھ ہموار انضمام
- SLMs/LLMs اور ملٹی موڈل ماڈلز کو ONNX Runtime کے ساتھ ڈیوائس اور کلاؤڈ پر چلائیں
- انٹرپرائز اسکیل تعیناتی کے لیے بہتر بنایا گیا
- مسلسل ماڈل اپ ڈیٹس اور مینجمنٹ کی حمایت

### استعمال کے کیسز
Azure EdgeAI نفاذ ان منظرناموں میں بہترین ہے جن میں انٹرپرائز گریڈ اے آئی تعیناتی کی ضرورت ہوتی ہے، کلاؤڈ مینجمنٹ کی صلاحیتوں کے ساتھ۔ اس میں ذہین دستاویز پروسیسنگ، ریئل ٹائم اینالیٹکس، اور ہائبرڈ اے آئی ورک فلو شامل ہیں جو کلاؤڈ اور ایج کمپیوٹنگ وسائل دونوں کا فائدہ اٹھاتے ہیں۔

**مزید جانیں**: [Azure EdgeAI SLM Engine](https://github.com/microsoft/onnxruntime-genai/tree/main/examples/slm_engine)

## 4. Windows ML کے ساتھ ایج اے آئی

Windows ML Microsoft کا جدید ترین رن ٹائم ہے جو ڈیوائس پر ماڈل انفرنس کو بہتر بناتا ہے اور تعیناتی کو آسان بناتا ہے، جو Windows AI Foundry کی بنیاد کے طور پر کام کرتا ہے۔ یہ پلیٹ فارم ڈویلپرز کو اے آئی سے چلنے والی Windows ایپلیکیشنز بنانے کے قابل بناتا ہے جو PC ہارڈویئر کی مکمل صلاحیتوں کا فائدہ اٹھاتی ہیں۔

### پلیٹ فارم کی صلاحیتیں
- تمام Windows 11 PCs پر کام کرتا ہے جو ورژن 24H2 (بلڈ 26100) یا اس سے زیادہ پر چل رہے ہیں
- تمام x64 اور ARM64 PC ہارڈویئر پر کام کرتا ہے، یہاں تک کہ وہ PCs جن میں NPUs یا GPUs نہیں ہیں
- ڈویلپرز کو اپنے ماڈلز لانے اور انہیں AMD، Intel، NVIDIA، اور Qualcomm کے سلیکون پارٹنر ایکو سسٹم میں مؤثر طریقے سے تعینات کرنے کے قابل بناتا ہے
- انفراسٹرکچر APIs کا فائدہ اٹھاتے ہوئے، ڈویلپرز کو مختلف سلیکون کو ہدف بنانے کے لیے اپنی ایپ کے متعدد بلڈز بنانے کی ضرورت نہیں ہے

### ڈویلپر فوائد
Windows ML ہارڈویئر اور ایکزیکیوشن پرووائیڈرز کو خلاصہ کرتا ہے، تاکہ آپ اپنے کوڈ پر توجہ مرکوز کر سکیں۔ اس کے علاوہ، Windows ML خود بخود اپ ڈیٹ ہوتا ہے تاکہ تازہ ترین NPUs، GPUs، اور CPUs کی حمایت کی جا سکے جیسے ہی وہ جاری کیے جاتے ہیں۔ یہ پلیٹ فارم Windows ہارڈویئر ایکو سسٹم میں اے آئی کی ترقی کے لیے ایک متحد فریم ورک فراہم کرتا ہے۔

**مزید جانیں**: 
- [Windows ML Overview](https://learn.microsoft.com/en-us/windows/ai/new-windows-ml/overview)
- [Windows EdgeAI Development Guide](../windowdeveloper.md) - Windows Edge AI ترقی کے لیے جامع گائیڈ

## 5. Foundry Local ایپلیکیشنز کے ساتھ ایج اے آئی

Foundry Local ڈویلپرز کو .NET میں Retrieval Augmented Generation (RAG) ایپلیکیشنز بنانے کے قابل بناتا ہے، جو مقامی وسائل کو استعمال کرتے ہوئے مقامی زبان ماڈلز کو سیمینٹک سرچ صلاحیتوں کے ساتھ جوڑتا ہے۔ یہ طریقہ پرائیویسی پر مبنی اے آئی حل فراہم کرتا ہے جو مکمل طور پر مقامی انفراسٹرکچر پر کام کرتے ہیں۔

### تکنیکی آرکیٹیکچر
- Phi-3 زبان ماڈل، مقامی ایمبیڈنگز، اور سیمینٹک کرنل کو RAG منظرنامہ بنانے کے لیے جوڑتا ہے
- ایمبیڈنگز کو فلوٹنگ پوائنٹ ویلیوز کے ویکٹرز (ایرے) کے طور پر استعمال کرتا ہے جو مواد اور اس کے سیمینٹک معنی کی نمائندگی کرتے ہیں
- سیمینٹک کرنل مرکزی آرکیسٹریٹر کے طور پر کام کرتا ہے، Phi-3 اور اسمارٹ کمپوننٹس کو مربوط کرتا ہے تاکہ ایک ہموار RAG پائپ لائن بنائی جا سکے
- مقامی ویکٹر ڈیٹا بیسز کی حمایت، بشمول SQLite اور Qdrant

### نفاذ کے فوائد
RAG، یا Retrieval Augmented Generation، صرف "کچھ معلومات تلاش کریں اور اسے پرامپٹ میں ڈالیں" کہنے کا ایک فینسی طریقہ ہے۔ یہ مقامی نفاذ ڈیٹا پرائیویسی کو یقینی بناتا ہے جبکہ ذہین جوابات فراہم کرتا ہے جو کسٹم نالج بیسز پر مبنی ہوتے ہیں۔ یہ طریقہ خاص طور پر انٹرپرائز منظرناموں کے لیے قیمتی ہے جن میں ڈیٹا خودمختاری اور آف لائن آپریشن کی صلاحیتیں درکار ہوتی ہیں۔

**مزید جانیں**: [Foundry Local RAG Samples](https://github.com/microsoft/Foundry-Local/tree/main/samples/dotNET/rag)

## Windows EdgeAI ترقی کے وسائل

خاص طور پر Windows پلیٹ فارم کو ہدف بنانے والے ڈویلپرز کے لیے، ہم نے ایک جامع گائیڈ تیار کی ہے جو مکمل Windows EdgeAI ایکو سسٹم کا احاطہ کرتی ہے۔ یہ وسیلہ Windows AI Foundry کے بارے میں تفصیلی معلومات فراہم کرتا ہے، بشمول APIs، ٹولز، اور Windows پر EdgeAI ترقی کے لیے بہترین طریقے۔

### Windows AI Foundry پلیٹ فارم
Windows AI Foundry پلیٹ فارم Edge AI ترقی کے لیے خاص طور پر ڈیزائن کیے گئے ٹولز اور APIs کا ایک جامع سوٹ فراہم کرتا ہے۔ اس میں NPU-تیز ہارڈویئر، Windows ML انضمام، اور پلیٹ فارم مخصوص اصلاحی تکنیکوں کے لیے خصوصی حمایت شامل ہے۔

**جامع گائیڈ**: [Windows EdgeAI Development Guide](../windowdeveloper.md)

یہ گائیڈ شامل کرتی ہے:
- Windows AI Foundry پلیٹ فارم کا جائزہ اور اجزاء
- NPU ہارڈویئر پر مؤثر انفرنس کے لیے Phi Silica API
- امیج پروسیسنگ اور OCR کے لیے کمپیوٹر وژن APIs
- Windows ML رن ٹائم انضمام اور اصلاح
- مقامی ترقی اور جانچ کے لیے Foundry Local CLI
- Windows ڈیوائسز کے لیے ہارڈویئر اصلاحی حکمت عملی
- عملی نفاذ کی مثالیں اور بہترین طریقے

### Edge AI ترقی کے لیے AI ٹول کٹ
Visual Studio Code استعمال کرنے والے ڈویلپرز کے لیے، AI Toolkit ایکسٹینشن Edge AI ایپلیکیشنز بنانے، جانچنے، اور تعینات کرنے کے لیے ایک جامع ترقیاتی ماحول فراہم کرتی ہے۔ یہ ٹول کٹ VS Code کے اندر Edge AI ترقی کے ورک فلو کو آسان بناتی ہے۔

**ترقیاتی گائیڈ**: [AI Toolkit for Edge AI Development](../aitoolkit.md)

AI Toolkit گائیڈ شامل کرتی ہے:
- ایج تعیناتی کے لیے ماڈل دریافت اور انتخاب
- مقامی جانچ اور اصلاحی ورک فلو
- ایج ماڈلز کے لیے ONNX اور Ollama انضمام
- ماڈل کنورژن اور کوانٹائزیشن تکنیک
- ایج منظرناموں کے لیے ایجنٹ کی ترقی
- کارکردگی کی تشخیص اور نگرانی
- تعیناتی کی تیاری اور بہترین طریقے

## نتیجہ

یہ پانچ ایج اے آئی نفاذ آج دستیاب ایج اے آئی حلوں کی پختگی اور تنوع کو ظاہر کرتے ہیں۔ ہارڈویئر سے تیز ایج ڈیوائسز جیسے Jetson Orin Nano سے لے کر سافٹ ویئر فریم ورکز جیسے ONNX Runtime GenAI اور Windows ML تک، ڈویلپرز کے پاس ایج پر ذہین ایپلیکیشنز تعینات کرنے کے لیے بے مثال اختیارات ہیں۔

ان تمام پلیٹ فارمز میں مشترکہ دھاگہ اے آئی صلاحیتوں کی جمہوریت ہے، جو مختلف مہارت کی سطحوں اور استعمال کے کیسز کے ڈویلپرز کے لیے جدید مشین لرننگ کو قابل رسائی بناتی ہے۔ چاہے موبائل ایپلیکیشنز، ڈیسک ٹاپ سافٹ ویئر، یا ایمبیڈڈ سسٹمز بنانا ہو، یہ ایج اے آئی حل اگلی نسل کی ذہین ایپلیکیشنز کے لیے بنیاد فراہم کرتے ہیں جو ایج پر مؤثر اور نجی طور پر کام کرتی ہیں۔

ہر پلیٹ فارم منفرد فوائد پیش کرتا ہے: Jetson Orin Nano ہارڈویئر سے تیز ایج کمپیوٹنگ کے لیے، ONNX Runtime GenAI کراس پلیٹ فارم موبائل ترقی کے لیے، Azure EdgeAI انٹرپرائز کلاؤڈ-ایج انضمام کے لیے، Windows ML Windows-نیٹو ایپلیکیشنز کے لیے، اور Foundry Local پرائیویسی پر مبنی RAG نفاذ کے لیے۔ یہ سب مل کر ایج اے آئی ترقی کے لیے ایک جامع ایکو سسٹم کی نمائندگی کرتے ہیں۔
