# Session 4: Banbrytande Modeller – LLMs, SLMs och Inference på Enheten

## Översikt

Jämför LLMs och SLMs, utvärdera för- och nackdelar med lokal kontra molnbaserad inference, och implementera demos som visar EdgeAI-scenarier med Phi och ONNX Runtime. Vi kommer också att lyfta fram Chainlit RAG, WebGPU-inference-alternativ och integration med Open WebUI.

Referenser:
- Foundry Local-dokumentation: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Open WebUI-guide (chattapp med Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Lärandemål
- Förstå för- och nackdelar med LLM och SLM när det gäller kostnad, latens och noggrannhet
- Välja mellan lokal och molnbaserad inference för specifika affärsbehov
- Implementera en enkel RAG-demo med Chainlit
- Utforska WebGPU för acceleration i webbläsaren
- Ansluta Open WebUI till Foundry Local

## Del 1: LLM vs SLM – Beslutsmatris

Att tänka på:
- Latens: SLMs på enheten levererar ofta svar på under en sekund
- Kostnad: lokal inference minskar molnkostnader
- Sekretess: känslig data stannar på enheten
- Kapacitet: LLMs kan överträffa SLMs i komplexa uppgifter
- Tillförlitlighet: hybridstrategier minskar risken för driftstopp

## Del 2: Lokal kontra Moln – Hybridmönster

- Lokal först med moln som reserv för stora/komplexa frågor
- Moln först med lokal för sekretesskänsliga eller offline-scenarier
- Rutt efter uppgiftstyp (kodgenerering till DeepSeek, generell chatt till Phi/Qwen)

## Del 3: RAG Chattapp med Chainlit (Minimal)

Installera beroenden:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Kör:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Utöka: lägg till en enkel sökfunktion (lokala filer) och lägg till den hämtade kontexten i användarens fråga.

## Del 4: WebGPU Inference (Förberedelse)

Kör små modeller direkt i webbläsaren med WebGPU. Detta är idealiskt för sekretessfokuserade demos och upplevelser utan installation. Nedan finns ett minimalt, steg-för-steg-exempel med ONNX Runtime Web och WebGPU-exekveringsleverantören.

1) Kontrollera WebGPU-stöd
- Chromium-webbläsare: chrome://gpu → bekräfta att “WebGPU” är aktiverat
- Programmatisk kontroll (vi kommer också att kontrollera i kod): `if (!('gpu' in navigator)) { /* ingen WebGPU */ }`

2) Skapa ett minimalt projekt
Skapa en mapp och två filer: `index.html` och `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Kör lokalt (Windows cmd.exe)
Använd en enkel statisk server så att webbläsaren kan hämta modellen.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Öppna http://localhost:5173 i din webbläsare. Du bör se initialiseringsloggar, sessionens skapande med WebGPU och en argmax-prediktion.

4) Felsökning
- Om WebGPU inte är tillgängligt: uppdatera Chrome/Edge och se till att GPU-drivrutiner är aktuella, kontrollera sedan chrome://flags för “Enable WebGPU”.
- Om CORS- eller fetch-fel uppstår: se till att du serverar filer via http:// (inte file://) och att modellens URL tillåter cross-origin-förfrågningar.
- Reserv till CPU: ändra `executionProviders: ['wasm']` för att verifiera grundläggande funktionalitet.

5) Nästa steg
- Byt ut till en domänspecifik ONNX-modell (t.ex. bildklassificering eller en liten textmodell).
- Lägg till logik för förbearbetning/efterbearbetning för riktiga indata.
- För större modeller eller produktionslatens, föredra Foundry Local eller ONNX Runtime Server.

## Del 5: Open WebUI + Foundry Local (Steg-för-steg)

Detta ansluter Open WebUI till Foundry Locals OpenAI-kompatibla endpoint för en lokal chatt-UI.

1) Förutsättningar
- Foundry Local installerat och fungerar (`foundry --version`)
- En modell redo att köras lokalt (t.ex. `phi-4-mini`)
- Docker Desktop installerat (rekommenderas för Open WebUI)

2) Starta en modell med Foundry Local
```powershell
foundry model run phi-4-mini
```
Detta exponerar en OpenAI-kompatibel API på `http://localhost:8000`.

3) Starta Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Noteringar:
- På Windows kan `host.docker.internal` låta containern nå din värd på `localhost`.
- Vi ställer in `OPENAI_API_BASE_URL` till Foundry Locals endpoint och en dummy `OPENAI_API_KEY`.

4) Konfigurera från Open WebUI UI (alternativ)
- Gå till http://localhost:3000
- Slutför den initiala inställningen (admin-användare)
- Gå till Inställningar → Modeller/Leverantörer
- Ställ in Bas-URL: `http://host.docker.internal:8000/v1`
- Ställ in API-nyckel: `local-key` (platshållare)
- Spara

5) Kör en testfråga
- I Open WebUI-chatt, välj eller ange modellnamn `phi-4-mini`
- Fråga: “Lista fem fördelar med AI-inference på enheten.”
- Du bör se ett svar strömmas från din lokala modell

6) Felsökning
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Valfritt: Spara Open WebUI-data
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
