## บทนำ

ยินดีต้อนรับสู่คู่มือการศึกษา EdgeAI สำหรับผู้เริ่มต้น! เอกสารนี้ออกแบบมาเพื่อช่วยให้คุณสำรวจเนื้อหาหลักสูตรได้อย่างมีประสิทธิภาพและเพิ่มประสบการณ์การเรียนรู้ของคุณให้สูงสุด โดยมีเส้นทางการเรียนรู้ที่มีโครงสร้าง ตารางการศึกษาแนะนำ สรุปแนวคิดสำคัญ และทรัพยากรเสริมเพื่อเพิ่มความเข้าใจในเทคโนโลยี EdgeAI

นี่คือหลักสูตรที่กระชับ ใช้เวลา 20 ชั่วโมง ที่มอบความรู้สำคัญเกี่ยวกับ EdgeAI ในรูปแบบที่ประหยัดเวลา เหมาะสำหรับมืออาชีพและนักเรียนที่มีเวลาจำกัดและต้องการพัฒนาทักษะในสาขานี้อย่างรวดเร็ว

## ภาพรวมหลักสูตร

หลักสูตรนี้แบ่งออกเป็นเจ็ดโมดูลที่ครอบคลุม:

1. **พื้นฐานและการเปลี่ยนแปลงของ EdgeAI** - ทำความเข้าใจแนวคิดหลักและการเปลี่ยนแปลงทางเทคโนโลยี
2. **พื้นฐานของ Small Language Model (SLM)** - สำรวจครอบครัว SLM และสถาปัตยกรรมของพวกเขา
3. **การปรับใช้ Small Language Model** - การปรับใช้ในเชิงปฏิบัติ
4. **การแปลงรูปแบบโมเดลและการลดขนาด** - การปรับแต่งขั้นสูงด้วย 6 เฟรมเวิร์ก รวมถึง OpenVINO
5. **SLMOps - การดำเนินการ Small Language Model** - การจัดการวงจรการผลิตและการปรับใช้
6. **ระบบ SLM Agentic** - AI agents, การเรียกฟังก์ชัน และ Model Context Protocol
7. **ตัวอย่างการใช้งาน EdgeAI** - AI Toolkit, การพัฒนาบน Windows และการปรับใช้เฉพาะแพลตฟอร์ม
8. **Microsoft Foundry Local – เครื่องมือพัฒนาครบวงจร** - การพัฒนาที่เน้นในพื้นที่พร้อมการผสาน Azure แบบไฮบริด (โมดูล 08)

## วิธีใช้คู่มือการศึกษา

- **การเรียนรู้แบบก้าวหน้า**: เรียนตามลำดับโมดูลเพื่อประสบการณ์การเรียนรู้ที่ต่อเนื่อง
- **จุดตรวจสอบความรู้**: ใช้คำถามประเมินตนเองหลังแต่ละส่วน
- **การฝึกปฏิบัติ**: ทำแบบฝึกหัดที่แนะนำเพื่อเสริมสร้างแนวคิดเชิงทฤษฎี
- **ทรัพยากรเสริม**: สำรวจเนื้อหาเพิ่มเติมสำหรับหัวข้อที่คุณสนใจมากที่สุด

## โมดูล 1: พื้นฐานและการเปลี่ยนแปลงของ EdgeAI

### วัตถุประสงค์การเรียนรู้สำคัญ

- เข้าใจความแตกต่างระหว่าง AI บนคลาวด์และ AI บน Edge
- เชี่ยวชาญเทคนิคการปรับแต่งหลักสำหรับสภาพแวดล้อมที่มีทรัพยากรจำกัด
- วิเคราะห์การใช้งานจริงของเทคโนโลยี EdgeAI
- ตั้งค่าสภาพแวดล้อมการพัฒนาสำหรับโปรเจกต์ EdgeAI

### จุดเน้นการศึกษา

#### ส่วนที่ 1: พื้นฐาน EdgeAI
- **แนวคิดสำคัญ**: 
  - การเปรียบเทียบระหว่าง Edge และ Cloud computing
  - เทคนิคการลดขนาดโมเดล
  - ตัวเลือกการเร่งความเร็วฮาร์ดแวร์ (NPUs, GPUs, CPUs)
  - ข้อดีด้านความเป็นส่วนตัวและความปลอดภัย

- **ทรัพยากรเสริม**:
  - [TensorFlow Lite Documentation](https://www.tensorflow.org/lite)
  - [ONNX Runtime GitHub](https://github.com/microsoft/onnxruntime)
  - [Edge Impulse Documentation](https://docs.edgeimpulse.com)

#### ส่วนที่ 2: กรณีศึกษาในโลกจริง
- **แนวคิดสำคัญ**: 
  - ระบบ Microsoft Phi & Mu model
  - การใช้งานจริงในอุตสาหกรรมต่าง ๆ
  - ข้อควรพิจารณาในการปรับใช้

#### ส่วนที่ 3: คู่มือการใช้งานในเชิงปฏิบัติ
- **แนวคิดสำคัญ**: 
  - การตั้งค่าสภาพแวดล้อมการพัฒนา
  - เครื่องมือการลดขนาดและการปรับแต่ง
  - วิธีการประเมินการใช้งาน EdgeAI

#### ส่วนที่ 4: ฮาร์ดแวร์สำหรับการปรับใช้ Edge
- **แนวคิดสำคัญ**: 
  - การเปรียบเทียบแพลตฟอร์มฮาร์ดแวร์
  - กลยุทธ์การปรับแต่งสำหรับฮาร์ดแวร์เฉพาะ
  - ข้อควรพิจารณาในการปรับใช้

### คำถามประเมินตนเอง

1. เปรียบเทียบและอธิบายความแตกต่างระหว่าง AI บนคลาวด์และ AI บน Edge
2. อธิบายเทคนิคสำคัญสามประการสำหรับการปรับแต่งโมเดลเพื่อการปรับใช้บน Edge
3. ข้อดีหลักของการรันโมเดล AI บน Edge คืออะไร?
4. อธิบายกระบวนการลดขนาดโมเดลและผลกระทบต่อประสิทธิภาพ
5. อธิบายว่าตัวเร่งความเร็วฮาร์ดแวร์ (NPUs, GPUs, CPUs) มีผลต่อการปรับใช้ EdgeAI อย่างไร

### แบบฝึกหัด

1. **การตั้งค่าสภาพแวดล้อมอย่างรวดเร็ว**: ตั้งค่าสภาพแวดล้อมการพัฒนาขั้นต่ำด้วยแพ็กเกจที่จำเป็น (30 นาที)
2. **การสำรวจโมเดล**: ดาวน์โหลดและตรวจสอบโมเดลภาษาเล็กที่ผ่านการฝึกมาแล้ว (1 ชั่วโมง)
3. **การลดขนาดพื้นฐาน**: ลองลดขนาดโมเดลเล็ก ๆ (1 ชั่วโมง)

## โมดูล 2: พื้นฐานของ Small Language Model

### วัตถุประสงค์การเรียนรู้สำคัญ

- เข้าใจหลักการสถาปัตยกรรมของครอบครัว SLM ต่าง ๆ
- เปรียบเทียบความสามารถของโมเดลในระดับพารามิเตอร์ต่าง ๆ
- ประเมินโมเดลตามประสิทธิภาพ ความสามารถ และข้อกำหนดการปรับใช้
- ระบุกรณีการใช้งานที่เหมาะสมสำหรับครอบครัวโมเดลต่าง ๆ

### จุดเน้นการศึกษา

#### ส่วนที่ 1: ครอบครัวโมเดล Microsoft Phi
- **แนวคิดสำคัญ**: 
  - วิวัฒนาการปรัชญาการออกแบบ
  - สถาปัตยกรรมที่เน้นประสิทธิภาพ
  - ความสามารถเฉพาะทาง

#### ส่วนที่ 2: ครอบครัว Qwen
- **แนวคิดสำคัญ**: 
  - การมีส่วนร่วมในโอเพ่นซอร์ส
  - ตัวเลือกการปรับใช้ที่ปรับขนาดได้
  - สถาปัตยกรรมการให้เหตุผลขั้นสูง

#### ส่วนที่ 3: ครอบครัว Gemma
- **แนวคิดสำคัญ**: 
  - นวัตกรรมที่ขับเคลื่อนด้วยการวิจัย
  - ความสามารถมัลติโหมด
  - การปรับแต่งสำหรับมือถือ

#### ส่วนที่ 4: ครอบครัว BitNET
- **แนวคิดสำคัญ**: 
  - เทคโนโลยีการลดขนาด 1-bit
  - เฟรมเวิร์กการปรับแต่งการอนุมาน
  - ข้อพิจารณาด้านความยั่งยืน

#### ส่วนที่ 5: โมเดล Microsoft Mu
- **แนวคิดสำคัญ**: 
  - สถาปัตยกรรมที่เน้นอุปกรณ์
  - การผสานระบบกับ Windows
  - การดำเนินการที่รักษาความเป็นส่วนตัว

#### ส่วนที่ 6: Phi-Silica
- **แนวคิดสำคัญ**: 
  - สถาปัตยกรรมที่ปรับแต่งสำหรับ NPU
  - ตัวชี้วัดประสิทธิภาพ
  - การผสานรวมสำหรับนักพัฒนา

### คำถามประเมินตนเอง

1. เปรียบเทียบแนวทางสถาปัตยกรรมของครอบครัวโมเดล Phi และ Qwen
2. อธิบายว่าเทคโนโลยีการลดขนาดของ BitNET แตกต่างจากการลดขนาดแบบดั้งเดิมอย่างไร
3. ข้อดีเฉพาะของโมเดล Mu สำหรับการผสานรวมกับ Windows คืออะไร?
4. อธิบายว่า Phi-Silica ใช้ฮาร์ดแวร์ NPU เพื่อเพิ่มประสิทธิภาพอย่างไร
5. สำหรับแอปพลิเคชันมือถือที่มีการเชื่อมต่อจำกัด ครอบครัวโมเดลใดที่เหมาะสมที่สุดและเพราะเหตุใด?

### แบบฝึกหัด

1. **การเปรียบเทียบโมเดล**: ทดสอบประสิทธิภาพของโมเดล SLM สองตัว (1 ชั่วโมง)
2. **การสร้างข้อความพื้นฐาน**: การใช้งานพื้นฐานของการสร้างข้อความด้วยโมเดลเล็ก (1 ชั่วโมง)
3. **การปรับแต่งอย่างรวดเร็ว**: ใช้เทคนิคการปรับแต่งหนึ่งอย่างเพื่อปรับปรุงความเร็วในการอนุมาน (1 ชั่วโมง)

## โมดูล 3: การปรับใช้ Small Language Model

### วัตถุประสงค์การเรียนรู้สำคัญ

- เลือกโมเดลที่เหมาะสมตามข้อจำกัดการปรับใช้
- เชี่ยวชาญเทคนิคการปรับแต่งสำหรับสถานการณ์การปรับใช้ต่าง ๆ
- ใช้งาน SLM ในทั้งสภาพแวดล้อมท้องถิ่นและคลาวด์
- ออกแบบการกำหนดค่าที่พร้อมสำหรับการผลิตสำหรับแอปพลิเคชัน EdgeAI

### จุดเน้นการศึกษา

#### ส่วนที่ 1: การเรียนรู้ขั้นสูงของ SLM
- **แนวคิดสำคัญ**: 
  - กรอบการจำแนกพารามิเตอร์
  - เทคนิคการปรับแต่งขั้นสูง
  - กลยุทธ์การได้มาของโมเดล

#### ส่วนที่ 2: การปรับใช้ในสภาพแวดล้อมท้องถิ่น
- **แนวคิดสำคัญ**: 
  - การปรับใช้แพลตฟอร์ม Ollama
  - โซลูชัน Microsoft Foundry Local
  - การวิเคราะห์เปรียบเทียบเฟรมเวิร์ก

#### ส่วนที่ 3: การปรับใช้คลาวด์แบบคอนเทนเนอร์
- **แนวคิดสำคัญ**: 
  - การอนุมานประสิทธิภาพสูงด้วย vLLM
  - การจัดการคอนเทนเนอร์
  - การใช้งาน ONNX Runtime

### คำถามประเมินตนเอง

1. ปัจจัยใดที่ควรพิจารณาเมื่อเลือกการปรับใช้ระหว่างสภาพแวดล้อมท้องถิ่นและคลาวด์?
2. เปรียบเทียบ Ollama และ Microsoft Foundry Local ในฐานะตัวเลือกการปรับใช้
3. อธิบายข้อดีของการใช้คอนเทนเนอร์สำหรับการปรับใช้ SLM
4. ตัวชี้วัดประสิทธิภาพหลักที่ควรตรวจสอบสำหรับ SLM ที่ปรับใช้บน Edge คืออะไร?
5. อธิบายขั้นตอนการปรับใช้ตั้งแต่การเลือกโมเดลจนถึงการใช้งานในผลิตภัณฑ์

### แบบฝึกหัด

1. **การปรับใช้ท้องถิ่น
- **แนวคิดสำคัญ**:  
  - การปรับแต่งเพื่อ Apple Silicon  
  - สถาปัตยกรรมหน่วยความจำแบบรวม  
  - ความสามารถในการปรับแต่ง LoRA  

#### ส่วนที่ 6: การสังเคราะห์เวิร์กโฟลว์การพัฒนา Edge AI  
- **แนวคิดสำคัญ**:  
  - สถาปัตยกรรมเวิร์กโฟลว์แบบรวม  
  - การตัดสินใจเลือกเฟรมเวิร์ก  
  - การตรวจสอบความพร้อมสำหรับการผลิต  
  - กลยุทธ์การเตรียมพร้อมสำหรับอนาคต  

### คำถามประเมินตนเอง  

1. เปรียบเทียบกลยุทธ์การปรับลดความแม่นยำในระดับต่าง ๆ (1-bit ถึง 8-bit)  
2. อธิบายข้อดีของรูปแบบ GGUF สำหรับการใช้งานใน Edge  
3. การปรับแต่งฮาร์ดแวร์ใน Microsoft Olive ช่วยเพิ่มประสิทธิภาพการใช้งานได้อย่างไร?  
4. ประโยชน์สำคัญของ OpenVINO's NNCF สำหรับการบีบอัดโมเดลคืออะไร?  
5. Apple MLX ใช้สถาปัตยกรรมหน่วยความจำแบบรวมเพื่อการปรับแต่งอย่างไร?  
6. การสังเคราะห์เวิร์กโฟลว์ช่วยในการเลือกเฟรมเวิร์กการปรับแต่งที่เหมาะสมได้อย่างไร?  

### แบบฝึกหัดปฏิบัติ  

1. **การปรับลดความแม่นยำของโมเดล**: ใช้ระดับการปรับลดความแม่นยำต่าง ๆ กับโมเดลและเปรียบเทียบผลลัพธ์ (1 ชั่วโมง)  
2. **การปรับแต่งด้วย OpenVINO**: ใช้ NNCF เพื่อบีบอัดโมเดลสำหรับฮาร์ดแวร์ Intel (1 ชั่วโมง)  
3. **การเปรียบเทียบเฟรมเวิร์ก**: ทดสอบโมเดลเดียวกันในเฟรมเวิร์กการปรับแต่งสามแบบ (1 ชั่วโมง)  
4. **การวัดประสิทธิภาพ**: วัดผลกระทบของการปรับแต่งต่อความเร็วในการประมวลผลและการใช้หน่วยความจำ (1 ชั่วโมง)  

## โมดูล 5: SLMOps - การดำเนินงานโมเดลภาษาขนาดเล็ก  

### วัตถุประสงค์การเรียนรู้หลัก  

- เข้าใจหลักการจัดการวงจรชีวิตของ SLMOps  
- เชี่ยวชาญเทคนิคการกลั่นและการปรับแต่งสำหรับการใช้งานใน Edge  
- ใช้กลยุทธ์การปรับใช้ในระดับการผลิตพร้อมการตรวจสอบ  
- สร้างเวิร์กโฟลว์การดำเนินงานและการบำรุงรักษา SLM ระดับองค์กร  

### พื้นที่การศึกษา  

#### ส่วนที่ 1: บทนำสู่ SLMOps  
- **แนวคิดสำคัญ**:  
  - การเปลี่ยนแปลงกระบวนทัศน์ของ SLMOps ในการดำเนินงาน AI  
  - สถาปัตยกรรมที่คุ้มค่าและเน้นความเป็นส่วนตัว  
  - ผลกระทบเชิงกลยุทธ์ต่อธุรกิจและข้อได้เปรียบทางการแข่งขัน  

#### ส่วนที่ 2: การกลั่นโมเดล  
- **แนวคิดสำคัญ**:  
  - เทคนิคการถ่ายโอนความรู้  
  - การดำเนินการกระบวนการกลั่นสองขั้นตอน  
  - เวิร์กโฟลว์การกลั่นใน Azure ML  

#### ส่วนที่ 3: กลยุทธ์การปรับแต่ง  
- **แนวคิดสำคัญ**:  
  - การปรับแต่งที่มีประสิทธิภาพต่อพารามิเตอร์ (PEFT)  
  - วิธีการขั้นสูง LoRA และ QLoRA  
  - การฝึกอบรมแบบหลายอะแดปเตอร์และการปรับแต่งไฮเปอร์พารามิเตอร์  

#### ส่วนที่ 4: การปรับใช้ในระดับการผลิต  
- **แนวคิดสำคัญ**:  
  - การแปลงและการปรับลดความแม่นยำของโมเดลสำหรับการผลิต  
  - การกำหนดค่าการปรับใช้ Foundry Local  
  - การวัดประสิทธิภาพและการตรวจสอบคุณภาพ  

### คำถามประเมินตนเอง  

1. SLMOps แตกต่างจาก MLOps แบบดั้งเดิมอย่างไร?  
2. อธิบายประโยชน์ของการกลั่นโมเดลสำหรับการใช้งานใน Edge  
3. ข้อควรพิจารณาสำคัญสำหรับการปรับแต่ง SLM ในสภาพแวดล้อมที่มีทรัพยากรจำกัดคืออะไร?  
4. อธิบายขั้นตอนการปรับใช้ในระดับการผลิตสำหรับแอปพลิเคชัน AI ใน Edge  

### แบบฝึกหัดปฏิบัติ  

1. **การกลั่นพื้นฐาน**: สร้างโมเดลขนาดเล็กจากโมเดลครูขนาดใหญ่ (1 ชั่วโมง)  
2. **การทดลองปรับแต่ง**: ปรับแต่งโมเดลสำหรับโดเมนเฉพาะ (1 ชั่วโมง)  
3. **เวิร์กโฟลว์การปรับใช้**: ตั้งค่าเวิร์กโฟลว์ CI/CD พื้นฐานสำหรับการปรับใช้โมเดล (1 ชั่วโมง)  

## โมดูล 6: ระบบ SLM Agentic - ตัวแทน AI และการเรียกฟังก์ชัน  

### วัตถุประสงค์การเรียนรู้หลัก  

- สร้างตัวแทน AI อัจฉริยะสำหรับสภาพแวดล้อม Edge โดยใช้โมเดลภาษาขนาดเล็ก  
- ใช้ความสามารถในการเรียกฟังก์ชันด้วยเวิร์กโฟลว์ที่เป็นระบบ  
- เชี่ยวชาญการรวม Model Context Protocol (MCP) เพื่อการโต้ตอบเครื่องมือที่ได้มาตรฐาน  
- สร้างระบบตัวแทนที่ซับซ้อนด้วยการแทรกแซงของมนุษย์น้อยที่สุด  

### พื้นที่การศึกษา  

#### ส่วนที่ 1: ตัวแทน AI และพื้นฐาน SLM  
- **แนวคิดสำคัญ**:  
  - กรอบการจำแนกประเภทตัวแทน (ตัวแทนสะท้อน, ตัวแทนตามโมเดล, ตัวแทนตามเป้าหมาย, ตัวแทนเรียนรู้)  
  - การวิเคราะห์ข้อแลกเปลี่ยนระหว่าง SLM และ LLM  
  - รูปแบบการออกแบบตัวแทนเฉพาะ Edge  
  - การเพิ่มประสิทธิภาพทรัพยากรสำหรับตัวแทน  

#### ส่วนที่ 2: การเรียกฟังก์ชันในโมเดลภาษาขนาดเล็ก  
- **แนวคิดสำคัญ**:  
  - การดำเนินการเวิร์กโฟลว์อย่างเป็นระบบ (การตรวจจับเจตนา, เอาต์พุต JSON, การดำเนินการภายนอก)  
  - การดำเนินการเฉพาะแพลตฟอร์ม (Phi-4-mini, โมเดล Qwen ที่เลือก, Microsoft Foundry Local)  
  - ตัวอย่างขั้นสูง (การทำงานร่วมกันของตัวแทนหลายตัว, การเลือกเครื่องมือแบบไดนามิก)  
  - ข้อควรพิจารณาสำหรับการผลิต (การจำกัดอัตรา, การบันทึกการตรวจสอบ, มาตรการรักษาความปลอดภัย)  

#### ส่วนที่ 3: การรวม Model Context Protocol (MCP)  
- **แนวคิดสำคัญ**:  
  - สถาปัตยกรรมโปรโตคอลและการออกแบบระบบแบบเลเยอร์  
  - การสนับสนุนหลายแบ็กเอนด์ (Ollama สำหรับการพัฒนา, vLLM สำหรับการผลิต)  
  - โปรโตคอลการเชื่อมต่อ (โหมด STDIO และ SSE)  
  - การใช้งานจริง (การทำงานอัตโนมัติบนเว็บ, การประมวลผลข้อมูล, การรวม API)  

### คำถามประเมินตนเอง  

1. ข้อควรพิจารณาด้านสถาปัตยกรรมที่สำคัญสำหรับตัวแทน AI ใน Edge คืออะไร?  
2. การเรียกฟังก์ชันช่วยเพิ่มความสามารถของตัวแทนได้อย่างไร?  
3. อธิบายบทบาทของ Model Context Protocol ในการสื่อสารของตัวแทน  

### แบบฝึกหัดปฏิบัติ  

1. **ตัวแทนง่าย ๆ**: สร้างตัวแทน AI พื้นฐานพร้อมการเรียกฟังก์ชัน (1 ชั่วโมง)  
2. **การรวม MCP**: ใช้ MCP ในแอปพลิเคชันตัวแทน (30 นาที)  

## โมดูล 7: ตัวอย่างการใช้งาน EdgeAI  

### วัตถุประสงค์การเรียนรู้หลัก  

- เชี่ยวชาญ AI Toolkit สำหรับ Visual Studio Code เพื่อเวิร์กโฟลว์การพัฒนา EdgeAI ที่ครอบคลุม

- ใช้ EdgeAI ในหลายแพลตฟอร์มฮาร์ดแวร์และสถานการณ์การปรับใช้  
- สร้างแอปพลิเคชัน EdgeAI ที่พร้อมสำหรับการผลิตด้วยการปรับแต่งเฉพาะแพลตฟอร์ม  

### พื้นที่การศึกษา

- **แนวคิดสำคัญ**:  
  - สภาพแวดล้อมการพัฒนา Edge AI ที่ครอบคลุมภายใน VS Code

6. เปรียบเทียบข้อดีของการปรับใช้ในเครื่องกับคลาวด์สำหรับแอปพลิเคชันที่เน้นความเป็นส่วนตัว  

### แบบฝึกหัดปฏิบัติ  

1. **การตั้งค่า AI Toolkit**: กำหนดค่า AI Toolkit และปรับแต่งโมเดล (1 ชั่วโมง)  
2. **Windows AI Foundry**: สร้างแอปพลิเคชัน Windows AI ง่าย ๆ โดยใช้ Phi Silica API (1 ชั่วโมง)  
3. **การปรับใช้ข้ามแพลตฟอร์ม**: ปรับใช้โมเดลเดียวกันในสองแพลตฟอร์มที่แตกต่างกัน (1 ชั่วโมง)  
4. **การเพิ่มประสิทธิภาพ NPU**: ทดสอบประสิทธิภาพ NPU ด้วยเครื่องมือ Windows AI Foundry (30 นาที)  

## โมดูล 8: Microsoft Foundry Local – ชุดเครื่องมือสำหรับนักพัฒนาที่สมบูรณ์  

### วัตถุประสงค์การเรียนรู้หลัก  

- ติดตั้งและกำหนดค่า Foundry Local บน Windows  
- เรียกใช้, ค้นหา, และจัดการโมเดลในเครื่องผ่าน Foundry CLI  
- รวมเข้ากับ REST และ SDK ที่เข้ากันได้กับ OpenAI  
- สร้างตัวอย่างการใช้งานจริง: Chainlit chat, ตัวแทน, และตัวเลือกโมเดล  
- เข้าใจรูปแบบไฮบริดด้วย Azure AI Foundry  

### พื้นที่การศึกษา  

- การติดตั้งและพื้นฐาน CLI (โมเดล, บริการ, แคช)  
- การรวม SDK (ไคลเอนต์ที่เข้ากันได้กับ OpenAI และ Azure OpenAI)  
- การตรวจสอบความพร้อมใช้งาน Open WebUI  
- รูปแบบตัวแทนและการเรียกฟังก์ชัน  
- โมเดลเป็นเครื่องมือ (การออกแบบตัวเลือกและการลงทะเบียน)  

### คำถามประเมินตนเอง  

1. คุณค้นหา endpoint ในเครื่องและรายการโมเดลที่มีอยู่ได้อย่างไร?  
2. ความแตกต่างระหว่างการใช้งาน Foundry Local REST และ Azure OpenAI คืออะไร?  
3. คุณจะออกแบบตัวเลือกง่าย ๆ เพื่อเลือกโมเดลเป็นเครื่องมือได้อย่างไร?  
4. หมวดหมู่ CLI ใดที่เกี่ยวข้องมากที่สุดกับการพัฒนาในแต่ละวัน?  
5. คุณตรวจสอบความพร้อมใช้งานของ Foundry Local ก่อนเรียกใช้งานแอปได้อย่างไร?  

### แบบฝึกหัดปฏิบัติ  

1. ติดตั้ง/อัปเกรด Foundry Local และเรียกใช้ `phi-4-mini` ในเครื่อง (30 นาที)  
2. เรียก `/v1/models` และเรียกใช้การสนทนาแบบง่ายผ่าน REST (30 นาที)  
3. เปิดตัวอย่างแอป Chainlit และสนทนาในเครื่อง (30 นาที)  
4. เรียกใช้ตัวประสานงานตัวแทนหลายตัวและตรวจสอบผลลัพธ์ (30 นาที)  
5. ทดลองใช้ตัวเลือกโมเดลเป็นเครื่องมือพร้อมการแทนที่ตามสภาพแวดล้อม (30 นาที)

**หากคุณสามารถอุทิศเวลาเต็ม 20 ชั่วโมง:**
