## บทนำ

ยินดีต้อนรับสู่คู่มือการศึกษา EdgeAI สำหรับผู้เริ่มต้น! เอกสารนี้ถูกออกแบบมาเพื่อช่วยให้คุณเข้าใจเนื้อหาหลักสูตรได้อย่างมีประสิทธิภาพและเพิ่มประสบการณ์การเรียนรู้ของคุณ โดยมีเส้นทางการเรียนรู้ที่มีโครงสร้าง ตารางการศึกษา สรุปแนวคิดสำคัญ และทรัพยากรเสริมเพื่อเพิ่มความเข้าใจในเทคโนโลยี EdgeAI

นี่คือหลักสูตรที่กระชับ ใช้เวลา 20 ชั่วโมง ที่มอบความรู้สำคัญเกี่ยวกับ EdgeAI ในรูปแบบที่ประหยัดเวลา เหมาะสำหรับมืออาชีพและนักเรียนที่ต้องการพัฒนาทักษะในสาขานี้อย่างรวดเร็ว

## ภาพรวมหลักสูตร

หลักสูตรนี้แบ่งออกเป็นเจ็ดโมดูลที่ครอบคลุม:

1. **พื้นฐานและการเปลี่ยนแปลงของ EdgeAI** - ทำความเข้าใจแนวคิดหลักและการเปลี่ยนแปลงของเทคโนโลยี
2. **พื้นฐานของ Small Language Model (SLM)** - สำรวจครอบครัว SLM และสถาปัตยกรรมของพวกเขา
3. **การปรับใช้ Small Language Model** - การปรับใช้ในเชิงปฏิบัติ
4. **การแปลงรูปแบบโมเดลและการลดขนาด** - การปรับแต่งขั้นสูงด้วย 6 เฟรมเวิร์ก รวมถึง OpenVINO
5. **SLMOps - การดำเนินการของ Small Language Model** - การจัดการวงจรการผลิตและการปรับใช้
6. **ระบบ SLM Agentic** - AI Agents, การเรียกฟังก์ชัน และ Model Context Protocol
7. **ตัวอย่างการใช้งาน EdgeAI** - AI Toolkit, การพัฒนาบน Windows และการใช้งานเฉพาะแพลตฟอร์ม

## วิธีใช้คู่มือการศึกษา

- **การเรียนรู้แบบก้าวหน้า**: เรียนตามลำดับโมดูลเพื่อประสบการณ์การเรียนรู้ที่ต่อเนื่อง
- **จุดตรวจสอบความรู้**: ใช้คำถามประเมินตนเองหลังแต่ละส่วน
- **การฝึกปฏิบัติ**: ทำแบบฝึกหัดที่แนะนำเพื่อเสริมแนวคิดเชิงทฤษฎี
- **ทรัพยากรเสริม**: สำรวจเนื้อหาเพิ่มเติมสำหรับหัวข้อที่คุณสนใจมากที่สุด

## โมดูล 1: พื้นฐานและการเปลี่ยนแปลงของ EdgeAI

### วัตถุประสงค์การเรียนรู้สำคัญ

- เข้าใจความแตกต่างระหว่าง AI บนคลาวด์และ AI บน Edge
- เชี่ยวชาญเทคนิคการปรับแต่งหลักสำหรับสภาพแวดล้อมที่มีทรัพยากรจำกัด
- วิเคราะห์การใช้งานจริงของเทคโนโลยี EdgeAI
- ตั้งค่าสภาพแวดล้อมการพัฒนาสำหรับโปรเจกต์ EdgeAI

### จุดเน้นการศึกษา

#### ส่วนที่ 1: พื้นฐาน EdgeAI
- **แนวคิดสำคัญ**: 
  - การเปรียบเทียบระหว่าง Edge และ Cloud computing
  - เทคนิคการลดขนาดโมเดล
  - ตัวเลือกการเร่งความเร็วฮาร์ดแวร์ (NPUs, GPUs, CPUs)
  - ข้อดีด้านความเป็นส่วนตัวและความปลอดภัย

- **ทรัพยากรเสริม**:
  - [TensorFlow Lite Documentation](https://www.tensorflow.org/lite)
  - [ONNX Runtime GitHub](https://github.com/microsoft/onnxruntime)
  - [Edge Impulse Documentation](https://docs.edgeimpulse.com)

#### ส่วนที่ 2: กรณีศึกษาในโลกจริง
- **แนวคิดสำคัญ**: 
  - ระบบ Microsoft Phi & Mu
  - การใช้งานจริงในอุตสาหกรรมต่าง ๆ
  - ข้อควรพิจารณาในการปรับใช้

#### ส่วนที่ 3: คู่มือการใช้งานจริง
- **แนวคิดสำคัญ**: 
  - การตั้งค่าสภาพแวดล้อมการพัฒนา
  - เครื่องมือการลดขนาดและการปรับแต่ง
  - วิธีการประเมินการใช้งาน EdgeAI

#### ส่วนที่ 4: ฮาร์ดแวร์สำหรับการปรับใช้ Edge
- **แนวคิดสำคัญ**: 
  - การเปรียบเทียบแพลตฟอร์มฮาร์ดแวร์
  - กลยุทธ์การปรับแต่งสำหรับฮาร์ดแวร์เฉพาะ
  - ข้อควรพิจารณาในการปรับใช้

### คำถามประเมินตนเอง

1. เปรียบเทียบและอธิบายความแตกต่างระหว่าง AI บนคลาวด์และ AI บน Edge
2. อธิบายเทคนิคสำคัญสามข้อสำหรับการปรับแต่งโมเดลสำหรับการปรับใช้บน Edge
3. ข้อดีหลักของการรันโมเดล AI บน Edge คืออะไร?
4. อธิบายกระบวนการลดขนาดโมเดลและผลกระทบต่อประสิทธิภาพ
5. อธิบายว่าตัวเร่งความเร็วฮาร์ดแวร์ (NPUs, GPUs, CPUs) มีผลต่อการปรับใช้ EdgeAI อย่างไร

### แบบฝึกหัด

1. **การตั้งค่าสภาพแวดล้อมอย่างรวดเร็ว**: ตั้งค่าสภาพแวดล้อมการพัฒนาขั้นต่ำด้วยแพ็กเกจที่จำเป็น (30 นาที)
2. **การสำรวจโมเดล**: ดาวน์โหลดและตรวจสอบโมเดลภาษาขนาดเล็กที่ผ่านการฝึกฝนแล้ว (1 ชั่วโมง)
3. **การลดขนาดพื้นฐาน**: ลองลดขนาดโมเดลเล็ก ๆ อย่างง่าย (1 ชั่วโมง)
- การเลือกเฟรมเวิร์คด้วยต้นไม้ตัดสินใจ  
- การตรวจสอบความพร้อมสำหรับการใช้งานในระบบผลิต  
- กลยุทธ์เพื่อความยั่งยืนในอนาคต  

### คำถามประเมินตนเอง  

1. เปรียบเทียบกลยุทธ์การควอนไทซ์ในระดับความแม่นยำต่าง ๆ (1-bit ถึง 8-bit)  
2. อธิบายข้อดีของรูปแบบ GGUF สำหรับการใช้งานในอุปกรณ์ปลายทาง  
3. การปรับแต่งฮาร์ดแวร์ใน Microsoft Olive ช่วยเพิ่มประสิทธิภาพการใช้งานได้อย่างไร?  
4. ประโยชน์สำคัญของ OpenVINO's NNCF สำหรับการบีบอัดโมเดลคืออะไร?  
5. อธิบายว่า Apple MLX ใช้สถาปัตยกรรมหน่วยความจำรวมเพื่อการปรับแต่งอย่างไร  
6. การสร้างเวิร์กโฟลว์ช่วยในการเลือกเฟรมเวิร์คการปรับแต่งที่เหมาะสมได้อย่างไร?  

### แบบฝึกหัดปฏิบัติ  

1. **การควอนไทซ์โมเดล**: ใช้ระดับการควอนไทซ์ต่าง ๆ กับโมเดลและเปรียบเทียบผลลัพธ์ (1 ชั่วโมง)  
2. **การปรับแต่งด้วย OpenVINO**: ใช้ NNCF เพื่อบีบอัดโมเดลสำหรับฮาร์ดแวร์ Intel (1 ชั่วโมง)  
3. **การเปรียบเทียบเฟรมเวิร์ค**: ทดสอบโมเดลเดียวกันในสามเฟรมเวิร์คการปรับแต่งที่แตกต่างกัน (1 ชั่วโมง)  
4. **การวัดประสิทธิภาพ**: วัดผลกระทบของการปรับแต่งต่อความเร็วในการประมวลผลและการใช้หน่วยความจำ (1 ชั่วโมง)  

## โมดูล 5: SLMOps - การดำเนินงานโมเดลภาษาขนาดเล็ก  

### วัตถุประสงค์การเรียนรู้สำคัญ  

- เข้าใจหลักการจัดการวงจรชีวิตของ SLMOps  
- เชี่ยวชาญเทคนิคการกลั่นและการปรับแต่งสำหรับการใช้งานในอุปกรณ์ปลายทาง  
- ใช้กลยุทธ์การปรับใช้ในระบบผลิตพร้อมการตรวจสอบ  
- สร้างเวิร์กโฟลว์การดำเนินงานและการบำรุงรักษา SLM ระดับองค์กร  

### พื้นที่การศึกษา  

#### ส่วนที่ 1: บทนำสู่ SLMOps  
- **แนวคิดสำคัญ**:  
  - การเปลี่ยนแปลงกระบวนทัศน์ของ SLMOps ในการดำเนินงาน AI  
  - สถาปัตยกรรมที่คุ้มค่าและเน้นความเป็นส่วนตัว  
  - ผลกระทบเชิงกลยุทธ์ต่อธุรกิจและข้อได้เปรียบทางการแข่งขัน  

#### ส่วนที่ 2: การกลั่นโมเดล  
- **แนวคิดสำคัญ**:  
  - เทคนิคการถ่ายโอนความรู้  
  - การดำเนินการกระบวนการกลั่นสองขั้นตอน  
  - เวิร์กโฟลว์การกลั่นใน Azure ML  

#### ส่วนที่ 3: กลยุทธ์การปรับแต่ง  
- **แนวคิดสำคัญ**:  
  - การปรับแต่งที่มีประสิทธิภาพต่อพารามิเตอร์ (PEFT)  
  - วิธีการขั้นสูง LoRA และ QLoRA  
  - การฝึกอบรมแบบหลายอะแดปเตอร์และการปรับแต่งไฮเปอร์พารามิเตอร์  

#### ส่วนที่ 4: การปรับใช้ในระบบผลิต  
- **แนวคิดสำคัญ**:  
  - การแปลงและการควอนไทซ์โมเดลสำหรับการใช้งานในระบบผลิต  
  - การตั้งค่าการปรับใช้ Foundry Local  
  - การวัดประสิทธิภาพและการตรวจสอบคุณภาพ  

### คำถามประเมินตนเอง  

1. SLMOps แตกต่างจาก MLOps แบบดั้งเดิมอย่างไร?  
2. อธิบายประโยชน์ของการกลั่นโมเดลสำหรับการใช้งานในอุปกรณ์ปลายทาง  
3. ข้อควรพิจารณาสำคัญสำหรับการปรับแต่ง SLM ในสภาพแวดล้อมที่มีทรัพยากรจำกัดคืออะไร?  
4. อธิบายกระบวนการปรับใช้ในระบบผลิตสำหรับแอปพลิเคชัน AI ในอุปกรณ์ปลายทาง  

### แบบฝึกหัดปฏิบัติ  

1. **การกลั่นพื้นฐาน**: สร้างโมเดลขนาดเล็กจากโมเดลครูขนาดใหญ่ (1 ชั่วโมง)  
2. **การทดลองปรับแต่ง**: ปรับแต่งโมเดลสำหรับโดเมนเฉพาะ (1 ชั่วโมง)  
3. **เวิร์กโฟลว์การปรับใช้**: ตั้งค่า CI/CD พื้นฐานสำหรับการปรับใช้โมเดล (1 ชั่วโมง)  

## โมดูล 6: ระบบตัวแทน SLM - ตัวแทน AI และการเรียกฟังก์ชัน  

### วัตถุประสงค์การเรียนรู้สำคัญ  

- สร้างตัวแทน AI อัจฉริยะสำหรับสภาพแวดล้อมปลายทางโดยใช้โมเดลภาษาขนาดเล็ก  
- ใช้ความสามารถในการเรียกฟังก์ชันด้วยเวิร์กโฟลว์ที่เป็นระบบ  
- เชี่ยวชาญการรวม Model Context Protocol (MCP) สำหรับการโต้ตอบเครื่องมือมาตรฐาน  
- สร้างระบบตัวแทนที่ซับซ้อนด้วยการแทรกแซงของมนุษย์น้อยที่สุด  

### พื้นที่การศึกษา  

#### ส่วนที่ 1: ตัวแทน AI และพื้นฐาน SLM  
- **แนวคิดสำคัญ**:  
  - กรอบการจำแนกประเภทตัวแทน (ตัวแทนสะท้อน, ตัวแทนตามโมเดล, ตัวแทนตามเป้าหมาย, ตัวแทนเรียนรู้)  
  - การวิเคราะห์ข้อแลกเปลี่ยนระหว่าง SLM และ LLM  
  - รูปแบบการออกแบบตัวแทนเฉพาะสำหรับปลายทาง  
  - การเพิ่มประสิทธิภาพทรัพยากรสำหรับตัวแทน  

#### ส่วนที่ 2: การเรียกฟังก์ชันในโมเดลภาษาขนาดเล็ก  
- **แนวคิดสำคัญ**:  
  - การดำเนินการเวิร์กโฟลว์อย่างเป็นระบบ (การตรวจจับเจตนา, เอาต์พุต JSON, การดำเนินการภายนอก)  
  - การดำเนินการเฉพาะแพลตฟอร์ม (Phi-4-mini, โมเดล Qwen ที่เลือก, Microsoft Foundry Local)  
  - ตัวอย่างขั้นสูง (การทำงานร่วมกันของตัวแทนหลายตัว, การเลือกเครื่องมือแบบไดนามิก)  
  - ข้อควรพิจารณาสำหรับการใช้งานในระบบผลิต (การจำกัดอัตรา, การบันทึกการตรวจสอบ, มาตรการรักษาความปลอดภัย)  

#### ส่วนที่ 3: การรวม Model Context Protocol (MCP)  
- **แนวคิดสำคัญ**:  
  - สถาปัตยกรรมโปรโตคอลและการออกแบบระบบแบบเลเยอร์  
  - การสนับสนุนหลายแบ็กเอนด์ (Ollama สำหรับการพัฒนา, vLLM สำหรับการผลิต)  
  - โปรโตคอลการเชื่อมต่อ (โหมด STDIO และ SSE)  
  - การใช้งานจริง (การทำงานอัตโนมัติบนเว็บ, การประมวลผลข้อมูล, การรวม API)  

### คำถามประเมินตนเอง  

1. ข้อควรพิจารณาสำคัญสำหรับสถาปัตยกรรมตัวแทน AI ในอุปกรณ์ปลายทางคืออะไร?  
2. การเรียกฟังก์ชันช่วยเพิ่มความสามารถของตัวแทนได้อย่างไร?  
3. อธิบายบทบาทของ Model Context Protocol ในการสื่อสารของตัวแทน  

### แบบฝึกหัดปฏิบัติ  

1. **ตัวแทนง่าย ๆ**: สร้างตัวแทน AI พื้นฐานพร้อมการเรียกฟังก์ชัน (1 ชั่วโมง)  
2. **การรวม MCP**: ใช้ MCP ในแอปพลิเคชันตัวแทน (30 นาที)  

## โมดูล 7: ตัวอย่างการใช้งาน EdgeAI  

### วัตถุประสงค์การเรียนรู้สำคัญ  

- เชี่ยวชาญ AI Toolkit สำหรับ Visual Studio Code เพื่อเวิร์กโฟลว์การพัฒนา EdgeAI ที่ครอบคลุม  
- ได้รับความเชี่ยวชาญในแพลตฟอร์ม Windows AI Foundry และกลยุทธ์การเพิ่มประสิทธิภาพ NPU  
- ใช้ EdgeAI ในหลายแพลตฟอร์มฮาร์ดแวร์และสถานการณ์การปรับใช้  
- สร้างแอปพลิเคชัน EdgeAI ที่พร้อมใช้งานในระบบผลิตด้วยการปรับแต่งเฉพาะแพลตฟอร์ม  

### พื้นที่การศึกษา  

#### ส่วนที่ 1: AI Toolkit สำหรับ Visual Studio Code  
- **แนวคิดสำคัญ**:  
  - สภาพแวดล้อมการพัฒนา Edge AI ที่ครอบคลุมภายใน VS Code  
  - แคตตาล็อกโมเดลและการค้นพบสำหรับการใช้งานในอุปกรณ์ปลายทาง  
  - เวิร์กโฟลว์การทดสอบในเครื่อง, การปรับแต่ง, และการพัฒนาตัวแทน  
  - การตรวจสอบประสิทธิภาพและการประเมินสำหรับสถานการณ์ปลายทาง  

#### ส่วนที่ 2: คู่มือการพัฒนา Windows EdgeAI  
- **แนวคิดสำคัญ**:  
  - ภาพรวมแพลตฟอร์ม Windows AI Foundry  
  - Phi Silica API สำหรับการอนุมาน NPU ที่มีประสิทธิภาพ  
  - Computer Vision API สำหรับการประมวลผลภาพและ OCR  
  - Foundry Local CLI สำหรับการพัฒนาและทดสอบในเครื่อง  

#### ส่วนที่ 3: การใช้งานเฉพาะแพลตฟอร์ม  
- **แนวคิดสำคัญ**:  
  - การปรับใช้ NVIDIA Jetson Orin Nano (ประสิทธิภาพ AI 67 TOPS)  
  - แอปพลิเคชันมือถือด้วย .NET MAUI และ ONNX Runtime GenAI  
  - โซลูชัน Azure EdgeAI ด้วยสถาปัตยกรรมไฮบริดคลาวด์-ปลายทาง  
  - การเพิ่มประสิทธิภาพ Windows ML ด้วยการสนับสนุนฮาร์ดแวร์สากล  
  - แอปพลิเคชัน Foundry Local พร้อมการใช้งาน RAG ที่เน้นความเป็นส่วนตัว  

### คำถามประเมินตนเอง  

1. AI Toolkit ช่วยปรับปรุงเวิร์กโฟลว์การพัฒนา EdgeAI ได้อย่างไร?  
2. เปรียบเทียบกลยุทธ์การปรับใช้ในแพลตฟอร์มฮาร์ดแวร์ต่าง ๆ  
3. ข้อดีของ Windows AI Foundry สำหรับการพัฒนาปลายทางคืออะไร?  
4. อธิบายบทบาทของการเพิ่มประสิทธิภาพ NPU ในแอปพลิเคชัน Edge AI สมัยใหม่  
5. Phi Silica API ใช้ฮาร์ดแวร์ NPU เพื่อเพิ่มประสิทธิภาพได้อย่างไร?  
6. เปรียบเทียบข้อดีของการปรับใช้ในเครื่องกับคลาวด์สำหรับแอปพลิเคชันที่เน้นความเป็นส่วนตัว  

### แบบฝึกหัดปฏิบัติ  

1. **การตั้งค่า AI Toolkit**: ตั้งค่า AI Toolkit และปรับแต่งโมเดล (1 ชั่วโมง)  
2. **Windows AI Foundry**: สร้างแอปพลิเคชัน Windows AI ง่าย ๆ โดยใช้ Phi Silica API (1 ชั่วโมง)  
3. **การปรับใช้ข้ามแพลตฟอร์ม**: ปรับใช้โมเดลเดียวกันในสองแพลตฟอร์มที่แตกต่างกัน (1 ชั่วโมง)  
4. **การเพิ่มประสิทธิภาพ NPU**: ทดสอบประสิทธิภาพ NPU ด้วยเครื่องมือ Windows AI Foundry (30 นาที)

เอกสารนี้ได้รับการแปลโดยใช้บริการแปลภาษา AI [Co-op Translator](https://github.com/Azure/co-op-translator) แม้ว่าเราจะพยายามให้การแปลมีความถูกต้องมากที่สุด แต่โปรดทราบว่าการแปลอัตโนมัติอาจมีข้อผิดพลาดหรือความไม่ถูกต้อง เอกสารต้นฉบับในภาษาดั้งเดิมควรถือเป็นแหล่งข้อมูลที่เชื่อถือได้ สำหรับข้อมูลที่สำคัญ ขอแนะนำให้ใช้บริการแปลภาษามืออาชีพ เราไม่รับผิดชอบต่อความเข้าใจผิดหรือการตีความผิดที่เกิดจากการใช้การแปลนี้
