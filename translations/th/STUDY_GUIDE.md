## บทนำ

ยินดีต้อนรับสู่คู่มือการศึกษา EdgeAI สำหรับผู้เริ่มต้น! เอกสารนี้ถูกออกแบบมาเพื่อช่วยให้คุณเข้าใจเนื้อหาหลักสูตรได้อย่างมีประสิทธิภาพและเพิ่มประสบการณ์การเรียนรู้ของคุณ โดยมีการจัดโครงสร้างเส้นทางการเรียนรู้ ตารางการศึกษา สรุปแนวคิดสำคัญ และทรัพยากรเสริมเพื่อเพิ่มความเข้าใจในเทคโนโลยี EdgeAI

นี่คือหลักสูตรที่กระชับ ใช้เวลา 20 ชั่วโมง ซึ่งมอบความรู้สำคัญเกี่ยวกับ EdgeAI ในรูปแบบที่ประหยัดเวลา เหมาะสำหรับมืออาชีพและนักเรียนที่ต้องการพัฒนาทักษะในสาขานี้อย่างรวดเร็ว

## ภาพรวมหลักสูตร

หลักสูตรนี้แบ่งออกเป็น 8 โมดูลที่ครอบคลุม:

0. **แนะนำ EdgeAI** - สร้างพื้นฐานและบริบท พร้อมตัวอย่างการใช้งานในอุตสาหกรรมและเป้าหมายการเรียนรู้  
1. **พื้นฐาน EdgeAI และการเปลี่ยนแปลง** - เข้าใจแนวคิดหลักและการเปลี่ยนแปลงทางเทคโนโลยี  
2. **พื้นฐาน Small Language Model (SLM)** - สำรวจครอบครัว SLM และสถาปัตยกรรมของพวกเขา  
3. **การปรับใช้ Small Language Model** - กลยุทธ์การปรับใช้ในทางปฏิบัติ  
4. **การแปลงรูปแบบโมเดลและการลดขนาด** - การปรับแต่งขั้นสูงด้วย 6 เฟรมเวิร์ก รวมถึง OpenVINO  
5. **SLMOps - การจัดการวงจรชีวิตและการปรับใช้โมเดล**  
6. **ระบบ SLM Agentic** - AI Agents, การเรียกฟังก์ชัน และ Model Context Protocol  
7. **ตัวอย่างการใช้งาน EdgeAI** - AI Toolkit, การพัฒนาบน Windows และการใช้งานเฉพาะแพลตฟอร์ม  
8. **Microsoft Foundry Local – ชุดเครื่องมือสำหรับนักพัฒนา** - การพัฒนาที่เน้นการใช้งานในพื้นที่ พร้อมการผสาน Azure แบบไฮบริด (โมดูล 08)

## วิธีใช้คู่มือการศึกษา

- **การเรียนรู้แบบก้าวหน้า**: เรียนตามลำดับโมดูลเพื่อประสบการณ์การเรียนรู้ที่ต่อเนื่อง  
- **จุดตรวจสอบความรู้**: ใช้คำถามประเมินตนเองหลังแต่ละส่วน  
- **การฝึกปฏิบัติ**: ทำแบบฝึกหัดที่แนะนำเพื่อเสริมสร้างแนวคิดเชิงทฤษฎี  
- **ทรัพยากรเสริม**: สำรวจเนื้อหาเพิ่มเติมสำหรับหัวข้อที่คุณสนใจมากที่สุด

## โมดูล 0: แนะนำ EdgeAI

### วัตถุประสงค์การเรียนรู้สำคัญ

- เข้าใจว่า Edge AI คืออะไรและเหตุใดจึงสำคัญในโลกเทคโนโลยีปัจจุบัน  
- ระบุอุตสาหกรรมหลักที่ถูกเปลี่ยนแปลงโดย Edge AI และกรณีการใช้งานเฉพาะ  
- เข้าใจข้อดีของ Small Language Models (SLMs) สำหรับการปรับใช้ที่ขอบ  
- กำหนดความคาดหวังและผลลัพธ์การเรียนรู้ที่ชัดเจนสำหรับหลักสูตรทั้งหมด  
- รับรู้โอกาสในอาชีพและทักษะที่จำเป็นในสาขา Edge AI  

### จุดเน้นการศึกษา

#### ส่วนที่ 1: แนวคิดและคำจำกัดความของ Edge AI
- **แนวคิดสำคัญ**:  
  - Edge AI เทียบกับการประมวลผล AI แบบคลาวด์  
  - การบรรจบกันของฮาร์ดแวร์ การปรับแต่งโมเดล และความต้องการทางธุรกิจ  
  - การปรับใช้ AI แบบเรียลไทม์ รักษาความเป็นส่วนตัว และประหยัดต้นทุน  

#### ส่วนที่ 2: การใช้งานในอุตสาหกรรม
- **แนวคิดสำคัญ**:  
  - การผลิต & อุตสาหกรรม 4.0: การบำรุงรักษาเชิงคาดการณ์และการควบคุมคุณภาพ  
  - การดูแลสุขภาพ: การวินิจฉัยภาพและการติดตามผู้ป่วย  
  - ระบบอัตโนมัติ: ยานพาหนะขับเคลื่อนอัตโนมัติและการขนส่ง  
  - เมืองอัจฉริยะ: การจัดการจราจรและความปลอดภัยสาธารณะ  
  - เทคโนโลยีสำหรับผู้บริโภค: สมาร์ทโฟน อุปกรณ์สวมใส่ และบ้านอัจฉริยะ  

#### ส่วนที่ 3: พื้นฐาน Small Language Models
- **แนวคิดสำคัญ**:  
  - ลักษณะและการเปรียบเทียบประสิทธิภาพของ SLM  
  - การแลกเปลี่ยนระหว่างประสิทธิภาพของพารามิเตอร์และความสามารถ  
  - ข้อจำกัดการปรับใช้ที่ขอบและกลยุทธ์การปรับแต่ง  

#### ส่วนที่ 4: กรอบการเรียนรู้และเส้นทางอาชีพ
- **แนวคิดสำคัญ**:  
  - สถาปัตยกรรมหลักสูตรและแนวทางการเรียนรู้แบบก้าวหน้า  
  - ทักษะทางเทคนิคและเป้าหมายการใช้งานจริง  
  - โอกาสในการพัฒนาอาชีพและการใช้งานในอุตสาหกรรม  

### คำถามประเมินตนเอง

1. แนวโน้มทางเทคโนโลยีสามประการที่ทำให้ Edge AI เป็นไปได้คืออะไร?  
2. เปรียบเทียบข้อดีและความท้าทายของ Edge AI กับ AI แบบคลาวด์  
3. ระบุสามอุตสาหกรรมที่ Edge AI มีคุณค่าทางธุรกิจสำคัญและอธิบายเหตุผล  
4. Small Language Models ทำให้ Edge AI ใช้งานได้จริงในโลกแห่งความเป็นจริงอย่างไร?  
5. ทักษะทางเทคนิคสำคัญที่คุณจะพัฒนาตลอดหลักสูตรนี้คืออะไร?  
6. อธิบายแนวทางการเรียนรู้แบบสี่ขั้นตอนที่ใช้ในหลักสูตรนี้  

### แบบฝึกหัด

1. **การวิจัยอุตสาหกรรม**: เลือกหนึ่งการใช้งานในอุตสาหกรรมและวิจัยการใช้งาน Edge AI ในโลกจริง (30 นาที)  
2. **การสำรวจโมเดล**: เรียกดู Small Language Models ที่มีอยู่ใน Hugging Face และเปรียบเทียบจำนวนพารามิเตอร์และความสามารถ (30 นาที)  
3. **การวางแผนการเรียนรู้**: ทบทวนโครงสร้างหลักสูตรทั้งหมดและสร้างตารางการศึกษาเฉพาะตัวของคุณ (15 นาที)

#### ส่วนที่ 1: ตัวแทน AI และพื้นฐาน SLM  
- **แนวคิดสำคัญ**:  
  - กรอบการจำแนกตัวแทน (ตัวแทนสะท้อน, ตัวแทนตามโมเดล, ตัวแทนตามเป้าหมาย, ตัวแทนเรียนรู้)  
  - การวิเคราะห์ข้อแลกเปลี่ยนระหว่าง SLM และ LLM  
  - รูปแบบการออกแบบตัวแทนเฉพาะสำหรับ Edge  
  - การปรับแต่งทรัพยากรสำหรับตัวแทน  

#### ส่วนที่ 2: การเรียกฟังก์ชันในโมเดลภาษาเล็ก  
- **แนวคิดสำคัญ**:  
  - การดำเนินการเวิร์กโฟลว์อย่างเป็นระบบ (การตรวจจับเจตนา, JSON output, การดำเนินการภายนอก)  
  - การใช้งานเฉพาะแพลตฟอร์ม (Phi-4-mini, โมเดล Qwen ที่เลือก, Microsoft Foundry Local)  
  - ตัวอย่างขั้นสูง (การทำงานร่วมกันของตัวแทนหลายตัว, การเลือกเครื่องมือแบบไดนามิก)  
  - ข้อควรพิจารณาสำหรับการผลิต (การจำกัดอัตรา, การบันทึกการตรวจสอบ, มาตรการรักษาความปลอดภัย)  

#### ส่วนที่ 3: การผสานรวม Model Context Protocol (MCP)  
- **แนวคิดสำคัญ**:  
  - สถาปัตยกรรมโปรโตคอลและการออกแบบระบบแบบเลเยอร์  
  - การสนับสนุนหลายแบ็กเอนด์ (Ollama สำหรับการพัฒนา, vLLM สำหรับการผลิต)  
  - โปรโตคอลการเชื่อมต่อ (โหมด STDIO และ SSE)  
  - การใช้งานจริง (การทำงานอัตโนมัติบนเว็บ, การประมวลผลข้อมูล, การผสานรวม API)  

### คำถามประเมินตนเอง  

1. ข้อควรพิจารณาด้านสถาปัตยกรรมที่สำคัญสำหรับตัวแทน AI ใน Edge คืออะไร?  
2. การเรียกฟังก์ชันช่วยเพิ่มความสามารถของตัวแทนอย่างไร?  
3. อธิบายบทบาทของ Model Context Protocol ในการสื่อสารของตัวแทน  

### แบบฝึกหัดปฏิบัติ  

1. **ตัวแทนง่ายๆ**: สร้างตัวแทน AI พื้นฐานพร้อมการเรียกฟังก์ชัน (1 ชั่วโมง)  
2. **การผสานรวม MCP**: ใช้ MCP ในแอปพลิเคชันตัวแทน (30 นาที)  

## โมดูล 7: ตัวอย่างการใช้งาน EdgeAI  

### วัตถุประสงค์การเรียนรู้หลัก  

- เชี่ยวชาญ AI Toolkit สำหรับ Visual Studio Code สำหรับเวิร์กโฟลว์การพัฒนา EdgeAI ที่ครอบคลุม  
- มีความเชี่ยวชาญในแพลตฟอร์ม Windows AI Foundry และกลยุทธ์การปรับแต่ง NPU  
- ใช้ EdgeAI ในหลายแพลตฟอร์มฮาร์ดแวร์และสถานการณ์การปรับใช้  
- สร้างแอปพลิเคชัน EdgeAI ที่พร้อมใช้งานจริงด้วยการปรับแต่งเฉพาะแพลตฟอร์ม  

### พื้นที่การศึกษา  

#### ส่วนที่ 1: AI Toolkit สำหรับ Visual Studio Code  
- **แนวคิดสำคัญ**:  
  - สภาพแวดล้อมการพัฒนา Edge AI ที่ครอบคลุมภายใน VS Code  
  - แคตตาล็อกโมเดลและการค้นพบสำหรับการปรับใช้ใน Edge  
  - การทดสอบในท้องถิ่น, การปรับแต่ง, และเวิร์กโฟลว์การพัฒนาตัวแทน  
  - การตรวจสอบประสิทธิภาพและการประเมินสำหรับสถานการณ์ใน Edge  

#### ส่วนที่ 2: คู่มือการพัฒนา Windows EdgeAI  
- **แนวคิดสำคัญ**:  
  - ภาพรวมแพลตฟอร์ม Windows AI Foundry  
  - Phi Silica API สำหรับการประมวลผล NPU ที่มีประสิทธิภาพ  
  - Computer Vision APIs สำหรับการประมวลผลภาพและ OCR  
  - Foundry Local CLI สำหรับการพัฒนาและทดสอบในท้องถิ่น  

#### ส่วนที่ 3: การใช้งานเฉพาะแพลตฟอร์ม  
- **แนวคิดสำคัญ**:  
  - การปรับใช้ NVIDIA Jetson Orin Nano (ประสิทธิภาพ AI 67 TOPS)  
  - แอปพลิเคชันมือถือด้วย .NET MAUI และ ONNX Runtime GenAI  
  - โซลูชัน Azure EdgeAI ด้วยสถาปัตยกรรมไฮบริดคลาวด์-Edge  
  - การปรับแต่ง Windows ML ด้วยการสนับสนุนฮาร์ดแวร์แบบสากล  
  - แอปพลิเคชัน Foundry Local พร้อมการใช้งาน RAG ที่เน้นความเป็นส่วนตัว

1. AI Toolkit ช่วยปรับปรุงเวิร์กโฟลว์การพัฒนา EdgeAI อย่างไร?  
2. เปรียบเทียบกลยุทธ์การปรับใช้ในแพลตฟอร์มฮาร์ดแวร์ต่างๆ

5. Phi Silica API ใช้ฮาร์ดแวร์ NPU เพื่อเพิ่มประสิทธิภาพได้อย่างไร?

## ไอเดียโครงการขนาดเล็ก

ลองทำหนึ่งในโครงการเหล่านี้เพื่อฝึกฝนแนวคิด EdgeAI (แต่ละโครงการออกแบบให้ใช้เวลา 2-4 ชั่วโมง):

### โครงการสำหรับผู้เริ่มต้น (2-3 ชั่วโมงต่อโครงการ)
1. **Edge Text Assistant**: สร้างเครื่องมือเติมข้อความแบบออฟไลน์ง่ายๆ โดยใช้โมเดลภาษาขนาดเล็ก
2. **Model Comparison Dashboard**: สร้างการแสดงผลพื้นฐานของตัวชี้วัดประสิทธิภาพระหว่าง SLMs ต่างๆ
3. **Optimization Experiment**: วัดผลกระทบของระดับการบีบอัดต่างๆ บนโมเดลพื้นฐานเดียวกัน

### โครงการระดับกลาง (3-4 ชั่วโมงต่อโครงการ)
4. **AI Toolkit Workflow**: ใช้ VS Code AI Toolkit เพื่อปรับแต่งและใช้งานโมเดลตั้งแต่ต้นจนจบ
5. **Windows AI Foundry Application**: สร้างแอป Windows โดยใช้ Phi Silica API และการปรับแต่ง NPU
6. **Cross-Platform Deployment**: ใช้งานโมเดลที่ปรับแต่งแล้วบน Windows (OpenVINO) และมือถือ (.NET MAUI)
7. **Function Calling Agent**: สร้าง AI agent ที่มีความสามารถในการเรียกฟังก์ชันสำหรับสถานการณ์ Edge

### โครงการบูรณาการขั้นสูง (4-5 ชั่วโมงต่อโครงการ)
8. **OpenVINO Optimization Pipeline**: ใช้งานการปรับแต่งโมเดลอย่างสมบูรณ์โดยใช้ NNCF และ GenAI toolkit
9. **SLMOps Pipeline**: ใช้งานวงจรชีวิตโมเดลตั้งแต่การฝึกอบรมจนถึงการใช้งานใน Edge
10. **Multi-Model Edge System**: ใช้งานโมเดลเฉพาะทางหลายตัวที่ทำงานร่วมกันบนฮาร์ดแวร์ Edge
11. **MCP Integration System**: สร้างระบบ agentic โดยใช้ Model Context Protocol สำหรับการโต้ตอบกับเครื่องมือ

## สรุป

EdgeAI เป็นแนวหน้าของการใช้งานปัญญาประดิษฐ์ที่นำความสามารถอันทรงพลังมาสู่เครื่องมือโดยตรง พร้อมแก้ไขปัญหาสำคัญเกี่ยวกับความเป็นส่วนตัว ความล่าช้า และการเชื่อมต่อ หลักสูตร 20 ชั่วโมงนี้ให้ความรู้และทักษะปฏิบัติที่จำเป็นเพื่อเริ่มต้นใช้งานเทคโนโลยี EdgeAI ได้ทันที

หลักสูตรนี้ออกแบบมาให้กระชับและเน้นแนวคิดสำคัญที่สุด เพื่อให้คุณได้รับความเชี่ยวชาญที่มีคุณค่าโดยไม่ต้องใช้เวลามากเกินไป อย่าลืมว่าการฝึกฝนด้วยตัวอย่างง่ายๆ เป็นกุญแจสำคัญในการเสริมสร้างสิ่งที่คุณได้เรียนรู้

ขอให้สนุกกับการเรียนรู้!
