# Module 08: การใช้งาน Microsoft Foundry Local - เครื่องมือสำหรับนักพัฒนาครบวงจร

## ภาพรวม

Microsoft Foundry Local เป็นเทคโนโลยีรุ่นใหม่สำหรับการพัฒนา AI บน Edge ที่มอบเครื่องมือทรงพลังให้กับนักพัฒนาในการสร้าง, ปรับใช้, และขยายแอปพลิเคชัน AI ในพื้นที่ พร้อมการผสานรวมที่ไร้รอยต่อกับ Azure AI Foundry โมดูลนี้ครอบคลุมทุกด้านของ Foundry Local ตั้งแต่การติดตั้งไปจนถึงการพัฒนาเอเจนต์ขั้นสูง

**เทคโนโลยีสำคัญ:**
- Microsoft Foundry Local CLI และ SDK
- การผสานรวมกับ Azure AI Foundry
- การประมวลผลโมเดลบนอุปกรณ์
- การแคชและปรับแต่งโมเดลในพื้นที่
- สถาปัตยกรรมแบบเอเจนต์

## วัตถุประสงค์การเรียนรู้ของโมดูล

เมื่อจบโมดูลนี้ คุณจะสามารถ:

- **เชี่ยวชาญการตั้งค่า Foundry Local**: ติดตั้ง, ตั้งค่า, และปรับแต่ง Foundry Local สำหรับการพัฒนาบน Windows 11
- **ปรับใช้โมเดลหลากหลาย**: รันโมเดล phi, qwen, deepseek, และ GPT-OSS-20B ในพื้นที่ด้วยคำสั่ง CLI
- **สร้างโซลูชันระดับการผลิต**: สร้างแอปพลิเคชัน AI ด้วยการออกแบบ prompt ขั้นสูงและการผสานรวมข้อมูล
- **ใช้ประโยชน์จากระบบโอเพ่นซอร์ส**: ผสานรวมโมเดล Hugging Face และการเพิ่มจากชุมชน
- **เปรียบเทียบสถาปัตยกรรม AI**: เข้าใจข้อดีข้อเสียระหว่าง LLMs และ SLMs และกลยุทธ์การปรับใช้
- **พัฒนา AI Agents**: สร้างเอเจนต์อัจฉริยะโดยใช้สถาปัตยกรรมและความสามารถของ Foundry Local
- **ใช้โมเดลเป็นเครื่องมือ**: สร้างโซลูชัน AI แบบโมดูลที่ปรับแต่งได้สำหรับแอปพลิเคชันในองค์กร

## โครงสร้างเซสชัน

### [1: เริ่มต้นใช้งาน Foundry Local](./01.FoundryLocalSetup.md)
**หัวข้อ**: การติดตั้ง, การตั้งค่า CLI, การแคชโมเดล, และการเร่งความเร็วฮาร์ดแวร์

**สิ่งที่คุณจะได้เรียนรู้:**
- การติดตั้ง Foundry Local บน Windows 11 อย่างสมบูรณ์
- การตั้งค่า CLI และโครงสร้างคำสั่ง
- กลยุทธ์การแคชโมเดลเพื่อประสิทธิภาพสูงสุด
- การตั้งค่าและปรับแต่งการเร่งความเร็วฮาร์ดแวร์
- การปรับใช้โมเดล phi, qwen, deepseek, และ GPT-OSS-20B แบบลงมือทำ

**ระยะเวลา**: 2-3 ชั่วโมง  
**ข้อกำหนดเบื้องต้น**: Windows 11, ความรู้พื้นฐานเกี่ยวกับ command line

### [2: สร้างโซลูชัน AI ด้วย Azure AI Foundry](./02.AzureAIFoundryIntegration.md)
**หัวข้อ**: การออกแบบ prompt ขั้นสูง, การผสานรวมข้อมูล, และงานที่นำไปใช้ได้จริง

**สิ่งที่คุณจะได้เรียนรู้:**
- เทคนิคการออกแบบ prompt ขั้นสูง
- รูปแบบการผสานรวมข้อมูลและแนวทางปฏิบัติที่ดีที่สุด

- การทำงานแบบผสานรวมกับ Azure AI Foundry อย่างไร้รอยต่อ
- การปรับแต่งประสิทธิภาพและการตรวจสอบ

**ระยะเวลา**: 2-3 ชั่วโมง  
**ข้อกำหนดเบื้องต้น**: การจบเซสชัน 1, บัญชี Azure (ไม่บังคับ)

### [3: โมเดลโอเพ่นซอร์สใน Foundry Local](./03.OpenSourceModels.md)
**หัวข้อ**: การผสานรวม Hugging Face, กลยุทธ์การเลือกโมเดล, และการเพิ่มจากชุมชน

- การปรับใช้และปรับแต่งโมเดลแบบกำหนดเอง
- เกณฑ์การประเมินและการเลือกโมเดลจากชุมชน

**ระยะเวลา**: 2-3 ชั่วโมง  
**ข้อกำหนดเบื้องต้น**: การจบเซสชัน 1-2, บัญชี Hugging Face

### [4: สำรวจโมเดลล้ำสมัย - LLMs, SLMs, และการประมวลผลบนอุปกรณ์](./04.CuttingEdgeModels.md)
**หัวข้อ**: การเปรียบเทียบโมเดล, EdgeAI ด้วย Phi และ ONNX Runtime, และเดโมขั้นสูง

**สิ่งที่คุณจะได้เรียนรู้:**

- ข้อดีข้อเสียระหว่างการประมวลผลในพื้นที่และบนคลาวด์
- การใช้งาน EdgeAI ด้วย Phi และ ONNX Runtime
- การพัฒนาและปรับใช้แอป Chainlit RAG Chat
- เทคนิคการปรับแต่งการประมวลผลด้วย WebGPU
- การผสานรวมและความสามารถของ AI PC SDK

**ระยะเวลา**: 3-4 ชั่วโมง  
**ข้อกำหนดเบื้องต้น**: การจบเซสชัน 1-3, ความเข้าใจเกี่ยวกับแนวคิดการประมวลผล

### [5: สร้างเอเจนต์ที่ขับเคลื่อนด้วย AI อย่างรวดเร็วด้วย Foundry Local](./05.AIPoweredAgents.md)
**หัวข้อ**: การพัฒนาแอป GenAI อย่างรวดเร็ว, system prompts, grounding, และสถาปัตยกรรมเอเจนต์

**สิ่งที่คุณจะได้เรียนรู้:**
- สถาปัตยกรรมและรูปแบบการออกแบบเอเจนต์ใน Foundry Local
- การออกแบบ system prompt เพื่อกำหนดพฤติกรรมของเอเจนต์
- เทคนิค grounding เพื่อให้เอเจนต์ตอบสนองได้อย่างน่าเชื่อถือ
- เวิร์กโฟลว์การพัฒนาแอป GenAI อย่างรวดเร็ว
- การจัดการเอเจนต์และระบบหลายเอเจนต์
- กลยุทธ์การปรับใช้เอเจนต์ในระดับการผลิต

**ระยะเวลา**: 3-4 ชั่วโมง  
**ข้อกำหนดเบื้องต้น**: การจบเซสชัน 1-4, ความเข้าใจพื้นฐานเกี่ยวกับ AI agents

### [6: Foundry Local - โมเดลในฐานะเครื่องมือ](./06.ModelsAsTools.md)
**หัวข้อ**: โซลูชัน AI แบบโมดูล, การปรับใช้บนอุปกรณ์, และการขยายในองค์กร

**สิ่งที่คุณจะได้เรียนรู้:**
- การใช้โมเดล AI เป็นเครื่องมือแบบโมดูลที่ปรับแต่งได้
- การปรับใช้บนอุปกรณ์โดยไม่ต้องพึ่งพาคลาวด์
- การประมวลผลที่มีความหน่วงต่ำ, ประหยัดค่าใช้จ่าย, และรักษาความเป็นส่วนตัว
- รูปแบบการผสานรวม SDK, API, และ CLI
- การปรับแต่งโมเดลสำหรับกรณีการใช้งานเฉพาะ
- กลยุทธ์การขยายจากพื้นที่ไปยัง Azure AI Foundry
- สถาปัตยกรรมแอปพลิเคชัน AI ที่พร้อมใช้งานในองค์กร

**ระยะเวลา**: 3-4 ชั่วโมง  
**ข้อกำหนดเบื้องต้น**: การจบทุกเซสชันก่อนหน้า, ประสบการณ์การพัฒนาในองค์กรจะเป็นประโยชน์

## ข้อกำหนดเบื้องต้น

### ความต้องการของระบบ
- **ระบบปฏิบัติการ**: Windows 11 (22H2 หรือใหม่กว่า)
- **หน่วยความจำ**: RAM 16GB (แนะนำ 32GB สำหรับโมเดลขนาดใหญ่)
- **พื้นที่จัดเก็บ**: พื้นที่ว่าง 50GB สำหรับการแคชโมเดล
- **ฮาร์ดแวร์**: อุปกรณ์ที่รองรับ NPU (Copilot+ PC), GPU ไม่บังคับ
- **เครือข่าย**: อินเทอร์เน็ตความเร็วสูงสำหรับการดาวน์โหลดโมเดลครั้งแรก

### สภาพแวดล้อมการพัฒนา
- Visual Studio Code พร้อมส่วนขยาย AI Toolkit
- Python 3.10+ และ pip
- Git สำหรับการควบคุมเวอร์ชัน
- PowerShell หรือ Command Prompt
- Azure CLI (ไม่บังคับสำหรับการผสานรวมคลาวด์)

### ความรู้เบื้องต้น
- ความเข้าใจพื้นฐานเกี่ยวกับแนวคิด AI/ML
- ความคุ้นเคยกับ command line
- พื้นฐานการเขียนโปรแกรม Python
- แนวคิด REST API
- ความรู้พื้นฐานเกี่ยวกับการออกแบบ prompt และการประมวลผลโมเดล

## ผลลัพธ์การเรียนรู้

เมื่อจบโมดูลนี้ คุณจะมีความสามารถใน:

### ความเชี่ยวชาญทางเทคนิค
- **การปรับใช้และจัดการ**: การติดตั้ง Foundry Local ในสภาพแวดล้อมการพัฒนาและการผลิต
- **การผสานรวมโมเดล**: การทำงานกับโมเดลหลากหลายจาก Microsoft, Hugging Face, และแหล่งชุมชน
- **การสร้างแอปพลิเคชัน**: การสร้างแอปพลิเคชัน AI ที่พร้อมใช้งานในระดับการผลิตด้วยฟีเจอร์ขั้นสูงและการปรับแต่ง
- **การพัฒนาเอเจนต์**: การสร้างเอเจนต์ AI ที่ซับซ้อนด้วย grounding, reasoning, และการผสานรวมเครื่องมือ

### ความเข้าใจเชิงกลยุทธ์
- **การตัดสินใจด้านสถาปัตยกรรม**: การเลือกใช้งานระหว่างการปรับใช้ในพื้นที่และบนคลาวด์
- **การปรับแต่งประสิทธิภาพ**: การปรับแต่งประสิทธิภาพการประมวลผลในฮาร์ดแวร์ที่แตกต่างกัน
- **การขยายในองค์กร**: การออกแบบแอปพลิเคชันที่ขยายจากต้นแบบในพื้นที่ไปสู่การใช้งานในองค์กร
- **ความเป็นส่วนตัวและความปลอดภัย**: การสร้างโซลูชัน AI ที่รักษาความเป็นส่วนตัวด้วยการประมวลผลในพื้นที่

### ความสามารถด้านนวัตกรรม
- **การสร้างต้นแบบอย่างรวดเร็ว**: การสร้างและทดสอบแนวคิดแอปพลิเคชัน AI อย่างรวดเร็ว
- **การผสานรวมชุมชน**: การใช้โมเดลโอเพ่นซอร์สและการมีส่วนร่วมในระบบนิเวศ
- **รูปแบบขั้นสูง**: การใช้งานรูปแบบ AI ล้ำสมัย เช่น RAG, เอเจนต์, และการผสานรวมเครื่องมือ
- **การพัฒนาที่พร้อมสำหรับอนาคต**: การสร้างแอปพลิเคชันที่พร้อมสำหรับเทคโนโลยีและรูปแบบ AI ที่เกิดขึ้นใหม่

5. **ฝึกฝนอย่างต่อเนื่อง**: ใช้แนวคิดผ่านการลงมือทำและโปรเจกต์

### การตั้งค่าตัวจัดการ (เซสชัน 6)
ตัวจัดการจะทำการตรวจสอบสุขภาพอย่างรวดเร็วและรองรับการตั้งค่าตามสภาพแวดล้อม:
```cmd
set BASE_URL=http://localhost:8000
set GENERAL_MODEL=phi-4-mini
set REASONING_MODEL=deepseek-r1-distill-qwen-7b
set CODE_MODEL=qwen2.5-7b-instruct
REM Or provide a full JSON registry
set TOOL_REGISTRY={"general":{"model":"phi-4-mini"}}
python samples\06\router.py "Pick the best model for code refactoring"
```

โมดูลนี้เป็นตัวแทนของการพัฒนา AI บน Edge ที่ล้ำสมัย โดยผสมผสานเครื่องมือระดับองค์กรของ Microsoft เข้ากับความยืดหยุ่นและนวัตกรรมของระบบโอเพ่นซอร์ส ด้วยการเชี่ยวชาญ Foundry Local คุณจะอยู่ในตำแหน่งแนวหน้าของการพัฒนาแอปพลิเคชัน AI

สำหรับ Azure OpenAI (เซสชัน 2) ดู README ตัวอย่างสำหรับตัวแปรสภาพแวดล้อมที่จำเป็นและการตั้งค่าเวอร์ชัน API
