### เพิ่ม - อัปเดตเวิร์กช็อปแบบครอบคลุม
- **เขียนใหม่ README.md ของเวิร์กช็อป**:
  - เพิ่มบทนำที่ครอบคลุม อธิบายคุณค่าของ Edge AI (ความเป็นส่วนตัว, ประสิทธิภาพ, ต้นทุน)
  - สร้างวัตถุประสงค์การเรียนรู้หลัก 6 ข้อ พร้อมความสามารถที่ละเอียด

  - เพิ่มคู่มือเริ่มต้นอย่างรวดเร็ว พร้อมข้อกำหนดเบื้องต้นและการตั้งค่า 3 ขั้นตอน
  - สร้างตารางทรัพยากรสำหรับตัวอย่าง Python (8 ไฟล์ พร้อมเวลาการรัน)
  - เพิ่มตาราง Jupyter notebooks (8 notebooks พร้อมระดับความยาก)

  - สร้าง `scripts/test_samples.py` - ตัวรันการทดสอบเบื้องต้นสำหรับตัวอย่าง Python ทั้งหมด

### เปลี่ยน - การปรับปรุงตัวอย่าง Python ในเวิร์กช็อป
- **อัปเดตตัวอย่าง Python ทั้ง 8 ตัวอย่างด้วยแนวปฏิบัติที่ดีที่สุด**:
  - ปรับปรุงการจัดการข้อผิดพลาดด้วยบล็อก try-except รอบการดำเนินการ I/O ทั้งหมด
  - เพิ่ม type hints และ docstrings ที่ครอบคลุม
  - ใช้รูปแบบการล็อก [INFO]/[ERROR]/[RESULT] อย่างสม่ำเสมอ
  - ป้องกัน imports ที่เป็นตัวเลือก พร้อมคำแนะนำการติดตั้ง
  - ปรับปรุงการตอบกลับผู้ใช้ในตัวอย่างทั้งหมด

- **session01/chat_bootstrap.py**:
  - ปรับปรุงการเริ่มต้น client พร้อมข้อความข้อผิดพลาดที่ครอบคลุม
  - ปรับปรุงการจัดการข้อผิดพลาดการสตรีมด้วยการตรวจสอบ chunk
  - เพิ่มการจัดการข้อยกเว้นที่ดีขึ้นสำหรับการไม่พร้อมใช้งานของบริการ

- **session02/rag_pipeline.py**:
  - เพิ่มการป้องกัน imports สำหรับ sentence-transformers พร้อมคำแนะนำการติดตั้ง
  - ปรับปรุงการจัดการข้อผิดพลาดสำหรับการฝังและการสร้างผลลัพธ์
  - ปรับปรุงการจัดรูปแบบผลลัพธ์ด้วยผลลัพธ์ที่มีโครงสร้าง

- **session02/rag_eval_ragas.py**:
  - ป้องกัน imports ที่เป็นตัวเลือก (ragas, datasets) พร้อมข้อความข้อผิดพลาดที่เป็นมิตรต่อผู้ใช้
  - เพิ่มการจัดการข้อผิดพลาดสำหรับเมตริกการประเมินผล
  - ปรับปรุงการจัดรูปแบบผลลัพธ์สำหรับผลการประเมิน

- **session03/benchmark_oss_models.py**:
  - ใช้การลดระดับอย่างสง่างาม (ดำเนินการต่อเมื่อเกิดข้อผิดพลาดของโมเดล)
  - เพิ่มการรายงานความคืบหน้าโดยละเอียดและการจัดการข้อผิดพลาดต่อโมเดล
  - ปรับปรุงการคำนวณสถิติด้วยการกู้คืนข้อผิดพลาดที่ครอบคลุม

- **session04/model_compare.py**:
  - เพิ่ม type hints (ประเภท Tuple ที่คืนค่า)
  - ปรับปรุงการจัดรูปแบบผลลัพธ์ด้วย JSON ที่มีโครงสร้าง
  - ใช้การจัดการข้อผิดพลาดต่อโมเดลพร้อมการกู้คืน

- **session05/agents_orchestrator.py**:
  - ปรับปรุง Agent.act() พร้อม docstrings ที่ครอบคลุม
  - เพิ่มการจัดการข้อผิดพลาดใน pipeline พร้อมการล็อกแบบขั้นตอนต่อขั้นตอน
  - ปรับปรุงการจัดการหน่วยความจำและการติดตามสถานะ

- **session06/models_router.py**:
  - ปรับปรุงเอกสารประกอบฟังก์ชันสำหรับส่วนประกอบการกำหนดเส้นทางทั้งหมด
  - เพิ่มการล็อกโดยละเอียดในฟังก์ชัน route()
  - ปรับปรุงผลการทดสอบด้วยผลลัพธ์ที่มีโครงสร้าง

- **session06/models_pipeline.py**:
  - เพิ่มการจัดการข้อผิดพลาดในฟังก์ชันช่วยเหลือ chat()
  - ปรับปรุง pipeline() พร้อมการล็อกขั้นตอนและการรายงานความคืบหน้า
  - ปรับปรุง main() พร้อมการกู้คืนข้อผิดพลาดที่ครอบคลุม

### เอกสาร - การปรับปรุงเอกสารเวิร์กช็อป
