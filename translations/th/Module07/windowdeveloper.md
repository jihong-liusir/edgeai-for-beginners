# คู่มือการพัฒนา Windows Edge AI

## บทนำ

ยินดีต้อนรับสู่การพัฒนา Windows Edge AI - คู่มือที่ครอบคลุมสำหรับการสร้างแอปพลิเคชันอัจฉริยะที่ใช้พลังของ AI บนอุปกรณ์ผ่านแพลตฟอร์ม Windows AI Foundry ของ Microsoft คู่มือนี้ออกแบบมาโดยเฉพาะสำหรับนักพัฒนา Windows ที่ต้องการผสานความสามารถ Edge AI ล้ำสมัยเข้ากับแอปพลิเคชันของตน พร้อมใช้ประโยชน์จากการเร่งความเร็วฮาร์ดแวร์ของ Windows อย่างเต็มที่

### ข้อได้เปรียบของ Windows AI

Windows AI Foundry เป็นแพลตฟอร์มที่รวมเป็นหนึ่งเดียว เชื่อถือได้ และปลอดภัย ซึ่งสนับสนุนวงจรชีวิตของนักพัฒนา AI อย่างครบถ้วน ตั้งแต่การเลือกและปรับแต่งโมเดลไปจนถึงการปรับแต่งและการใช้งานบน CPU, GPU, NPU และสถาปัตยกรรมคลาวด์แบบไฮบริด แพลตฟอร์มนี้ช่วยให้การพัฒนา AI เป็นเรื่องง่ายขึ้นโดยมอบ:

- **การแยกฮาร์ดแวร์**: การใช้งานที่ราบรื่นบนชิป AMD, Intel, NVIDIA และ Qualcomm
- **ความฉลาดบนอุปกรณ์**: AI ที่รักษาความเป็นส่วนตัวและทำงานบนฮาร์ดแวร์ในพื้นที่
- **ประสิทธิภาพที่ปรับแต่ง**: โมเดลที่ปรับแต่งล่วงหน้าสำหรับการกำหนดค่าฮาร์ดแวร์ Windows
- **พร้อมสำหรับองค์กร**: คุณสมบัติด้านความปลอดภัยและการปฏิบัติตามข้อกำหนดในระดับการผลิต

### ทำไมต้อง Windows สำหรับ Edge AI?

**การสนับสนุนฮาร์ดแวร์แบบสากล**  
Windows ML มอบการปรับแต่งฮาร์ดแวร์อัตโนมัติทั่วทั้งระบบ Windows เพื่อให้แน่ใจว่าแอปพลิเคชัน AI ของคุณทำงานได้อย่างเหมาะสมที่สุดโดยไม่คำนึงถึงสถาปัตยกรรมชิปพื้นฐาน

**AI Runtime ที่ผสานรวม**  
เครื่องมือการอนุมาน Windows ML ที่ติดตั้งมาพร้อมช่วยลดความยุ่งยากในการตั้งค่า ทำให้นักพัฒนาสามารถมุ่งเน้นไปที่ตรรกะของแอปพลิเคชันแทนที่จะกังวลเรื่องโครงสร้างพื้นฐาน

**Copilot+ PC Optimization**  
API ที่ออกแบบมาโดยเฉพาะสำหรับอุปกรณ์ Windows รุ่นใหม่ที่มี Neural Processing Units (NPUs) มอบประสิทธิภาพที่ยอดเยี่ยมต่อการใช้พลังงาน

**ระบบนิเวศของนักพัฒนา**  
เครื่องมือที่หลากหลาย เช่น การผสานรวมกับ Visual Studio เอกสารประกอบที่ครอบคลุม และแอปพลิเคชันตัวอย่างที่ช่วยเร่งวงจรการพัฒนา

## วัตถุประสงค์การเรียนรู้

เมื่อคุณทำคู่มือการพัฒนา Windows Edge AI นี้เสร็จสมบูรณ์ คุณจะมีทักษะสำคัญในการสร้างแอปพลิเคชัน AI ที่พร้อมใช้งานจริงบนแพลตฟอร์ม Windows

### ความสามารถทางเทคนิคหลัก

**ความเชี่ยวชาญใน Windows AI Foundry**  
- เข้าใจสถาปัตยกรรมและองค์ประกอบของแพลตฟอร์ม Windows AI Foundry  
- สำรวจวงจรชีวิตการพัฒนา AI อย่างครบถ้วนในระบบ Windows  
- ใช้แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยสำหรับแอปพลิเคชัน AI บนอุปกรณ์  
- ปรับแต่งแอปพลิเคชันให้เหมาะสมกับการกำหนดค่าฮาร์ดแวร์ Windows ต่างๆ  

**ความเชี่ยวชาญในการผสานรวม API**  
- เชี่ยวชาญ Windows AI APIs สำหรับแอปพลิเคชันด้านข้อความ ภาพ และมัลติโหมด  
- ใช้ Phi Silica language model สำหรับการสร้างข้อความและการให้เหตุผล  
- ใช้ความสามารถด้านการมองเห็นด้วยคอมพิวเตอร์ผ่าน API การประมวลผลภาพในตัว  
- ปรับแต่งโมเดลที่ผ่านการฝึกอบรมล่วงหน้าด้วยเทคนิค LoRA (Low-Rank Adaptation)  

**การใช้งาน Foundry Local**  
- ค้นหา ประเมิน และใช้งานโมเดลภาษาโอเพ่นซอร์สผ่าน Foundry Local CLI  
- เข้าใจการปรับแต่งและการลดขนาดโมเดลสำหรับการใช้งานในพื้นที่  
- ใช้ความสามารถ AI แบบออฟไลน์ที่ทำงานได้โดยไม่ต้องเชื่อมต่ออินเทอร์เน็ต  
- จัดการวงจรชีวิตและการอัปเดตโมเดลในสภาพแวดล้อมการผลิต  

**การใช้งาน Windows ML**  
- นำโมเดล ONNX ที่กำหนดเองมาใช้ในแอปพลิเคชัน Windows ผ่าน Windows ML  
- ใช้การเร่งความเร็วฮาร์ดแวร์อัตโนมัติบน CPU, GPU และ NPU  
- ใช้การอนุมานแบบเรียลไทม์ด้วยการใช้ทรัพยากรอย่างเหมาะสม  
- ออกแบบแอปพลิเคชัน AI ที่ปรับขนาดได้สำหรับอุปกรณ์ Windows ที่หลากหลาย  

### ทักษะการพัฒนาแอปพลิเคชัน

**การพัฒนา Windows ข้ามแพลตฟอร์ม**  
- สร้างแอปพลิเคชันที่ขับเคลื่อนด้วย AI โดยใช้ .NET MAUI สำหรับการใช้งาน Windows แบบสากล  
- ผสานรวมความสามารถ AI เข้ากับ Win32, UWP และ Progressive Web Applications  
- ใช้การออกแบบ UI ที่ตอบสนองซึ่งปรับให้เข้ากับสถานะการประมวลผล AI  
- จัดการการดำเนินการ AI แบบอะซิงโครนัสด้วยรูปแบบประสบการณ์ผู้ใช้ที่เหมาะสม  

**การปรับแต่งประสิทธิภาพ**  
- วิเคราะห์และปรับแต่งประสิทธิภาพการอนุมาน AI บนการกำหนดค่าฮาร์ดแวร์ต่างๆ  
- ใช้การจัดการหน่วยความจำอย่างมีประสิทธิภาพสำหรับโมเดลภาษาขนาดใหญ่  
- ออกแบบแอปพลิเคชันที่ลดระดับการทำงานอย่างเหมาะสมตามความสามารถของฮาร์ดแวร์  
- ใช้กลยุทธ์การแคชสำหรับการดำเนินการ AI ที่ใช้บ่อย  

**ความพร้อมใช้งานในระดับการผลิต**  
- ใช้การจัดการข้อผิดพลาดและกลไกสำรองที่ครอบคลุม  
- ออกแบบการวัดผลและการตรวจสอบประสิทธิภาพแอปพลิเคชัน AI  
- ใช้แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยสำหรับการจัดเก็บและการดำเนินการโมเดล AI ในพื้นที่  
- วางแผนกลยุทธ์การใช้งานสำหรับแอปพลิเคชันองค์กรและผู้บริโภค  

### ความเข้าใจด้านธุรกิจและกลยุทธ์

**สถาปัตยกรรมแอปพลิเคชัน AI**  
- ออกแบบสถาปัตยกรรมแบบไฮบริดที่ปรับให้เหมาะสมระหว่างการประมวลผล AI ในพื้นที่และคลาวด์  
- ประเมินข้อแลกเปลี่ยนระหว่างขนาดโมเดล ความแม่นยำ และความเร็วในการอนุมาน  
- วางแผนสถาปัตยกรรมการไหลของข้อมูลที่รักษาความเป็นส่วนตัวในขณะที่เพิ่มความฉลาด  
- ใช้โซลูชัน AI ที่คุ้มค่าซึ่งปรับขนาดได้ตามความต้องการของผู้ใช้  

**การวางตำแหน่งในตลาด**  
- เข้าใจข้อได้เปรียบเชิงแข่งขันของแอปพลิเคชัน AI ที่เป็น Windows-native  
- ระบุกรณีการใช้งานที่ AI บนอุปกรณ์มอบประสบการณ์ผู้ใช้ที่เหนือกว่า  
- พัฒนากลยุทธ์การเข้าสู่ตลาดสำหรับแอปพลิเคชัน Windows ที่เพิ่มประสิทธิภาพด้วย AI  
- วางตำแหน่งแอปพลิเคชันเพื่อใช้ประโยชน์จากข้อดีของระบบนิเวศ Windows  

## องค์ประกอบของแพลตฟอร์ม Windows AI Foundry

### 1. Windows AI APIs

Windows AI APIs มอบความสามารถ AI ที่พร้อมใช้งานซึ่งขับเคลื่อนโดยโมเดลในพื้นที่ที่ปรับให้เหมาะสมสำหรับประสิทธิภาพบนอุปกรณ์ Copilot+ PC โดยมีการตั้งค่าที่จำเป็นน้อยที่สุด

#### หมวดหมู่ API หลัก

**Phi Silica Language Model**  
- โมเดลภาษาขนาดเล็กแต่ทรงพลังสำหรับการสร้างข้อความและการให้เหตุผล  
- ปรับให้เหมาะสมสำหรับการอนุมานแบบเรียลไทม์ด้วยการใช้พลังงานต่ำ  
- รองรับการปรับแต่งเฉพาะด้วยเทคนิค LoRA  
- ผสานรวมกับการค้นหาเชิงความหมายและการดึงข้อมูลความรู้ของ Windows  

**Computer Vision APIs**  
- **Text Recognition (OCR)**: ดึงข้อความจากภาพด้วยความแม่นยำสูง  
- **Image Super Resolution**: เพิ่มความละเอียดของภาพด้วยโมเดล AI ในพื้นที่  
- **Image Segmentation**: ระบุและแยกวัตถุเฉพาะในภาพ  
- **Image Description**: สร้างคำอธิบายข้อความที่ละเอียดสำหรับเนื้อหาภาพ  
- **Object Erase**: ลบวัตถุที่ไม่ต้องการออกจากภาพด้วยการเติมภาพที่ขับเคลื่อนด้วย AI  

**Multimodal Capabilities**  
- **Vision-Language Integration**: ผสานความเข้าใจข้อความและภาพ  
- **Semantic Search**: เปิดใช้งานการค้นหาภาษาธรรมชาติในเนื้อหามัลติมีเดีย  
- **Knowledge Retrieval**: สร้างประสบการณ์การค้นหาอัจฉริยะด้วยข้อมูลในพื้นที่  

### 2. Foundry Local

Foundry Local มอบการเข้าถึงโมเดลภาษาโอเพ่นซอร์สที่พร้อมใช้งานบน Windows Silicon ให้กับนักพัฒนา พร้อมความสามารถในการเรียกดู ทดสอบ โต้ตอบ และใช้งานโมเดลในแอปพลิเคชันในพื้นที่

#### คุณสมบัติสำคัญ

**Model Catalog**  
- คอลเลกชันที่ครอบคลุมของโมเดลโอเพ่นซอร์สที่ปรับแต่งล่วงหน้า  
- โมเดลที่ปรับแต่งสำหรับ CPU, GPU และ NPU เพื่อการใช้งานทันที  
- รองรับตระกูลโมเดลยอดนิยม เช่น Llama, Mistral, Phi และโมเดลเฉพาะด้าน  

**CLI Integration**  
- อินเทอร์เฟซบรรทัดคำสั่งสำหรับการจัดการและใช้งานโมเดล  
- เวิร์กโฟลว์การปรับแต่งและการลดขนาดโมเดลอัตโนมัติ  
- การผสานรวมกับสภาพแวดล้อมการพัฒนายอดนิยมและ CI/CD  

**Local Deployment**  
- การดำเนินการแบบออฟไลน์อย่างสมบูรณ์โดยไม่มีการพึ่งพาคลาวด์  
- รองรับรูปแบบและการกำหนดค่าโมเดลที่กำหนดเอง  
- การให้บริการโมเดลที่มีประสิทธิภาพพร้อมการปรับแต่งฮาร์ดแวร์อัตโนมัติ  

### 3. Windows ML

Windows ML เป็นแพลตฟอร์ม AI หลักและ runtime การอนุมานที่ผสานรวมบน Windows ซึ่งช่วยให้นักพัฒนาสามารถใช้งานโมเดลที่กำหนดเองได้อย่างมีประสิทธิภาพทั่วทั้งระบบฮาร์ดแวร์ Windows

#### ประโยชน์ของสถาปัตยกรรม

**การสนับสนุนฮาร์ดแวร์แบบสากล**  
- การปรับแต่งอัตโนมัติสำหรับชิป AMD, Intel, NVIDIA และ Qualcomm  
- รองรับการดำเนินการบน CPU, GPU และ NPU พร้อมการสลับที่โปร่งใส  
- การแยกฮาร์ดแวร์ที่ช่วยลดงานปรับแต่งเฉพาะแพลตฟอร์ม  

**ความยืดหยุ่นของโมเดล**  
- รองรับรูปแบบโมเดล ONNX พร้อมการแปลงอัตโนมัติจากเฟรมเวิร์กยอดนิยม  
- การใช้งานโมเดลที่กำหนดเองด้วยประสิทธิภาพในระดับการผลิต  
- การผสานรวมกับสถาปัตยกรรมแอปพลิเคชัน Windows ที่มีอยู่  

**การผสานรวมระดับองค์กร**  
- เข้ากันได้กับกรอบความปลอดภัยและการปฏิบัติตามข้อกำหนดของ Windows  
- รองรับเครื่องมือการใช้งานและการจัดการระดับองค์กร  
- การผสานรวมกับระบบการจัดการและการตรวจสอบอุปกรณ์ Windows  

## เวิร์กโฟลว์การพัฒนา

### เฟส 1: การตั้งค่าสภาพแวดล้อมและการกำหนดค่าเครื่องมือ

**การเตรียมสภาพแวดล้อมการพัฒนา**  
1. ติดตั้ง Visual Studio พร้อมส่วนขยาย AI Toolkit  
2. กำหนดค่าเครื่องมือ CLI ของ Windows AI Foundry  
3. ตั้งค่าสภาพแวดล้อมการทดสอบโมเดลในพื้นที่  
4. จัดเตรียมเครื่องมือวิเคราะห์ประสิทธิภาพและการตรวจสอบ  

**การสำรวจ AI Dev Gallery**  
- สำรวจแอปพลิเคชันตัวอย่างและการใช้งานอ้างอิง  
- ทดสอบ Windows AI APIs ด้วยการสาธิตแบบโต้ตอบ  
- ทบทวนซอร์สโค้ดเพื่อแนวทางปฏิบัติและรูปแบบที่ดีที่สุด  
- ระบุตัวอย่างที่เกี่ยวข้องกับกรณีการใช้งานเฉพาะของคุณ  

### เฟส 2: การเลือกและการผสานรวมโมเดล

**การวิเคราะห์ความต้องการ**  
- กำหนดความต้องการด้านฟังก์ชันสำหรับความสามารถ AI  
- กำหนดข้อจำกัดด้านประสิทธิภาพและเป้าหมายการปรับแต่ง  
- ประเมินความต้องการด้านความเป็นส่วนตัวและความปลอดภัย  
- วางแผนสถาปัตยกรรมการใช้งานและกลยุทธ์การปรับขนาด  

**การประเมินโมเดล**  
- ใช้ Foundry Local เพื่อทดสอบโมเดลโอเพ่นซอร์สสำหรับกรณีการใช้งานของคุณ  
- เปรียบเทียบ Windows AI APIs กับความต้องการโมเดลที่กำหนดเอง  
- ประเมินข้อแลกเปลี่ยนระหว่างขนาดโมเดล ความแม่นยำ และความเร็วในการอนุมาน  
- สร้างต้นแบบวิธีการผสานรวมกับโมเดลที่เลือก  

### เฟส 3: การพัฒนาแอปพลิเคชัน

**การผสานรวมหลัก**  
- ใช้การผสานรวม Windows AI API พร้อมการจัดการข้อผิดพลาดที่เหมาะสม  
- ออกแบบส่วนติดต่อผู้ใช้ที่รองรับเวิร์กโฟลว์การประมวลผล AI  
- ใช้กลยุทธ์การแคชและการปรับแต่งสำหรับการอนุมานโมเดล  
- เพิ่มการวัดผลและการตรวจสอบสำหรับประสิทธิภาพการดำเนินการ AI  

**การทดสอบและการตรวจสอบ**  
- ทดสอบแอปพลิเคชันบนการกำหนดค่าฮาร์ดแวร์ Windows ต่างๆ  
- ตรวจสอบเมตริกประสิทธิภาพภายใต้เงื่อนไขการโหลดต่างๆ  
- ใช้การทดสอบอัตโนมัติสำหรับความน่าเชื่อถือของฟังก์ชัน AI  
- ดำเนินการทดสอบประสบการณ์ผู้ใช้ด้วยคุณสมบัติที่เพิ่มประสิทธิภาพด้วย AI  

### เฟส 4: การปรับแต่งและการใช้งาน

**การปรับแต่งประสิทธิภาพ**  
- วิเคราะห์ประสิทธิภาพแอปพลิเคชันบนการกำหนดค่าฮาร์ดแวร์เป้าหมาย  
- ปรับแต่งการใช้หน่วยความจำและกลยุทธ์การโหลดโมเดล  
- ใช้พฤติกรรมที่ปรับเปลี่ยนได้ตามความสามารถของฮาร์ดแวร์ที่มีอยู่  
- ปรับแต่งประสบการณ์ผู้ใช้สำหรับสถานการณ์ประสิทธิภาพที่แตกต่างกัน  

**การใช้งานในระดับการผลิต**  
- บรรจุแอปพลิเคชันพร้อมการพึ่งพาโมเดล AI ที่เหมาะสม  
- ใช้กลไกการอัปเดตสำหรับโมเดลและตรรกะของแอปพลิเคชัน  
- กำหนดค่าการตรวจสอบและการวิเคราะห์สำหรับสภาพแวดล้อมการผลิต  
- วางแผนกลยุทธ์การเปิดตัวสำหรับการใช้งานองค์กรและผู้บริโภค  

## ตัวอย่างการใช้งานจริง

### ตัวอย่าง 1: แอปพลิเคชันการประมวลผลเอกสารอัจฉริยะ

สร้างแอปพลิเคชัน Windows ที่ประมวลผลเอกสารโดยใช้ความสามารถ AI หลายอย่าง:

**เทคโนโลยีที่ใช้:**  
- Phi Silica สำหรับการสรุปเอกสารและการตอบคำถาม  
- OCR APIs สำหรับการดึงข้อความจากเอกสารที่สแกน  
- Image Description APIs สำหรับการวิเคราะห์แผนภูมิและไดอะแกรม  
- โมเดล ONNX
- ใช้ Foundry Local CLI สำหรับการทดสอบและตรวจสอบโมเดล
- ใช้เครื่องมือทดสอบ Windows AI API เพื่อยืนยันการทำงานร่วมกัน
- เพิ่มระบบบันทึกแบบกำหนดเองสำหรับการติดตามการทำงานของ AI
- สร้างการทดสอบอัตโนมัติสำหรับความน่าเชื่อถือของฟังก์ชัน AI

## การเตรียมแอปพลิเคชันให้พร้อมสำหรับอนาคต

### เทคโนโลยีที่กำลังมาแรง

**ฮาร์ดแวร์ยุคใหม่**
- ออกแบบแอปพลิเคชันให้รองรับความสามารถของ NPU ในอนาคต
- วางแผนสำหรับขนาดโมเดลที่ใหญ่ขึ้นและความซับซ้อนที่เพิ่มขึ้น
- ใช้สถาปัตยกรรมที่ปรับตัวได้สำหรับฮาร์ดแวร์ที่พัฒนาไปเรื่อย ๆ
- พิจารณาอัลกอริทึมที่พร้อมสำหรับควอนตัมเพื่อความเข้ากันได้ในอนาคต

**ความสามารถ AI ขั้นสูง**
- เตรียมพร้อมสำหรับการรวม AI แบบมัลติโหมดที่รองรับข้อมูลหลากหลายประเภท
- วางแผนสำหรับ AI ที่ทำงานร่วมกันแบบเรียลไทม์ระหว่างอุปกรณ์หลายตัว
- ออกแบบให้รองรับการเรียนรู้แบบกระจาย (Federated Learning)
- พิจารณาสถาปัตยกรรมอัจฉริยะแบบไฮบริดระหว่าง Edge และ Cloud

### การเรียนรู้และปรับตัวอย่างต่อเนื่อง

**การอัปเดตโมเดล**
- ใช้กลไกการอัปเดตโมเดลที่ราบรื่น
- ออกแบบแอปพลิเคชันให้ปรับตัวเข้ากับความสามารถของโมเดลที่พัฒนาขึ้น
- วางแผนสำหรับความเข้ากันได้ย้อนหลังกับโมเดลที่มีอยู่
- ใช้การทดสอบแบบ A/B เพื่อประเมินประสิทธิภาพของโมเดล

**การพัฒนาฟีเจอร์**
- ออกแบบสถาปัตยกรรมแบบโมดูลาร์ที่รองรับความสามารถ AI ใหม่ ๆ
- วางแผนสำหรับการรวม Windows AI API ที่กำลังพัฒนา
- ใช้ฟีเจอร์แฟล็กสำหรับการเปิดตัวความสามารถใหม่อย่างค่อยเป็นค่อยไป
- ออกแบบอินเทอร์เฟซผู้ใช้ที่ปรับตัวเข้ากับฟีเจอร์ AI ที่พัฒนาขึ้น

## สรุป

การพัฒนา Windows Edge AI เป็นการรวมความสามารถ AI ที่ทรงพลังเข้ากับแพลตฟอร์ม Windows ที่มีความปลอดภัย แข็งแกร่ง และสามารถขยายตัวได้ โดยการเรียนรู้และใช้งานระบบ Windows AI Foundry นักพัฒนาสามารถสร้างแอปพลิเคชันอัจฉริยะที่มอบประสบการณ์การใช้งานที่ยอดเยี่ยม พร้อมทั้งรักษามาตรฐานสูงสุดในด้านความเป็นส่วนตัว ความปลอดภัย และประสิทธิภาพ

การผสมผสานระหว่าง Windows AI APIs, Foundry Local และ Windows ML มอบรากฐานที่ไม่มีใครเทียบได้สำหรับการสร้างแอปพลิเคชัน Windows อัจฉริยะรุ่นใหม่ ในขณะที่ AI ยังคงพัฒนาไปเรื่อย ๆ แพลตฟอร์ม Windows จะช่วยให้แอปพลิเคชันของคุณสามารถขยายตัวไปพร้อมกับเทคโนโลยีที่เกิดขึ้นใหม่ และยังคงรักษาความเข้ากันได้และประสิทธิภาพในระบบฮาร์ดแวร์ Windows ที่หลากหลาย

ไม่ว่าคุณจะสร้างแอปพลิเคชันสำหรับผู้บริโภค โซลูชันสำหรับองค์กร หรือเครื่องมือเฉพาะทางในอุตสาหกรรม การพัฒนา Windows Edge AI จะช่วยให้คุณสร้างประสบการณ์ที่ชาญฉลาด ตอบสนอง และผสานรวมอย่างลึกซึ้ง โดยใช้ศักยภาพเต็มรูปแบบของอุปกรณ์ Windows สมัยใหม่
