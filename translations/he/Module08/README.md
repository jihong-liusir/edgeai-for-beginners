# מודול 08: עבודה מעשית עם Microsoft Foundry Local - ערכת כלים מלאה למפתחים

## סקירה כללית

Microsoft Foundry Local מייצג את הדור הבא של פיתוח AI בקצה, ומספק למפתחים כלים עוצמתיים לבנייה, פריסה והרחבה של יישומי AI באופן מקומי תוך שמירה על אינטגרציה חלקה עם Azure AI Foundry. מודול זה מספק כיסוי מקיף של Foundry Local, החל מהתקנה ועד פיתוח מתקדם של סוכנים.

**טכנולוגיות מרכזיות:**
- Microsoft Foundry Local CLI ו-SDK
- אינטגרציה עם Azure AI Foundry
- הסקת מודלים על המכשיר
- שמירה ואופטימיזציה של מודלים באופן מקומי
- ארכיטקטורות מבוססות סוכנים

## מטרות למידה של המודול

עם סיום המודול, תוכלו:

- **לשלוט בהגדרת Foundry Local**: להתקין, להגדיר ולבצע אופטימיזציה ל-Foundry Local עבור פיתוח ב-Windows 11  
- **לפרוס מודלים מגוונים**: להריץ מודלים כמו phi, qwen, deepseek ו-GPT-OSS-20B באופן מקומי באמצעות פקודות CLI  
- **לבנות פתרונות ייצור**: ליצור יישומי AI עם הנדסת הנחיות מתקדמת ואינטגרציית נתונים  
- **לנצל את האקוסיסטם הפתוח**: לשלב מודלים של Hugging Face ותוספות מבוססות קהילה  
- **להשוות ארכיטקטורות AI**: להבין את ההבדלים בין LLMs ל-SLMs ואת אסטרטגיות הפריסה  
- **לפתח סוכני AI**: לבנות סוכנים חכמים באמצעות הארכיטקטורה והיכולות של Foundry Local  
- **ליישם מודלים ככלים**: ליצור פתרונות AI מודולריים ומותאמים ליישומים ארגוניים

## דרישות מוקדמות

### דרישות מערכת
- **מערכת הפעלה**: Windows 11 (22H2 או גרסה מאוחרת יותר)  
- **זיכרון**: 16GB RAM (מומלץ 32GB עבור מודלים גדולים יותר)  
- **אחסון**: 50GB שטח פנוי לשמירת מודלים  
- **חומרה**: מכשיר עם NPU מומלץ (Copilot+ PC), GPU אופציונלי  
- **רשת**: אינטרנט מהיר להורדת מודלים ראשונית  

### סביבת פיתוח
- Visual Studio Code עם הרחבת AI Toolkit  
- Python 3.10+ ו-pip  
- Git לניהול גרסאות  
- PowerShell או שורת פקודה  
- Azure CLI (אופציונלי לאינטגרציה עם הענן)  

### ידע מוקדם
- הבנה בסיסית של מושגי AI/ML  
- היכרות עם שורת הפקודה  
- יסודות תכנות ב-Python  
- מושגי REST API  
- ידע בסיסי בהנחיות והסקת מודלים

## התחלה מהירה לדוגמאות

### 1) הגדרת סביבה (Windows cmd.exe)
```cmd
cd Module08
py -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```
  
### 2) התחלת מודל מקומי (טרמינל חדש)
```cmd
foundry model list
foundry model run phi-4-mini
```
  
### 3) הרצת הדגמת Chainlit (מפגש 4)
```cmd
cd Module08
.\.venv\Scripts\activate
chainlit run samples\04\app.py -w
```
  
### 4) הרצת מתאם רב-סוכנים (מפגש 5)
```cmd
cd Module08
.\.venv\Scripts\activate
python -m samples.05.agents.coordinator
```
  
אם אתם רואים שגיאות חיבור, בדקו את Foundry Local:
```cmd
curl http://localhost:8000/v1/models
```
  
### הגדרת נתב (מפגש 6)
הנתב מבצע בדיקת בריאות מהירה ותומך בהגדרות מבוססות סביבה:
```cmd
set BASE_URL=http://localhost:8000
set GENERAL_MODEL=phi-4-mini
set REASONING_MODEL=deepseek-r1-distill-qwen-7b
set CODE_MODEL=qwen2.5-7b-instruct
REM Or provide a full JSON registry
set TOOL_REGISTRY={"general":{"model":"phi-4-mini"}}
python samples\06\router.py "Pick the best model for code refactoring"
```
  
מודול זה מייצג את חזית הפיתוח של AI בקצה, ומשלב את הכלים הארגוניים של מיקרוסופט עם הגמישות והחדשנות של האקוסיסטם הפתוח. עם שליטה ב-Foundry Local, תהיו בעמדה מובילה בפיתוח יישומי AI.

עבור Azure OpenAI (מפגש 2), ראו את קובץ README לדוגמא עבור משתני סביבה נדרשים והגדרות גרסת API.
