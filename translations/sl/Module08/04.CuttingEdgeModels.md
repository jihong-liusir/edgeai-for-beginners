# Seja 4: Napredni modeli – LLM-ji, SLM-ji in sklepanje na napravi

## Pregled

Primerjajte LLM-je in SLM-je, ocenite kompromise med lokalnim in oblačnim sklepanjem ter izvedite demonstracije, ki prikazujejo scenarije EdgeAI z uporabo Phi in ONNX Runtime. Poudarili bomo tudi Chainlit RAG, možnosti sklepanja z WebGPU in integracijo Open WebUI.

Reference:
- Dokumentacija Foundry Local: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Navodila za Open WebUI (aplikacija za klepet z Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Cilji učenja
- Razumeti kompromise med LLM-ji in SLM-ji glede stroškov, zakasnitve in natančnosti
- Izbrati med lokalnim in oblačnim sklepanjem glede na specifične poslovne potrebe
- Izvesti majhno RAG demonstracijo z Chainlit
- Raziskati WebGPU za pospeševanje na strani brskalnika
- Povezati Open WebUI z Foundry Local

## 1. del: LLM vs SLM – Odločitvena matrika

Razmislite o:
- Zakasnitvi: SLM-ji na napravi pogosto zagotavljajo odzive v manj kot sekundi
- Stroških: lokalno sklepanje zmanjšuje stroške oblačnih storitev
- Zasebnosti: občutljivi podatki ostanejo na napravi
- Zmogljivosti: LLM-ji lahko prekašajo SLM-je pri kompleksnih nalogah
- Zanesljivosti: hibridne strategije zmanjšujejo tveganje izpadov

## 2. del: Lokalno vs oblak – Hibridni vzorci

- Lokalno najprej z oblačno podporo za velike/kompleksne zahteve
- Oblak najprej z lokalnim za scenarije, občutljive na zasebnost ali brez povezave
- Usmerjanje glede na vrsto naloge (generiranje kode na DeepSeek, splošni klepet na Phi/Qwen)

## 3. del: RAG aplikacija za klepet z Chainlit (Minimalno)

Namestite odvisnosti:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Zaženite:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Razširite: dodajte preprost iskalnik (lokalne datoteke) in pred uporabniško zahtevo dodajte pridobljeni kontekst.

## 4. del: Sklepanje z WebGPU (Opozorilo)

Zaženite majhne modele neposredno v brskalniku z uporabo WebGPU. To je idealno za demonstracije, ki dajejo prednost zasebnosti, in izkušnje brez namestitve. Spodaj je minimalen, korak za korakom primer z uporabo ONNX Runtime Web z izvajalcem WebGPU.

1) Preverite podporo za WebGPU
- Brskalniki Chromium: chrome://gpu → potrdite, da je “WebGPU” omogočen
- Programatska preverba (preverili bomo tudi v kodi): `if (!('gpu' in navigator)) { /* no WebGPU */ }`

2) Ustvarite minimalen projekt
Ustvarite mapo in dve datoteki: `index.html` in `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Strežite lokalno (Windows cmd.exe)
Uporabite preprost statični strežnik, da brskalnik lahko pridobi model.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Odprite http://localhost:5173 v brskalniku. Videti bi morali dnevnike inicializacije, ustvarjanje seje z WebGPU in napoved argmax.

4) Odpravljanje težav
- Če WebGPU ni na voljo: posodobite Chrome/Edge in zagotovite, da so gonilniki GPU posodobljeni, nato preverite chrome://flags za “Enable WebGPU”.
- Če pride do napak CORS ali pridobivanja: zagotovite, da strežete datoteke prek http:// (ne file://) in da URL modela omogoča zahteve iz drugih izvorov.
- Preklop na CPU: spremenite `executionProviders: ['wasm']`, da preverite osnovno delovanje.

5) Naslednji koraki
- Zamenjajte z modelom ONNX, specifičnim za domeno (npr. klasifikacija slik ali majhen besedilni model).
- Dodajte logiko za predhodno/po obdelavo za resnične vnose.
- Za večje modele ali produkcijsko zakasnitev raje uporabite Foundry Local ali ONNX Runtime Server.

## 5. del: Open WebUI + Foundry Local (Korak za korakom)

To povezuje Open WebUI z OpenAI-kompatibilno končno točko Foundry Local za lokalni uporabniški vmesnik za klepet.

1) Predpogoji
- Foundry Local nameščen in delujoč (`foundry --version`)
- En model pripravljen za lokalno izvajanje (npr. `phi-4-mini`)
- Nameščen Docker Desktop (priporočeno za Open WebUI)

2) Zaženite model z Foundry Local
```powershell
foundry model run phi-4-mini
```
To omogoča OpenAI-kompatibilni API na `http://localhost:8000`.

3) Zaženite Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Opombe:
- Na Windowsu `host.docker.internal` omogoča, da vsebnik doseže vaš gostitelj na `localhost`.
- Nastavimo `OPENAI_API_BASE_URL` na končno točko Foundry Local in lažni `OPENAI_API_KEY`.

4) Konfigurirajte iz uporabniškega vmesnika Open WebUI (alternativa)
- Odprite http://localhost:3000
- Dokončajte začetno nastavitev (admin uporabnik)
- Pojdite na Nastavitve → Modeli/Ponudniki
- Nastavite osnovni URL: `http://host.docker.internal:8000/v1`
- Nastavite API ključ: `local-key` (nadomestni ključ)
- Shranite

5) Preizkusite zahtevo
- V klepetu Open WebUI izberite ali vnesite ime modela `phi-4-mini`
- Zahteva: “Naštej pet prednosti sklepanja na napravi.”
- Videti bi morali odziv, ki se pretaka iz vašega lokalnega modela.

6) Odpravljanje težav
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Neobvezno: Ohranjanje podatkov Open WebUI
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
