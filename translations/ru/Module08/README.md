# Модуль 08: Практическое знакомство с Microsoft Foundry Local - Полный набор инструментов для разработчиков

## Обзор

Microsoft Foundry Local представляет собой новое поколение разработки Edge AI, предоставляя разработчикам мощные инструменты для создания, развертывания и масштабирования AI-приложений локально, с бесшовной интеграцией с Azure AI Foundry. Этот модуль охватывает Foundry Local от установки до разработки продвинутых агентов.

**Ключевые технологии:**
- Microsoft Foundry Local CLI и SDK
- Интеграция с Azure AI Foundry
- Локальная инференция моделей
- Кэширование и оптимизация моделей на устройстве
- Архитектуры, основанные на агентах

## Цели обучения модуля

После завершения этого модуля вы сможете:

- **Освоить настройку Foundry Local**: Установить, настроить и оптимизировать Foundry Local для разработки на Windows 11
- **Развернуть различные модели**: Запускать модели phi, qwen, deepseek и GPT-OSS-20B локально с помощью CLI-команд
- **Создавать производственные решения**: Разрабатывать AI-приложения с использованием продвинутой инженерии подсказок и интеграции данных
- **Использовать экосистему с открытым исходным кодом**: Интегрировать модели Hugging Face и дополнения, созданные сообществом
- **Сравнивать архитектуры AI**: Понимать компромиссы между LLM и SLM и стратегии их развертывания
- **Разрабатывать AI-агентов**: Создавать интеллектуальных агентов с использованием архитектуры Foundry Local и возможностей привязки
- **Реализовывать модели как инструменты**: Создавать модульные, настраиваемые AI-решения для корпоративных приложений

## Предварительные требования

### Системные требования
- **Операционная система**: Windows 11 (22H2 или новее)
- **Память**: 16GB RAM (32GB рекомендуется для больших моделей)
- **Хранилище**: 50GB свободного места для кэширования моделей
- **Оборудование**: Устройство с поддержкой NPU рекомендуется (Copilot+ PC), GPU опционально
- **Сеть**: Высокоскоростной интернет для первоначальной загрузки моделей

### Среда разработки
- Visual Studio Code с расширением AI Toolkit
- Python 3.10+ и pip
- Git для контроля версий
- PowerShell или Command Prompt
- Azure CLI (опционально для облачной интеграции)

### Знания
- Базовое понимание концепций AI/ML
- Знакомство с командной строкой
- Основы программирования на Python
- Концепции REST API
- Базовые знания о подсказках и инференции моделей

## Быстрый старт для примеров

### 1) Настройка среды (Windows cmd.exe)
```cmd
cd Module08
py -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Запуск локальной модели (новый терминал)
```cmd
foundry model list
foundry model run phi-4-mini
```

### 3) Запуск демонстрации Chainlit (Сессия 4)
```cmd
cd Module08
.\.venv\Scripts\activate
chainlit run samples\04\app.py -w
```

### 4) Запуск координатора для нескольких агентов (Сессия 5)
```cmd
cd Module08
.\.venv\Scripts\activate
python -m samples.05.agents.coordinator
```

Если вы видите ошибки подключения, проверьте Foundry Local:
```cmd
curl http://localhost:8000/v1/models
```

### Настройка маршрутизатора (Сессия 6)
Маршрутизатор выполняет быструю проверку состояния и поддерживает конфигурацию на основе переменных среды:
```cmd
set BASE_URL=http://localhost:8000
set GENERAL_MODEL=phi-4-mini
set REASONING_MODEL=deepseek-r1-distill-qwen-7b
set CODE_MODEL=qwen2.5-7b-instruct
REM Or provide a full JSON registry
set TOOL_REGISTRY={"general":{"model":"phi-4-mini"}}
python samples\06\router.py "Pick the best model for code refactoring"
```

Этот модуль представляет собой передовой подход к разработке Edge AI, объединяя корпоративные инструменты Microsoft с гибкостью и инновациями экосистемы с открытым исходным кодом. Освоив Foundry Local, вы окажетесь на передовой линии разработки AI-приложений.

Для Azure OpenAI (Сессия 2) ознакомьтесь с README примера для необходимых переменных среды и настроек версии API.
