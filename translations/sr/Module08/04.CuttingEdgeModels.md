# Сесија 4: Најсавременији модели – LLM, SLM и инференција на уређају

## Преглед

Упоредите LLM и SLM, процените компромисе између локалне и облачне инференције, и имплементирајте демонстрације које приказују EdgeAI сценарије користећи Phi и ONNX Runtime. Такође ћемо истакнути Chainlit RAG, опције инференције преко WebGPU и интеграцију Open WebUI.

Референце:
- Foundry Local документација: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Open WebUI упутство (апликација за ћаскање са Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Циљеви учења
- Разумети компромисе између LLM и SLM у погледу трошкова, кашњења и тачности
- Изабрати између локалне и облачне инференције за специфичне пословне потребе
- Имплементирати мали RAG демонстрациони пример са Chainlit
- Истражити WebGPU за убрзање у претраживачу
- Повезати Open WebUI са Foundry Local

## Део 1: LLM vs SLM – Матрица одлуке

Размотрите:
- Кашњење: SLM на уређају често пружа одговоре у мање од секунде
- Трошкови: локална инференција смањује трошкове облака
- Приватност: осетљиви подаци остају на уређају
- Способност: LLM може надмашити SLM у сложеним задацима
- Поузданост: хибридне стратегије смањују ризик од застоја

## Део 2: Локално vs Облак – Хибридни обрасци

- Локално-прво са облаком као резервом за велике/сложене упите
- Облак-прво са локалним за сценарије осетљиве на приватност или без интернет конекције
- Рута по типу задатка (генерација кода ка DeepSeek, општи разговор ка Phi/Qwen)

## Део 3: RAG апликација за ћаскање са Chainlit (Минимално)

Инсталирајте зависности:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Покрените:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Проширите: додајте једноставан претраживач (локалне датотеке) и додајте пронађени контекст на почетак корисничког упита.

## Део 4: WebGPU инференција (Упозорење)

Покрените мале моделе директно у претраживачу користећи WebGPU. Ово је идеално за демонстрације које приоритет дају приватности и искуства без инсталације. Испод је минималан, корак-по-корак пример користећи ONNX Runtime Web са WebGPU провајдером за извршавање.

1) Проверите подршку за WebGPU
- Chromium претраживачи: chrome://gpu → потврдите да је “WebGPU” омогућен
- Програмска провера (такође ћемо проверити у коду): `if (!('gpu' in navigator)) { /* нема WebGPU */ }`

2) Направите минималан пројекат
Направите фасциклу и две датотеке: `index.html` и `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Сервирајте локално (Windows cmd.exe)
Користите једноставан статички сервер како би претраживач могао да преузме модел.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Отворите http://localhost:5173 у вашем претраживачу. Требало би да видите логове иницијализације, креирање сесије са WebGPU и предвиђање са argmax.

4) Решавање проблема
- Ако WebGPU није доступан: ажурирајте Chrome/Edge и уверите се да су GPU драјвери ажурни, затим проверите chrome://flags за “Enable WebGPU”.
- Ако се јављају CORS или fetch грешке: уверите се да сервирате датотеке преко http:// (не file://) и да URL модела дозвољава захтеве из других домена.
- Прелазак на CPU: промените `executionProviders: ['wasm']` да проверите основно понашање.

5) Следећи кораци
- Замените модел специфичан за домен (нпр. класификација слика или мали текстуални модел).
- Додајте логику за предобраду/постобраду за стварне уносе.
- За веће моделе или производно кашњење, преферирајте Foundry Local или ONNX Runtime Server.

## Део 5: Open WebUI + Foundry Local (Корак-по-корак)

Ово повезује Open WebUI са OpenAI-компатибилним крајњим тачкама Foundry Local за локални UI за ћаскање.

1) Предуслови
- Foundry Local инсталиран и функционалан (`foundry --version`)
- Један модел спреман за локално покретање (нпр. `phi-4-mini`)
- Docker Desktop инсталиран (препоручено за Open WebUI)

2) Покрените модел са Foundry Local
```powershell
foundry model run phi-4-mini
```
Ово излаже OpenAI-компатибилни API на `http://localhost:8000`.

3) Покрените Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Напомене:
- На Windows-у, `host.docker.internal` омогућава контејнеру да приступи вашем хосту на `localhost`.
- Постављамо `OPENAI_API_BASE_URL` на крајњу тачку Foundry Local и лажни `OPENAI_API_KEY`.

4) Конфигуришите из Open WebUI UI (алтернатива)
- Отворите http://localhost:3000
- Завршите почетну конфигурацију (администраторски корисник)
- Идите на Settings → Models/Providers
- Поставите Base URL: `http://host.docker.internal:8000/v1`
- Поставите API Key: `local-key` (замена)
- Сачувајте

5) Покрените тестни упит
- У Open WebUI ћаскању, изаберите или унесите име модела `phi-4-mini`
- Упит: “Наведите пет предности инференције на уређају.”
- Требало би да видите одговор који се стримује из вашег локалног модела

6) Решавање проблема
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Опционо: Перзистирајте податке Open WebUI
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
