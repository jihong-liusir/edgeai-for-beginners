# সেশন ৩: ওপেন-সোর্স মডেল আবিষ্কার এবং ব্যবস্থাপনা

## সংক্ষিপ্ত বিবরণ

এই সেশনটি Foundry Local ব্যবহার করে মডেল আবিষ্কার এবং ব্যবস্থাপনার উপর ভিত্তি করে। আপনি উপলব্ধ মডেলগুলোর তালিকা তৈরি, বিভিন্ন অপশন পরীক্ষা এবং মৌলিক পারফরম্যান্স বৈশিষ্ট্যগুলো বুঝতে শিখবেন। এই পদ্ধতি হাতে-কলমে Foundry CLI ব্যবহার করে সঠিক মডেল নির্বাচন করার উপর জোর দেয়।

## শেখার লক্ষ্যসমূহ

- মডেল আবিষ্কার এবং ব্যবস্থাপনার জন্য Foundry CLI কমান্ড আয়ত্ত করা
- মডেল ক্যাশ এবং লোকাল স্টোরেজ প্যাটার্নগুলো বোঝা
- দ্রুত বিভিন্ন মডেল পরীক্ষা এবং তুলনা করা
- মডেল নির্বাচন এবং বেঞ্চমার্কিংয়ের জন্য ব্যবহারিক কর্মপ্রবাহ প্রতিষ্ঠা করা
- Foundry Local-এর মাধ্যমে উপলব্ধ মডেলগুলোর ক্রমবর্ধমান ইকোসিস্টেম অন্বেষণ করা

## পূর্বশর্ত

- সেশন ১: Foundry Local-এর সাথে শুরু করার কাজ সম্পন্ন
- Foundry Local CLI ইনস্টল এবং অ্যাক্সেসযোগ্য
- মডেল ডাউনলোডের জন্য পর্যাপ্ত স্টোরেজ স্পেস (মডেলগুলোর আকার ১GB থেকে ২০GB+ হতে পারে)
- মডেল টাইপ এবং ব্যবহার ক্ষেত্র সম্পর্কে মৌলিক ধারণা

## অংশ ৬: হাতে-কলমে অনুশীলন

### অনুশীলন: মডেল আবিষ্কার এবং তুলনা

Sample 03-এর উপর ভিত্তি করে আপনার নিজস্ব মডেল মূল্যায়ন স্ক্রিপ্ট তৈরি করুন:

```cmd
REM create_model_test.cmd
@echo off
echo Model Discovery and Testing Script
echo =====================================

echo.
echo Step 1: List available models
foundry model list

echo.
echo Step 2: Check what's cached
foundry cache list

echo.
echo Step 3: Start phi-4-mini for testing
foundry model run phi-4-mini --verbose

echo.
echo Step 4: Test with a simple prompt
curl -X POST http://localhost:8000/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"phi-4-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"Hello, please introduce yourself.\"}],\"max_tokens\":100}"

echo.
echo Model test complete!
```

### আপনার কাজ

1. **Sample 03 স্ক্রিপ্ট চালান**: `samples\03\list_and_bench.cmd`
2. **বিভিন্ন মডেল চেষ্টা করুন**: অন্তত ৩টি ভিন্ন মডেল পরীক্ষা করুন
3. **পারফরম্যান্স তুলনা করুন**: গতি এবং প্রতিক্রিয়ার গুণমানের পার্থক্য লক্ষ্য করুন
4. **ফলাফল নথিভুক্ত করুন**: একটি সহজ তুলনা চার্ট তৈরি করুন

### উদাহরণ তুলনা ফরম্যাট

```
Model Comparison Results:
========================
phi-4-mini:        Fast (~2s), good for general chat
qwen2.5-7b:       Slower (~5s), better reasoning  
deepseek-r1:      Medium (~3s), excellent for code

Recommendation: Start with phi-4-mini for development, 
switch to qwen2.5-7b for production reasoning tasks.
```

## অংশ ৭: সমস্যা সমাধান এবং সেরা অনুশীলন

### সাধারণ সমস্যা এবং সমাধান

**মডেল চালু হচ্ছে না:**
```cmd
REM Check service status
foundry service status

REM Restart service if needed
foundry service stop
foundry service start

REM Try with verbose output
foundry model run phi-4-mini --verbose
```

**অপর্যাপ্ত মেমোরি:**
- ছোট মডেল দিয়ে শুরু করুন (`phi-4-mini`)
- অন্যান্য অ্যাপ্লিকেশন বন্ধ করুন
- যদি বারবার সীমা অতিক্রম হয়, RAM আপগ্রেড করুন

**ধীর পারফরম্যান্স:**
- নিশ্চিত করুন মডেল সম্পূর্ণ লোড হয়েছে (verbose আউটপুট চেক করুন)
- অপ্রয়োজনীয় ব্যাকগ্রাউন্ড অ্যাপ্লিকেশন বন্ধ করুন
- দ্রুত স্টোরেজ (SSD) বিবেচনা করুন

### সেরা অনুশীলন

1. **ছোট থেকে শুরু করুন**: সেটআপ যাচাই করতে `phi-4-mini` দিয়ে শুরু করুন
2. **একটি মডেল একবারে**: নতুন মডেল চালানোর আগে পূর্ববর্তী মডেল বন্ধ করুন
3. **রিসোর্স মনিটর করুন**: মেমোরি ব্যবহারের দিকে নজর রাখুন
4. **সঙ্গতিপূর্ণভাবে পরীক্ষা করুন**: ন্যায্য তুলনার জন্য একই প্রম্পট ব্যবহার করুন
5. **ফলাফল নথিভুক্ত করুন**: আপনার ব্যবহার ক্ষেত্রে মডেলের পারফরম্যান্স সম্পর্কে নোট রাখুন

## অংশ ১: Foundry CLI দিয়ে মডেল আবিষ্কার

### মৌলিক মডেল ব্যবস্থাপনা কমান্ড

Foundry CLI মডেল আবিষ্কার এবং ব্যবস্থাপনার জন্য সহজ কমান্ড প্রদান করে:

```cmd
REM List all available models in the catalog
foundry model list

REM List cached (downloaded) models
foundry cache list

REM Check cache directory location
foundry cache ls
```

### আপনার প্রথম মডেল চালানো

পারফরম্যান্স বৈশিষ্ট্যগুলো বুঝতে জনপ্রিয় এবং ভালোভাবে পরীক্ষিত মডেল দিয়ে শুরু করুন:

```cmd
REM Run Phi-4-Mini (lightweight, fast)
foundry model run phi-4-mini --verbose

REM Run Qwen 2.5 7B (larger, more capable)
foundry model run qwen2.5-7b-instruct --verbose

REM Run DeepSeek (specialized for coding)
foundry model run deepseek-r1-distill-qwen-7b --verbose
```

**নোট:** `--verbose` ফ্ল্যাগটি বিস্তারিত স্টার্টআপ তথ্য প্রদান করে, যার মধ্যে রয়েছে:
- প্রথমবার চালানোর সময় মডেল ডাউনলোডের অগ্রগতি
- মেমোরি বরাদ্দের বিবরণ
- সার্ভিস বাইন্ডিং তথ্য
- পারফরম্যান্স ইনিশিয়ালাইজেশন মেট্রিক্স

### মডেল ক্যাটাগরি বোঝা

**ছোট ভাষা মডেল (SLMs):**
- `phi-4-mini`: দ্রুত, দক্ষ, সাধারণ চ্যাটের জন্য চমৎকার
- `phi-4`: আরও সক্ষম সংস্করণ, উন্নত যুক্তি সহ

**মাঝারি মডেল:**
- `qwen2.5-7b-instruct`: চমৎকার যুক্তি এবং দীর্ঘতর প্রসঙ্গ
- `deepseek-r1-distill-qwen-7b`: কোড জেনারেশনের জন্য অপ্টিমাইজড

**বড় মডেল:**
- `llama-3.2`: মেটার সর্বশেষ ওপেন-সোর্স মডেল
- `qwen2.5-14b-instruct`: এন্টারপ্রাইজ-গ্রেড যুক্তি

## অংশ ২: দ্রুত মডেল পরীক্ষা এবং তুলনা

### Sample 03 পদ্ধতি: সহজ তালিকা এবং বেঞ্চ

Sample 03 প্যাটার্নের উপর ভিত্তি করে, এখানে ন্যূনতম কর্মপ্রবাহ:

```cmd
@echo off
REM Sample 03 - List and bench pattern
echo Listing available models...
foundry model list

echo.
echo Checking cached models...
foundry cache list

echo.
echo Starting phi-4-mini with verbose output...
foundry model run phi-4-mini --verbose
```

### মডেল পারফরম্যান্স পরীক্ষা

একটি মডেল চালানোর পরে, সঙ্গতিপূর্ণ প্রম্পট দিয়ে এটি পরীক্ষা করুন:

```cmd
REM Test via curl (Windows Command Prompt)
curl -X POST http://localhost:8000/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"phi-4-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"Explain edge AI in one sentence.\"}],\"max_tokens\":50}"
```

### PowerShell পরীক্ষার বিকল্প

```powershell
# PowerShell approach for testing
$body = @{
    model = "phi-4-mini"
    messages = @(
        @{
            role = "user"
            content = "Explain edge AI in one sentence."
        }
    )
    max_tokens = 50
} | ConvertTo-Json -Depth 3

Invoke-RestMethod -Uri "http://localhost:8000/v1/chat/completions" -Method Post -Body $body -ContentType "application/json"
```

## অংশ ৩: মডেল ক্যাশ এবং স্টোরেজ ব্যবস্থাপনা

### মডেল ক্যাশ বোঝা

Foundry Local স্বয়ংক্রিয়ভাবে মডেল ডাউনলোড এবং ক্যাশিং পরিচালনা করে:

```cmd
REM Check cache directory and contents
foundry cache ls

REM View cache location
foundry cache cd

REM Clean up unused models (if needed)
foundry cache clean
```

### মডেল স্টোরেজ বিবেচনা

**সাধারণ মডেল আকার:**
- `phi-4-mini`: ~২.৫ GB
- `qwen2.5-7b-instruct`: ~৪.১ GB  
- `deepseek-r1-distill-qwen-7b`: ~৪.৩ GB
- `llama-3.2`: ~৪.৯ GB
- `qwen2.5-14b-instruct`: ~৮.২ GB

**স্টোরেজ সেরা অনুশীলন:**
- দ্রুত সুইচিংয়ের জন্য ২-৩টি মডেল ক্যাশ রাখুন
- অপ্রয়োজনীয় মডেল সরিয়ে স্থান মুক্ত করুন: `foundry cache clean`
- ছোট SSD-তে ডিস্ক ব্যবহার মনিটর করুন
- মডেলের আকার বনাম সক্ষমতার মধ্যে ভারসাম্য বিবেচনা করুন

### মডেল পারফরম্যান্স মনিটরিং

মডেল চালানোর সময়, সিস্টেম রিসোর্স মনিটর করুন:

**Windows Task Manager:**
- মেমোরি ব্যবহার দেখুন (মডেল RAM-এ লোড থাকে)
- ইনফারেন্সের সময় CPU ব্যবহার পর্যবেক্ষণ করুন
- প্রাথমিক মডেল লোডিংয়ের সময় ডিস্ক I/O চেক করুন

**কমান্ড লাইন মনিটরিং:**
```cmd
REM Check memory usage (PowerShell)
Get-Process | Where-Object {$_.ProcessName -like "*foundry*"} | Select-Object ProcessName, WorkingSet64

REM Monitor running models
foundry service ps
```

## অংশ ৪: ব্যবহারিক মডেল নির্বাচন নির্দেশিকা

## অংশ ৫: সহজ মডেল বেঞ্চমার্কিং

### মৌলিক পারফরম্যান্স পরীক্ষা

মডেল পারফরম্যান্স তুলনা করার একটি সরল পদ্ধতি এখানে:

```python
# simple_bench.py - Based on Sample 03 patterns
import time
import requests
import json

def test_model_response(model_name, prompt="Explain edge AI in one sentence."):
    """Test a single model with a prompt and measure response time."""
    start_time = time.time()
    
    try:
        response = requests.post(
            "http://localhost:8000/v1/chat/completions",
            headers={"Content-Type": "application/json"},
            json={
                "model": model_name,
                "messages": [{"role": "user", "content": prompt}],
                "max_tokens": 64
            },
            timeout=30
        )
        
        elapsed = time.time() - start_time
        
        if response.status_code == 200:
            result = response.json()
            return {
                "model": model_name,
                "latency_sec": round(elapsed, 3),
                "response": result["choices"][0]["message"]["content"],
                "status": "success"
            }
        else:
            return {
                "model": model_name,
                "status": "error",
                "error": f"HTTP {response.status_code}"
            }
            
    except Exception as e:
        return {
            "model": model_name,
            "status": "error", 
            "error": str(e)
        }

# Test the currently running model
if __name__ == "__main__":
    # Test with different models (start each model first)
    test_models = ["phi-4-mini", "qwen2.5-7b-instruct", "deepseek-r1-distill-qwen-7b"]
    
    print("Model Performance Test")
    print("=" * 50)
    
    for model in test_models:
        print(f"\nTesting {model}...")
        print("Note: Make sure this model is running first with 'foundry model run {model}'")
        
        result = test_model_response(model)
        
        if result["status"] == "success":
            print(f"✅ {model}: {result['latency_sec']}s")
            print(f"   Response: {result['response'][:100]}...")
        else:
            print(f"❌ {model}: {result['error']}")
```
