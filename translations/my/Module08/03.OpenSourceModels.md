# အစည်းအဝေး ၃: အခမဲ့ မော်ဒယ်ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်း

## အကျဉ်းချုပ်

ဒီအစည်းအဝေးမှာ Foundry Local ကို အသုံးပြုပြီး မော်ဒယ်ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်းကို လက်တွေ့ကျကျ လေ့လာပါမည်။ မော်ဒယ်များကို စာရင်းပြုစုခြင်း၊ ရွေးချယ်ခြင်းနှင့် အခြေခံစွမ်းဆောင်ရည်လက္ခဏာများကို နားလည်ခြင်းအပေါ် အခြေခံထားသည်။ Foundry CLI ကို အသုံးပြု၍ သင့်အသုံးအတွက် သင့်တော်သော မော်ဒယ်များကို ရွေးချယ်နိုင်ရန် လက်တွေ့ကျကျ လေ့လာမှုကို အဓိကထားပါသည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

- မော်ဒယ်ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်းအတွက် Foundry CLI အမိန့်များကို ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ရန်
- မော်ဒယ် cache နှင့် ဒေသခံသိုလှောင်မှု ပုံစံများကို နားလည်ရန်
- မော်ဒယ်များကို အလျင်အမြန် စမ်းသပ်ပြီး နှိုင်းယှဉ်နိုင်ရန်
- မော်ဒယ်ရွေးချယ်ခြင်းနှင့် စွမ်းဆောင်ရည်တိုင်းတာမှုအတွက် လက်တွေ့ workflow များကို တည်ဆောက်ရန်
- Foundry Local မှတဆင့် ရရှိနိုင်သော မော်ဒယ်များ၏ တိုးတက်လာသော ecosystem ကို ရှာဖွေရန်

## ကြိုတင်လိုအပ်ချက်များ

- အစည်းအဝေး ၁: Foundry Local ကို စတင်အသုံးပြုခြင်းကို ပြီးစီးထားရမည်
- Foundry Local CLI ကို ထည့်သွင်းပြီး အသုံးပြုနိုင်ရမည်
- မော်ဒယ်များကို ဒေါင်းလုဒ်လုပ်ရန် လုံလောက်သော သိုလှောင်မှုနေရာ (မော်ဒယ်များသည် ၁GB မှ ၂၀GB+ အထိ ရှိနိုင်သည်)
- မော်ဒယ်အမျိုးအစားများနှင့် အသုံးပြုမှုများအပေါ် အခြေခံနားလည်မှု

## အကျဉ်းချုပ်

ဒီအစည်းအဝေးမှာ Foundry Local ကို အသုံးပြု၍ အခမဲ့ မော်ဒယ်များကို ရယူခြင်း၊ Hugging Face အကြောင်းအရာများကို ပေါင်းစည်းခြင်းနှင့် “သင့်မော်ဒယ်ကို သင့်ကိုယ်တိုင် ယူလာပါ” (BYOM) မဟာဗျူဟာများကို လေ့လာပါမည်။ Model Mondays စီးရီးကိုလည်း ရှာဖွေခြင်းနှင့် မော်ဒယ်များကို ဆက်လက်လေ့လာရန် ရှာဖွေပါမည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

- ဒေသခံ inference အတွက် အခမဲ့ မော်ဒယ်များကို ရှာဖွေပြီး အကဲဖြတ်နိုင်ရန်
- Foundry Local တွင် Hugging Face မော်ဒယ်များကို ရွေးချယ်ပြီး အကဲဖြတ်နိုင်ရန်
- တိကျမှု၊ latency နှင့် အရင်းအမြစ်လိုအပ်ချက်များအပေါ် မော်ဒယ်ရွေးချယ်မှု မဟာဗျူဟာများကို အသုံးပြုနိုင်ရန်
- Cache နှင့် versioning ဖြင့် ဒေသခံတွင် မော်ဒယ်များကို စီမံခန့်ခွဲနိုင်ရန်

## အပိုင်း ၁: Foundry CLI ဖြင့် မော်ဒယ်ရှာဖွေခြင်း

### မော်ဒယ်စီမံခန့်ခွဲမှုအတွက် အခြေခံအမိန့်များ

Foundry CLI သည် မော်ဒယ်ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်းအတွက် ရိုးရှင်းသော အမိန့်များကို ပေးသည်။

```cmd
REM List all available models in the catalog
foundry model list

REM List cached (downloaded) models
foundry cache list

REM Check cache directory location
foundry cache ls
```

### သင့်ပထမဆုံး မော်ဒယ်များကို အလုပ်လုပ်စေခြင်း

စွမ်းဆောင်ရည်လက္ခဏာများကို နားလည်ရန် လူကြိုက်များပြီး စမ်းသပ်ပြီးသား မော်ဒယ်များဖြင့် စတင်ပါ။

```cmd
REM Run Phi-4-Mini (lightweight, fast)
foundry model run phi-4-mini --verbose

REM Run Qwen 2.5 7B (larger, more capable)
foundry model run qwen2.5-7b --verbose

REM Run DeepSeek (specialized for coding)
foundry model run deepseek-r1-7b --verbose
```

**မှတ်ချက်:** `--verbose` flag သည် စတင်အလုပ်လုပ်စဉ်အတွင်း အောက်ပါအချက်အလက်များကို အသေးစိတ်ပေးသည်။
- မော်ဒယ်ဒေါင်းလုဒ်တိုးတက်မှု (ပထမဆုံးအကြိမ်တွင်)
- မှတ်ဉာဏ်ခွဲဝေမှုအသေးစိတ်
- ဝန်ဆောင်မှုချိတ်ဆက်မှုအချက်အလက်
- စွမ်းဆောင်ရည်စတင်မှု metrics

### မော်ဒယ်သိုလှောင်မှုအရေးပါချက်များ

**မော်ဒယ်အရွယ်အစားများ:**
- `phi-4-mini`: ~2.5 GB
- `qwen2.5-7b`: ~4.1 GB  
- `deepseek-r1-7b`: ~4.3 GB
- `llama-3.2`: ~4.9 GB
- `qwen2.5-14b`: ~8.2 GB

**သိုလှောင်မှုအကောင်းဆုံးအလေ့အကျင့်များ:**
- မြန်ဆန်စွာ ပြောင်းလဲနိုင်ရန် မော်ဒယ် ၂-၃ ခုကို cache ထားပါ
- အသုံးမပြုသော မော်ဒယ်များကို ဖယ်ရှားရန်: `foundry cache clean`
- သိုလှောင်မှုနေရာကို စောင့်ကြည့်ပါ၊ အထူးသဖြင့် အသေးစား SSD များတွင်
- မော်ဒယ်အရွယ်အစားနှင့် စွမ်းရည်အကြား trade-offs ကို စဉ်းစားပါ

### မော်ဒယ်စွမ်းဆောင်ရည်ကို စောင့်ကြည့်ခြင်း

## အပိုင်း ၅: ရိုးရှင်းသော မော်ဒယ် Benchmarking

### အခြေခံစွမ်းဆောင်ရည်စမ်းသပ်ခြင်း

မော်ဒယ်စွမ်းဆောင်ရည်ကို နှိုင်းယှဉ်ရန် ရိုးရှင်းသော နည်းလမ်းဖြစ်သည်။

```python
# simple_bench.py - Based on Sample 03 patterns
import time
import requests
import json

def test_model_response(model_name, prompt="Explain edge AI in one sentence."):
    """Test a single model with a prompt and measure response time."""
    start_time = time.time()
    
    try:
        response = requests.post(
            "http://localhost:8000/v1/chat/completions",
            headers={"Content-Type": "application/json"},
            json={
                "model": model_name,
                "messages": [{"role": "user", "content": prompt}],
                "max_tokens": 64
            },
            timeout=30
        )
        
        elapsed = time.time() - start_time
        
        if response.status_code == 200:
            result = response.json()
            return {
                "model": model_name,
                "latency_sec": round(elapsed, 3),
                "response": result["choices"][0]["message"]["content"],
                "status": "success"
            }
        else:
            return {
                "model": model_name,
                "status": "error",
                "error": f"HTTP {response.status_code}"
            }
            
    except Exception as e:
        return {
            "model": model_name,
            "status": "error", 
            "error": str(e)
        }

# Test the currently running model
if __name__ == "__main__":
    # Test with different models (start each model first)
    test_models = ["phi-4-mini", "qwen2.5-7b", "deepseek-r1-7b"]
    
    print("Model Performance Test")
    print("=" * 50)
    
    for model in test_models:
        print(f"\nTesting {model}...")
        print("Note: Make sure this model is running first with 'foundry model run {model}'")
        
        result = test_model_response(model)
        
        if result["status"] == "success":
            print(f"✅ {model}: {result['latency_sec']}s")
            print(f"   Response: {result['response'][:100]}...")
        else:
            print(f"❌ {model}: {result['error']}")
```
