# အခန်း ၃: အခမဲ့ မော်ဒယ် ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်း

## အကျဉ်းချုပ်

ဒီအခန်းမှာ Foundry Local ကို အသုံးပြုပြီး မော်ဒယ် ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်းကို လက်တွေ့ကျကျ လေ့လာပါမည်။ သင့်အတွက် သင့်လျော်သော မော်ဒယ်များကို ရွေးချယ်နိုင်ရန် Foundry CLI ကို အသုံးပြု၍ မော်ဒယ်များကို စစ်ဆေးခြင်း၊ ရွေးချယ်ခြင်းနှင့် အခြေခံ စွမ်းဆောင်ရည် လက္ခဏာများကို နားလည်ခြင်းတို့ကို လေ့လာပါမည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

- မော်ဒယ် ရှာဖွေခြင်းနှင့် စီမံခန့်ခွဲခြင်းအတွက် Foundry CLI အမိန့်များကို ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ရန်
- မော်ဒယ် cache နှင့် local storage ပုံစံများကို နားလည်ရန်
- မော်ဒယ်များကို အလျင်အမြန် စစ်ဆေးပြီး နှိုင်းယှဉ်နိုင်ရန်
- မော်ဒယ် ရွေးချယ်ခြင်းနှင့် စွမ်းဆောင်ရည် စမ်းသပ်မှုအတွက် လက်တွေ့ workflow များကို တည်ဆောက်ရန်
- Foundry Local မှတဆင့် ရရှိနိုင်သော မော်ဒယ်များ၏ တိုးတက်လာသော ecosystem ကို လေ့လာရန်

## ကြိုတင်လိုအပ်ချက်များ

- အခန်း ၁: Foundry Local ကို စတင်အသုံးပြုခြင်းကို ပြီးစီးထားရမည်
- Foundry Local CLI ကို ထည့်သွင်းပြီး အသုံးပြုနိုင်ရမည်
- မော်ဒယ်များကို ဒေါင်းလုဒ်လုပ်ရန် လိုအပ်သော storage နေရာ (မော်ဒယ်များသည် ၁GB မှ ၂၀GB+ အထိရှိနိုင်သည်)
- မော်ဒယ်အမျိုးအစားများနှင့် အသုံးပြုမှုများကို အခြေခံအားဖြင့် နားလည်ထားရမည်

## အခန်း ၆: လက်တွေ့ လေ့ကျင့်မှု

### လေ့ကျင့်မှု: မော်ဒယ် ရှာဖွေခြင်းနှင့် နှိုင်းယှဉ်ခြင်း

Sample 03 ကို အခြေခံပြီး သင့်ကိုယ်ပိုင် မော်ဒယ် အကဲဖြတ် script တစ်ခု ဖန်တီးပါ:

```cmd
REM create_model_test.cmd
@echo off
echo Model Discovery and Testing Script
echo =====================================

echo.
echo Step 1: List available models
foundry model list

echo.
echo Step 2: Check what's cached
foundry cache list

echo.
echo Step 3: Start phi-4-mini for testing
foundry model run phi-4-mini --verbose

echo.
echo Step 4: Test with a simple prompt
curl -X POST http://localhost:8000/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"phi-4-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"Hello, please introduce yourself.\"}],\"max_tokens\":100}"

echo.
echo Model test complete!
```

### သင့်လုပ်ဆောင်ရမည့်အရာ

1. **Sample 03 script ကို run လုပ်ပါ**: `samples\03\list_and_bench.cmd`
2. **မော်ဒယ်များကို စမ်းသပ်ပါ**: မော်ဒယ် ၃ ခုအနည်းဆုံး စမ်းသပ်ပါ
3. **စွမ်းဆောင်ရည်ကို နှိုင်းယှဉ်ပါ**: အမြန်နှုန်းနှင့် တုံ့ပြန်မှု အရည်အသွေးကွာခြားချက်များကို မှတ်သားပါ
4. **ရလဒ်များကို မှတ်တမ်းတင်ပါ**: ရိုးရှင်းသော နှိုင်းယှဉ်ဇယားတစ်ခု ဖန်တီးပါ

### နှိုင်းယှဉ်မှု ပုံစံ ဥပမာ

```
Model Comparison Results:
========================
phi-4-mini:        Fast (~2s), good for general chat
qwen2.5-7b:       Slower (~5s), better reasoning  
deepseek-r1:      Medium (~3s), excellent for code

Recommendation: Start with phi-4-mini for development, 
switch to qwen2.5-7b for production reasoning tasks.
```

## အခန်း ၇: ပြဿနာများ ဖြေရှင်းခြင်းနှင့် အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းများ

### ရှိနေသော ပြဿနာများနှင့် ဖြေရှင်းနည်းများ

**မော်ဒယ် မစတင်နိုင်ခြင်း:**
```cmd
REM Check service status
foundry service status

REM Restart service if needed
foundry service stop
foundry service start

REM Try with verbose output
foundry model run phi-4-mini --verbose
```

**မှတ်ဉာဏ် မလုံလောက်ခြင်း:**
- သေးငယ်သော မော်ဒယ်များ (`phi-4-mini`) ဖြင့် စတင်ပါ
- အခြား application များကို ပိတ်ပါ
- မှတ်ဉာဏ်ကို frequently limit ကို ရောက်လျှင် RAM ကို upgrade လုပ်ပါ

**စွမ်းဆောင်ရည် နှေးကွေးခြင်း:**
- မော်ဒယ်ကို အပြည့်အဝ load လုပ်ထားသည်ကို သေချာပါ (verbose output ကို စစ်ဆေးပါ)
- မလိုအပ်သော background application များကို ပိတ်ပါ
- ပိုမိုမြန်ဆန်သော storage (SSD) ကို စဉ်းစားပါ

### အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းများ

1. **သေးငယ်သော မော်ဒယ်များဖြင့် စတင်ပါ**: `phi-4-mini` ဖြင့် setup ကို အတည်ပြုပါ
2. **တစ်ခါတစ်မော်ဒယ်သာ အသုံးပြုပါ**: မော်ဒယ်အသစ်ကို စတင်မပြုမီ မော်ဒယ်ဟောင်းကို ပိတ်ပါ
3. **အရင်းအမြစ်များကို စောင့်ကြည့်ပါ**: memory usage ကို စောင့်ကြည့်ပါ
4. **တူညီသော prompt များကို အသုံးပြု၍ စမ်းသပ်ပါ**: နှိုင်းယှဉ်မှု တရားမျှတစေရန်
5. **ရလဒ်များကို မှတ်တမ်းတင်ပါ**: သင့်အသုံးပြုမှုအတွက် မော်ဒယ် performance ကို မှတ်တမ်းတင်ပါ

## အခန်း ၈: နောက်တစ်ဆင့်များနှင့် ရင်းမြစ်များ

### အခန်း ၄ အတွက် ပြင်ဆင်ခြင်း

- **အခန်း ၄ အဓိကအကြောင်းအရာ**: Optimization tools နှင့် နည်းလမ်းများ
- **ကြိုတင်လိုအပ်ချက်များ**: မော်ဒယ်များကို ပြောင်းလဲခြင်းနှင့် အခြေခံ စွမ်းဆောင်ရည် စမ်းသပ်မှုများကို ကျွမ်းကျင်စွာ အသုံးပြုနိုင်ရမည်
- **အကြံပြုချက်**: ဒီအခန်းမှ မော်ဒယ် ၂-၃ ခုကို ရွေးချယ်ထားရန်

### အဓိက အကျဉ်းချုပ်

✅ **မော်ဒယ် ရှာဖွေခြင်း**: `foundry model list` ကို အသုံးပြု၍ ရရှိနိုင်သော မော်ဒယ်များကို ရှာဖွေပါ  
✅ **အလျင်အမြန် စမ်းသပ်ခြင်း**: `list_and_bench.cmd` pattern ကို အသုံးပြု၍ စမ်းသပ်ပါ  
✅ **စွမ်းဆောင်ရည် စောင့်ကြည့်ခြင်း**: အခြေခံ resource usage နှင့် response time ကို တိုင်းတာပါ  
✅ **မော်ဒယ် ရွေးချယ်ခြင်း**: အသုံးပြုမှုအလိုက် မော်ဒယ်များကို ရွေးချယ်ရန် လက်တွေ့ လမ်းညွှန်ချက်များ  
✅ **Cache စီမံခန့်ခွဲမှု**: storage နှင့် cleanup လုပ်ငန်းစဉ်များကို နားလည်ပါ  

Foundry Local ရဲ့ ရိုးရှင်းတဲ့ CLI နည်းလမ်းကို အသုံးပြု၍ သင့် AI application များအတွက် သင့်လျော်သော မော်ဒယ်များကို ရှာဖွေ၊ စမ်းသပ်၊ ရွေးချယ်နိုင်ရန် လက်တွေ့ကျကျ ကျွမ်းကျင်မှု ရရှိထားပါပြီ။
