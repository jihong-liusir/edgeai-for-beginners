# جلسه ۴: مدل‌های پیشرفته – LLMها، SLMها و استنتاج روی دستگاه

## مرور کلی

مقایسه LLMها و SLMها، ارزیابی مزایا و معایب استنتاج محلی در مقابل ابری، و پیاده‌سازی دموهایی که سناریوهای EdgeAI را با استفاده از Phi و ONNX Runtime نمایش می‌دهند. همچنین به Chainlit RAG، گزینه‌های استنتاج WebGPU و یکپارچه‌سازی Open WebUI اشاره خواهیم کرد.

منابع:
- مستندات Foundry Local: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- راهنمای Open WebUI (اپلیکیشن چت با Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## اهداف آموزشی
- درک مزایا و معایب LLMها در مقابل SLMها از نظر هزینه، تأخیر و دقت
- انتخاب بین استنتاج محلی و ابری برای نیازهای خاص کسب‌وکار
- پیاده‌سازی یک دمو کوچک RAG با Chainlit
- بررسی WebGPU برای تسریع در مرورگر
- اتصال Open WebUI به Foundry Local

## بخش ۱: LLM در مقابل SLM – ماتریس تصمیم‌گیری

موارد قابل توجه:
- تأخیر: SLMها روی دستگاه معمولاً پاسخ‌های زیر یک ثانیه ارائه می‌دهند
- هزینه: استنتاج محلی هزینه‌های ابری را کاهش می‌دهد
- حریم خصوصی: داده‌های حساس روی دستگاه باقی می‌مانند
- قابلیت: LLMها ممکن است در وظایف پیچیده عملکرد بهتری داشته باشند
- قابلیت اطمینان: استراتژی‌های ترکیبی خطر خرابی را کاهش می‌دهند

## بخش ۲: محلی در مقابل ابری – الگوهای ترکیبی

- اولویت محلی با استفاده از ابری برای درخواست‌های بزرگ/پیچیده
- اولویت ابری با استفاده از محلی برای سناریوهای حساس به حریم خصوصی یا آفلاین
- مسیریابی بر اساس نوع وظیفه (تولید کد به DeepSeek، چت عمومی به Phi/Qwen)

## بخش ۳: اپلیکیشن چت RAG با Chainlit (حداقلی)

نصب وابستگی‌ها:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

اجرا:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

گسترش: یک بازیاب ساده (فایل‌های محلی) اضافه کنید و متن بازیابی‌شده را به درخواست کاربر اضافه کنید.

## بخش ۴: استنتاج WebGPU (نکات مهم)

مدل‌های کوچک را مستقیماً در مرورگر با استفاده از WebGPU اجرا کنید. این گزینه برای دموهای اولویت‌دار حریم خصوصی و تجربه‌های بدون نصب مناسب است. در ادامه یک مثال حداقلی و گام‌به‌گام با استفاده از ONNX Runtime Web و ارائه‌دهنده اجرای WebGPU آورده شده است.

۱) بررسی پشتیبانی WebGPU
- مرورگرهای Chromium: chrome://gpu → تأیید کنید که “WebGPU” فعال است
- بررسی برنامه‌نویسی (همچنین در کد بررسی خواهیم کرد): `if (!('gpu' in navigator)) { /* no WebGPU */ }`

۲) ایجاد یک پروژه حداقلی
یک پوشه و دو فایل ایجاد کنید: `index.html` و `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

۳) سرویس‌دهی محلی (Windows cmd.exe)
از یک سرور استاتیک ساده استفاده کنید تا مرورگر بتواند مدل را دریافت کند.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

آدرس http://localhost:5173 را در مرورگر خود باز کنید. باید لاگ‌های اولیه، ایجاد جلسه با WebGPU و پیش‌بینی argmax را مشاهده کنید.

۴) رفع اشکال
- اگر WebGPU در دسترس نیست: Chrome/Edge را به‌روزرسانی کنید و مطمئن شوید که درایورهای GPU به‌روز هستند، سپس chrome://flags را برای “Enable WebGPU” بررسی کنید.
- اگر خطاهای CORS یا fetch رخ داد: مطمئن شوید که فایل‌ها را از طریق http:// (نه file://) سرویس‌دهی می‌کنید و URL مدل اجازه درخواست‌های cross-origin را می‌دهد.
- بازگشت به CPU: `executionProviders: ['wasm']` را تغییر دهید تا رفتار پایه را بررسی کنید.

۵) مراحل بعدی
- یک مدل ONNX خاص دامنه (مانند طبقه‌بندی تصویر یا یک مدل متن کوچک) جایگزین کنید.
- منطق پیش‌پردازش/پس‌پردازش را برای ورودی‌های واقعی اضافه کنید.
- برای مدل‌های بزرگ‌تر یا تأخیر تولید، Foundry Local یا ONNX Runtime Server را ترجیح دهید.

## بخش ۵: Open WebUI + Foundry Local (گام‌به‌گام)

این بخش Open WebUI را به نقطه پایانی سازگار با OpenAI Foundry Local برای یک رابط چت محلی متصل می‌کند.

۱) پیش‌نیازها
- Foundry Local نصب و فعال (`foundry --version`)
- یک مدل آماده اجرا به صورت محلی (مانند `phi-4-mini`)
- Docker Desktop نصب شده (توصیه‌شده برای Open WebUI)

۲) اجرای یک مدل با Foundry Local
```powershell
foundry model run phi-4-mini
```
این یک API سازگار با OpenAI را در `http://localhost:8000` فراهم می‌کند.

۳) اجرای Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
نکات:
- در ویندوز، `host.docker.internal` به کانتینر اجازه می‌دهد به میزبان شما در `localhost` دسترسی پیدا کند.
- ما `OPENAI_API_BASE_URL` را به نقطه پایانی Foundry Local و یک `OPENAI_API_KEY` جعلی تنظیم می‌کنیم.

۴) پیکربندی از رابط Open WebUI (جایگزین)
- به آدرس http://localhost:3000 بروید
- تنظیمات اولیه (کاربر مدیر) را کامل کنید
- به Settings → Models/Providers بروید
- تنظیم Base URL: `http://host.docker.internal:8000/v1`
- تنظیم API Key: `local-key` (جایگزین)
- ذخیره کنید

۵) اجرای یک درخواست آزمایشی
- در چت Open WebUI، نام مدل `phi-4-mini` را انتخاب یا وارد کنید
- درخواست: “پنج مزیت استنتاج هوش مصنوعی روی دستگاه را فهرست کنید.”
- باید پاسخ را که از مدل محلی شما پخش می‌شود مشاهده کنید

۶) رفع اشکال
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

۷) اختیاری: ذخیره داده‌های Open WebUI
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
