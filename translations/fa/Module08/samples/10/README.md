# ادغام Foundry Local به عنوان ابزارها

یک چارچوب جامع برای ادغام Microsoft Foundry Local به عنوان ابزارهای قابل فراخوانی در برنامه‌های بزرگ‌تر، مطابق با الگوهای رسمی مایکروسافت برای ادغام هوش مصنوعی مبتنی بر ابزار.

## مرور کلی

این نمونه نشان می‌دهد که چگونه می‌توان مدل‌های Foundry Local را به عنوان ابزارهای قابل استفاده مجدد ارائه کرد که می‌توانند در برنامه‌ها، جریان‌های کاری و محیط‌های توسعه موجود ادغام شوند. این نمونه الگوهای پیشنهادی مایکروسافت برای ادغام ابزارها و فراخوانی توابع را به نمایش می‌گذارد.

## مفاهیم کلیدی

### 🔧 **معماری مبتنی بر ابزار**
- مدل‌های Foundry Local به عنوان توابع قابل فراخوانی
- رابط‌ها و طرح‌های استاندارد ابزار
- ادغام بی‌دردسر با کدهای موجود
- تعریف و اعتبارسنجی ابزارهای نوع-ایمن

### ⚡ **الگوهای فراخوانی توابع**
- پیاده‌سازی فراخوانی توابع Microsoft Foundry Local
- تعریف ابزارهای سازگار با OpenAI
- اعتبارسنجی و تبدیل خودکار پارامترها
- مدیریت خطا و قالب‌بندی پاسخ‌ها

### 🔌 **چارچوب‌های ادغام**
- **ادغام LangChain**: پشتیبانی ابزار بومی LangChain
- **Semantic Kernel**: توابع Microsoft Semantic Kernel
- **REST API**: نقاط پایانی ابزار مبتنی بر HTTP
- **ابزارهای CLI**: ادغام رابط خط فرمان
- **دفترچه‌های Jupyter**: ابزارهای توسعه تعاملی

### 🎯 **الگوهای موارد استفاده**
- ابزارهای تحلیل و تولید کد
- پردازش و خلاصه‌سازی محتوا

        return analysis_result

### 3. ابزارهای جریان‌سازی
```python
from foundry_tools import StreamingFoundryTool
from typing import AsyncGenerator

@StreamingFoundryTool(
    name="code_generator",
    description="Generate code with real-time streaming",
    model="qwen2.5-coder-0.5b"
)
async def generate_code(
    specification: str,
    language: str = "python",
    include_tests: bool = False
) -> AsyncGenerator[Dict[str, Any], None]:
    """Generate code with streaming responses."""
    
    # Yield metadata first
    yield {
        "type": "metadata",
        "language": language,
        "include_tests": include_tests,
        "estimated_lines": 50
    }
    
    # Stream code generation
    async for chunk in foundry_stream({
        "prompt": f"Generate {language} code: {specification}",
        "stream": True
    }):
        yield {
            "type": "code_chunk",
            "content": chunk.content,
            "complete": chunk.finish_reason is not None
        }
    
    # Yield final result
    if include_tests:
        async for test_chunk in foundry_stream({
            "prompt": f"Generate unit tests for the above {language} code",
            "stream": True
        }):
            yield {
                "type": "test_chunk", 
                "content": test_chunk.content,
                "complete": test_chunk.finish_reason is not None
            }
```

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI(title="Foundry Local Tools API")

foundry_framework = FoundryRESTFramework()

class AnalysisRequest(BaseModel):
    code: str
    language: str = "python"

@app.post("/api/v1/analyze")
async def analyze_code_endpoint(request: AnalysisRequest):
    try:
        result = await foundry_framework.call_tool(
            "code_analyzer",
            code=request.code,
            language=request.language
        )
        return {"success": True, "analysis": result}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

        "active_models": status.loaded_models,

### ادغام خط فرمان
```python
import typer
from rich.console import Console
from rich.table import Table
from foundry_tools.cli import FoundryCLI

app = typer.Typer(name="foundry-tools")
console = Console()
foundry_cli = FoundryCLI()

@app.command()
async def analyze(
    file_path: str = typer.Argument(..., help="Path to code file"),
    language: str = typer.Option("python", help="Programming language"),
    output: str = typer.Option("table", help="Output format (table, json, yaml)")
):
    """Analyze code file using Foundry Local."""
    
    try:
        with open(file_path, 'r') as f:
            code = f.read()
        
        result = await foundry_cli.call_tool(
            "code_analyzer",
            code=code,
            language=language
        )
        
        if output == "table":
            table = Table(title=f"Code Analysis: {file_path}")
            table.add_column("Metric", style="cyan")
            table.add_column("Value", style="magenta")
            
            for key, value in result.items():
                table.add_row(key, str(value))
            
            console.print(table)
        
        elif output == "json":
            console.print_json(data=result)
        
        else:
            console.print(result)
            
    except Exception as e:
        console.print(f"[red]Error: {e}[/red]")
        raise typer.Exit(1)

@app.command()
async def list_tools():
    """List all available Foundry tools."""
    
    tools = await foundry_cli.list_available_tools()
    
    table = Table(title="Available Foundry Tools")
    table.add_column("Name", style="cyan")
    table.add_column("Description", style="white")
    table.add_column("Model", style="yellow")
    
    for tool in tools:
        table.add_row(
            tool["name"],
            tool["description"][:50] + "..." if len(tool["description"]) > 50 else tool["description"],
            tool["model"]
        )
    
    console.print(table)

if __name__ == "__main__":
    app()
```

## نتایج یادگیری

پس از تکمیل این نمونه، شما موارد زیر را درک خواهید کرد:

1. **الگوهای ادغام ابزار**
   - طراحی ابزارهای مبتنی بر توابع و کلاس‌ها
   - الگوهای ادغام Microsoft Foundry Local
   - اعتبارسنجی طرح و ایمنی نوع
   - مدیریت خطا و بازیابی

2. **ادغام چارچوب‌ها**
   - توسعه ابزارهای LangChain
   - ادغام توابع Semantic Kernel
   - ادغام چارچوب REST API
   - توسعه برنامه‌های CLI

3. **ملاحظات تولید**
   - استراتژی‌های بهینه‌سازی عملکرد
   - مدیریت کشینگ و منابع
   - نظارت و مشاهده‌پذیری
   - امنیت و اعتبارسنجی

4. **الگوهای پیشرفته ابزار**
   - ترکیب و زنجیره‌سازی ابزارها
   - پردازش آگاه به زمینه
   - عملیات دسته‌ای و جریان‌سازی
   - توسعه ادغام‌های سفارشی

## مراحل بعدی

- **پروژه‌های ادغام**: ادغام‌های سفارشی با چارچوب‌های مورد علاقه خود ایجاد کنید
- **توسعه ابزار**: ابزارهای تخصصی برای حوزه خود ایجاد کنید
- **تنظیم عملکرد**: برای موارد استفاده خاص خود بهینه‌سازی کنید
- **استقرار تولید**: ابزارها را برای استفاده در سطح سازمانی مقیاس دهید
