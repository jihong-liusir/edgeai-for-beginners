## مقدمه

به راهنمای مطالعه EdgeAI برای مبتدیان خوش آمدید! این سند طراحی شده است تا به شما کمک کند مواد دوره را به طور مؤثر مرور کنید و تجربه یادگیری خود را به حداکثر برسانید. این راهنما مسیرهای یادگیری ساختاریافته، برنامه‌های پیشنهادی مطالعه، خلاصه مفاهیم کلیدی و منابع تکمیلی را برای درک عمیق‌تر فناوری‌های EdgeAI ارائه می‌دهد.

این دوره مختصر 20 ساعته دانش ضروری درباره EdgeAI را در قالبی کارآمد ارائه می‌دهد و آن را برای حرفه‌ای‌های پرمشغله و دانشجویانی که می‌خواهند به سرعت مهارت‌های عملی در این حوزه نوظهور کسب کنند، ایده‌آل می‌سازد.

## نمای کلی دوره

این دوره به هفت ماژول جامع تقسیم شده است:

1. **مبانی و تحول EdgeAI** - درک مفاهیم اصلی و تغییرات فناوری
2. **مبانی مدل‌های کوچک زبانی (SLM)** - بررسی خانواده‌های مختلف SLM و معماری‌های آن‌ها
3. **استقرار مدل‌های کوچک زبانی (SLM)** - پیاده‌سازی استراتژی‌های عملی استقرار
4. **تبدیل فرمت مدل و کوانتیزاسیون** - بهینه‌سازی پیشرفته با 6 چارچوب از جمله OpenVINO
5. **SLMOps - عملیات مدل‌های کوچک زبانی** - مدیریت چرخه عمر تولید و استقرار
6. **سیستم‌های عامل SLM** - عوامل هوش مصنوعی، فراخوانی توابع و پروتکل زمینه مدل
7. **نمونه‌های پیاده‌سازی EdgeAI** - ابزارهای هوش مصنوعی، توسعه در ویندوز و پیاده‌سازی‌های خاص پلتفرم

## ماژول 1: مبانی و تحول EdgeAI

### اهداف کلیدی یادگیری

- درک تفاوت‌های بین هوش مصنوعی مبتنی بر ابر و مبتنی بر لبه
- تسلط بر تکنیک‌های بهینه‌سازی اصلی برای محیط‌های با منابع محدود
- تحلیل کاربردهای واقعی فناوری‌های EdgeAI
- راه‌اندازی محیط توسعه برای پروژه‌های EdgeAI

### حوزه‌های تمرکز مطالعه

#### بخش 1: مبانی EdgeAI
- **مفاهیم اولویت‌دار**: 
  - پارادایم‌های محاسبات لبه در مقابل ابر
  - تکنیک‌های کوانتیزاسیون مدل
  - گزینه‌های شتاب‌دهی سخت‌افزاری (NPUها، GPUها، CPUها)
  - مزایای حریم خصوصی و امنیت

- **مواد تکمیلی**:
  - [مستندات TensorFlow Lite](https://www.tensorflow.org/lite)
  - [گیت‌هاب ONNX Runtime](https://github.com/microsoft/onnxruntime)
  - [مستندات Edge Impulse](https://docs.edgeimpulse.com)

#### بخش 2: مطالعات موردی واقعی
- **مفاهیم اولویت‌دار**: 
  - اکوسیستم مدل Microsoft Phi & Mu
  - پیاده‌سازی‌های عملی در صنایع مختلف
  - ملاحظات استقرار

#### بخش 3: راهنمای پیاده‌سازی عملی
- **مفاهیم اولویت‌دار**: 
  - راه‌اندازی محیط توسعه
  - ابزارهای کوانتیزاسیون و بهینه‌سازی
  - روش‌های ارزیابی برای پیاده‌سازی‌های EdgeAI

#### بخش 4: سخت‌افزار استقرار لبه
- **مفاهیم اولویت‌دار**: 
  - مقایسه پلتفرم‌های سخت‌افزاری
  - استراتژی‌های بهینه‌سازی برای سخت‌افزار خاص
  - ملاحظات استقرار

### سوالات خودارزیابی

1. تفاوت‌های بین هوش مصنوعی مبتنی بر ابر و هوش مصنوعی مبتنی بر لبه را مقایسه کنید.
2. سه تکنیک کلیدی برای بهینه‌سازی مدل‌ها برای استقرار لبه را توضیح دهید.
3. مزایای اصلی اجرای مدل‌های هوش مصنوعی در لبه چیست؟
4. فرآیند کوانتیزاسیون یک مدل را توضیح دهید و تأثیر آن بر عملکرد را شرح دهید.
5. توضیح دهید که چگونه شتاب‌دهنده‌های سخت‌افزاری مختلف (NPUها، GPUها، CPUها) بر استقرار EdgeAI تأثیر می‌گذارند.

### تمرین‌های عملی

1. **راه‌اندازی سریع محیط**: یک محیط توسعه حداقلی با بسته‌های ضروری پیکربندی کنید (30 دقیقه)
2. **بررسی مدل**: یک مدل کوچک زبانی از پیش‌آموزش‌دیده را دانلود و بررسی کنید (1 ساعت)
3. **کوانتیزاسیون پایه**: کوانتیزاسیون ساده را روی یک مدل کوچک امتحان کنید (1 ساعت)
- درخت‌های تصمیم‌گیری برای انتخاب چارچوب  
- اعتبارسنجی آمادگی برای تولید  
- استراتژی‌های آینده‌نگری  

### سوالات خودارزیابی  

1. استراتژی‌های کوانتایزیشن را در سطوح دقت مختلف (از 1 بیت تا 8 بیت) مقایسه کنید.  
2. مزایای فرمت GGUF برای استقرار در لبه چیست؟  
3. بهینه‌سازی سخت‌افزارمحور در Microsoft Olive چگونه کارایی استقرار را بهبود می‌بخشد؟  
4. مزایای کلیدی NNCF در OpenVINO برای فشرده‌سازی مدل چیست؟  
5. توضیح دهید که Apple MLX چگونه از معماری حافظه یکپارچه برای بهینه‌سازی استفاده می‌کند.  
6. سنتز جریان کاری چگونه به انتخاب چارچوب‌های بهینه‌سازی کمک می‌کند؟  

### تمرین‌های عملی  

1. **کوانتایزیشن مدل**: سطوح مختلف کوانتایزیشن را روی یک مدل اعمال کرده و نتایج را مقایسه کنید (1 ساعت)  
2. **بهینه‌سازی OpenVINO**: از NNCF برای فشرده‌سازی یک مدل برای سخت‌افزار Intel استفاده کنید (1 ساعت)  
3. **مقایسه چارچوب‌ها**: یک مدل را در سه چارچوب بهینه‌سازی مختلف آزمایش کنید (1 ساعت)  
4. **بنچمارک عملکرد**: تأثیر بهینه‌سازی بر سرعت استنتاج و استفاده از حافظه را اندازه‌گیری کنید (1 ساعت)  

## ماژول ۵: SLMOps - عملیات مدل‌های زبانی کوچک  

### اهداف کلیدی یادگیری  

- اصول مدیریت چرخه عمر SLMOps را درک کنید  
- تکنیک‌های تقطیر و تنظیم دقیق برای استقرار در لبه را بیاموزید  
- استراتژی‌های استقرار تولید با نظارت را پیاده‌سازی کنید  
- جریان‌های کاری عملیاتی و نگهداری SLM در سطح سازمانی ایجاد کنید  

### حوزه‌های تمرکز مطالعه  

#### بخش ۱: مقدمه‌ای بر SLMOps  
- **مفاهیم اولویت‌دار**:  
  - تغییر پارادایم SLMOps در عملیات هوش مصنوعی  
  - معماری مقرون‌به‌صرفه و اولویت‌دار برای حریم خصوصی  
  - تأثیر استراتژیک بر کسب‌وکار و مزایای رقابتی  

#### بخش ۲: تقطیر مدل  
- **مفاهیم اولویت‌دار**:  
  - تکنیک‌های انتقال دانش  
  - پیاده‌سازی فرآیند تقطیر دو مرحله‌ای  
  - جریان‌های کاری تقطیر در Azure ML  

#### بخش ۳: استراتژی‌های تنظیم دقیق  
- **مفاهیم اولویت‌دار**:  
  - تنظیم دقیق کارآمد پارامترها (PEFT)  
  - روش‌های پیشرفته LoRA و QLoRA  
  - آموزش چند آداپتر و بهینه‌سازی ابرپارامترها  

#### بخش ۴: استقرار تولید  
- **مفاهیم اولویت‌دار**:  
  - تبدیل و کوانتایزیشن مدل برای تولید  
  - پیکربندی استقرار Foundry Local  
  - بنچمارک عملکرد و اعتبارسنجی کیفیت  

### سوالات خودارزیابی  

1. SLMOps چه تفاوتی با MLOps سنتی دارد؟  
2. مزایای تقطیر مدل برای استقرار در لبه چیست؟  
3. ملاحظات کلیدی برای تنظیم دقیق SLM‌ها در محیط‌های با منابع محدود چیست؟  
4. یک خط لوله کامل استقرار تولید برای برنامه‌های هوش مصنوعی لبه را توصیف کنید.  

### تمرین‌های عملی  

1. **تقطیر پایه‌ای**: یک مدل کوچک‌تر از یک مدل معلم بزرگ‌تر ایجاد کنید (1 ساعت)  
2. **آزمایش تنظیم دقیق**: یک مدل را برای یک حوزه خاص تنظیم دقیق کنید (1 ساعت)  
3. **خط لوله استقرار**: یک خط لوله CI/CD پایه برای استقرار مدل تنظیم کنید (1 ساعت)  

## ماژول ۶: سیستم‌های عامل SLM - عوامل هوش مصنوعی و فراخوانی توابع  

### اهداف کلیدی یادگیری  

- عوامل هوش مصنوعی هوشمند برای محیط‌های لبه با استفاده از مدل‌های زبانی کوچک ایجاد کنید  
- قابلیت‌های فراخوانی توابع را با جریان‌های کاری سیستماتیک پیاده‌سازی کنید  
- ادغام پروتکل زمینه مدل (MCP) برای تعامل استاندارد ابزارها را بیاموزید  
- سیستم‌های عامل پیچیده با حداقل مداخله انسانی ایجاد کنید  

### حوزه‌های تمرکز مطالعه  

#### بخش ۱: عوامل هوش مصنوعی و مبانی SLM  
- **مفاهیم اولویت‌دار**:  
  - چارچوب طبقه‌بندی عوامل (بازتابی، مبتنی بر مدل، مبتنی بر هدف، عوامل یادگیرنده)  
  - تحلیل مزایا و معایب SLM در مقابل LLM  
  - الگوهای طراحی خاص لبه برای عوامل  
  - بهینه‌سازی منابع برای عوامل  

#### بخش ۲: فراخوانی توابع در مدل‌های زبانی کوچک  
- **مفاهیم اولویت‌دار**:  
  - پیاده‌سازی جریان کاری سیستماتیک (تشخیص نیت، خروجی JSON، اجرای خارجی)  
  - پیاده‌سازی‌های خاص پلتفرم (Phi-4-mini، مدل‌های منتخب Qwen، Microsoft Foundry Local)  
  - مثال‌های پیشرفته (همکاری چند عامل، انتخاب ابزار پویا)  
  - ملاحظات تولید (محدودیت نرخ، ثبت گزارش، اقدامات امنیتی)  

#### بخش ۳: ادغام پروتکل زمینه مدل (MCP)  
- **مفاهیم اولویت‌دار**:  
  - معماری پروتکل و طراحی سیستم لایه‌ای  
  - پشتیبانی چند بک‌اند (Ollama برای توسعه، vLLM برای تولید)  
  - پروتکل‌های اتصال (حالت‌های STDIO و SSE)  
  - کاربردهای دنیای واقعی (اتوماسیون وب، پردازش داده، ادغام API)  

### سوالات خودارزیابی  

1. ملاحظات کلیدی معماری برای عوامل هوش مصنوعی لبه چیست؟  
2. فراخوانی توابع چگونه قابلیت‌های عامل را افزایش می‌دهد؟  
3. نقش پروتکل زمینه مدل در ارتباطات عامل چیست؟  

### تمرین‌های عملی  

1. **عامل ساده**: یک عامل هوش مصنوعی پایه با قابلیت فراخوانی توابع ایجاد کنید (1 ساعت)  
2. **ادغام MCP**: MCP را در یک برنامه عامل پیاده‌سازی کنید (30 دقیقه)  

## ماژول ۷: نمونه‌های پیاده‌سازی EdgeAI

- پیاده‌سازی EdgeAI در چندین پلتفرم سخت‌افزاری و سناریوهای استقرار  
- ایجاد برنامه‌های EdgeAI آماده تولید با بهینه‌سازی‌های خاص پلتفرم  

### حوزه‌های تمرکز مطالعه  

#### بخش ۱: ابزار هوش مصنوعی برای Visual Studio Code  
- **مفاهیم اولویت‌دار**:  
  - محیط جامع توسعه Edge AI در VS Code  
  - کاتالوگ مدل و کشف برای استقرار لبه  
  - جریان‌های کاری آزمایش محلی، بهینه‌سازی و توسعه عامل  
  - نظارت بر عملکرد و ارزیابی برای سناریوهای لبه  

#### بخش ۲: راهنمای توسعه Windows EdgeAI  
- **مفاهیم اولویت‌دار**:  
  - مرور جامع پلتفرم Windows AI Foundry  
  - API سیلیکا فی برای استنتاج کارآمد NPU  
  - API‌های بینایی کامپیوتری برای پردازش تصویر و OCR  
  - CLI Foundry Local برای توسعه و آزمایش محلی  

#### بخش ۳: پیاده‌سازی‌های خاص پلتفرم  
- **مفاهیم اولویت‌دار**:  
  - استقرار NVIDIA Jetson Orin Nano (عملکرد AI 67 TOPS)  
  - برنامه‌های موبایل با .NET MAUI و ONNX Runtime GenAI  
  - راه‌حل‌های Azure EdgeAI با معماری ترکیبی ابر-لبه  
  - بهینه‌سازی Windows ML با پشتیبانی سخت‌افزاری جهانی  
  - برنامه‌های Foundry Local با پیاده‌سازی RAG متمرکز بر حریم خصوصی  

### سوالات خودارزیابی  

1. ابزار هوش مصنوعی چگونه جریان کاری توسعه EdgeAI را ساده می‌کند؟  
2. استراتژی‌های استقرار در پلتفرم‌های سخت‌افزاری مختلف را مقایسه کنید.  
3. مزایای Windows AI Foundry برای توسعه لبه چیست؟  
4. نقش بهینه‌سازی NPU در برنامه‌های مدرن EdgeAI چیست؟  
5. API سیلیکا فی چگونه از سخت‌افزار NPU برای بهینه‌سازی عملکرد استفاده می‌کند؟  
6. مزایای استقرار محلی در مقابل ابری برای برنامه‌های حساس به حریم خصوصی چیست؟  

### تمرین‌های عملی  

1. **راه‌اندازی ابزار هوش مصنوعی**: ابزار هوش مصنوعی را پیکربندی کرده و یک مدل را بهینه کنید (1 ساعت)  
2. **Windows AI Foundry**: یک برنامه ساده Windows AI با استفاده از API سیلیکا فی ایجاد کنید (1 ساعت)  
3. **استقرار چند پلتفرمی**: یک مدل را در دو پلتفرم مختلف مستقر کنید (1 ساعت)  
4. **بهینه‌سازی NPU**: عملکرد NPU را با ابزارهای Windows AI Foundry آزمایش کنید (30 دقیقه)
