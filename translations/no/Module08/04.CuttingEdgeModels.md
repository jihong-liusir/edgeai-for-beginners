# Sesjon 4: Banebrytende modeller – LLMs, SLMs og lokal inferens

## Oversikt

Sammenlign LLMs og SLMs, vurder fordeler og ulemper ved lokal vs skybasert inferens, og implementer demoer som viser EdgeAI-scenarier ved bruk av Phi og ONNX Runtime. Vi vil også fremheve Chainlit RAG, WebGPU-inferensalternativer og integrasjon med Open WebUI.

Referanser:
- Foundry Local-dokumentasjon: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Open WebUI veiledning (chat-app med Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Læringsmål
- Forstå fordeler og ulemper ved LLM vs SLM når det gjelder kostnad, latens og nøyaktighet
- Velge mellom lokal og skybasert inferens for spesifikke forretningsbehov
- Implementere en enkel RAG-demo med Chainlit
- Utforske WebGPU for akselerasjon i nettleseren
- Koble Open WebUI til Foundry Local

## Del 1: LLM vs SLM – Beslutningsmatrise

Vurder:
- Latens: SLMs på enheten gir ofte svar på under ett sekund
- Kostnad: lokal inferens reduserer skybaserte kostnader
- Personvern: sensitiv data forblir på enheten
- Kapasitet: LLMs kan overgå SLMs i komplekse oppgaver
- Pålitelighet: hybride strategier reduserer risiko for nedetid

## Del 2: Lokal vs sky – Hybride mønstre

- Lokal først med sky som fallback for store/komplekse forespørsler
- Sky først med lokal for personvernfølsomme eller offline-scenarier
- Ruting basert på oppgavetype (kodegenerering til DeepSeek, generell chat til Phi/Qwen)

## Del 3: RAG Chat-app med Chainlit (Minimal)

Installer avhengigheter:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Kjør:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Utvid: legg til en enkel søker (lokale filer) og legg til hentet kontekst foran brukerens forespørsel.

## Del 4: WebGPU-inferens (Forhåndsinfo)

Kjør små modeller direkte i nettleseren ved bruk av WebGPU. Dette er ideelt for personvernfokuserte demoer og opplevelser uten installasjon. Nedenfor er et minimalt, steg-for-steg eksempel ved bruk av ONNX Runtime Web med WebGPU som utførelsesleverandør.

1) Sjekk WebGPU-støtte
- Chromium-nettlesere: chrome://gpu → bekreft at “WebGPU” er aktivert
- Programmatisk sjekk (vi sjekker også i kode): `if (!('gpu' in navigator)) { /* ingen WebGPU */ }`

2) Lag et minimalt prosjekt
Lag en mappe og to filer: `index.html` og `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Server lokalt (Windows cmd.exe)
Bruk en enkel statisk server slik at nettleseren kan hente modellen.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Åpne http://localhost:5173 i nettleseren din. Du bør se initialiseringslogger, sesjonsopprettelse med WebGPU og en argmax-prediksjon.

4) Feilsøking
- Hvis WebGPU ikke er tilgjengelig: oppdater Chrome/Edge og sørg for at GPU-driverne er oppdaterte, sjekk deretter chrome://flags for “Enable WebGPU”.
- Hvis CORS- eller fetch-feil oppstår: sørg for at du serverer filer over http:// (ikke file://) og at modellens URL tillater kryss-opprinnelsesforespørsler.
- Fallback til CPU: endre `executionProviders: ['wasm']` for å verifisere grunnleggende oppførsel.

5) Neste steg
- Bytt til en domenespesifikk ONNX-modell (f.eks. bildeklassifisering eller en liten tekstmodell).
- Legg til logikk for forhånds- og etterbehandling av reelle input.
- For større modeller eller produksjonslatens, foretrekk Foundry Local eller ONNX Runtime Server.

## Del 5: Open WebUI + Foundry Local (Steg-for-steg)

Dette kobler Open WebUI til Foundry Locals OpenAI-kompatible endepunkt for en lokal chat-UI.

1) Forutsetninger
- Foundry Local installert og fungerer (`foundry --version`)
- En modell klar til å kjøre lokalt (f.eks. `phi-4-mini`)
- Docker Desktop installert (anbefalt for Open WebUI)

2) Start en modell med Foundry Local
```powershell
foundry model run phi-4-mini
```
Dette eksponerer en OpenAI-kompatibel API på `http://localhost:8000`.

3) Start Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Notater:
- På Windows lar `host.docker.internal` containeren nå din vert på `localhost`.
- Vi setter `OPENAI_API_BASE_URL` til Foundry Locals endepunkt og en dummy `OPENAI_API_KEY`.

4) Konfigurer fra Open WebUI UI (alternativ)
- Gå til http://localhost:3000
- Fullfør den innledende oppsettet (admin-bruker)
- Gå til Innstillinger → Modeller/Leverandører
- Sett Base URL: `http://host.docker.internal:8000/v1`
- Sett API-nøkkel: `local-key` (plassholder)
- Lagre

5) Kjør en testforespørsel
- I Open WebUI chat, velg eller skriv inn modellnavn `phi-4-mini`
- Forespørsel: “List opp fem fordeler med AI-inferens på enheten.”
- Du bør se et svar strømmet fra din lokale modell

6) Feilsøking
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Valgfritt: Bevar Open WebUI-data
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
