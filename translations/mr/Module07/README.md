# अध्याय ०७ : EdgeAI नमुने

Edge AI म्हणजे कृत्रिम बुद्धिमत्तेचे (AI) आणि एज संगणनाचे एकत्रीकरण, ज्यामुळे डिव्हाइसवरच बुद्धिमान प्रक्रिया करता येते, क्लाउड कनेक्टिव्हिटीवर अवलंबून न राहता. या अध्यायात विविध प्लॅटफॉर्म आणि फ्रेमवर्कवर आधारित पाच वेगवेगळ्या EdgeAI अंमलबजावणींचा अभ्यास केला आहे, ज्यामुळे एजवर AI मॉडेल्स चालवण्याची बहुपर्यायीता आणि सामर्थ्य दिसून येते.

## १. NVIDIA Jetson Orin Nano मध्ये EdgeAI

NVIDIA Jetson Orin Nano हे एज AI संगणनातील एक क्रांतिकारी साधन आहे, जे क्रेडिट कार्डाच्या आकाराच्या कॉम्पॅक्ट स्वरूपात ६७ TOPS पर्यंत AI कार्यक्षमता प्रदान करते. हे सामर्थ्यवान एज AI प्लॅटफॉर्म हौशी, विद्यार्थी आणि व्यावसायिक विकसकांसाठी जनरेटिव्ह AI विकास लोकशाहीसाठी उपलब्ध करते.

### मुख्य वैशिष्ट्ये
- ६७ TOPS पर्यंत AI कार्यक्षमता प्रदान करते—त्याच्या पूर्ववर्तीपेक्षा १.७ पट सुधारणा
- १०२४ CUDA कोर आणि ३२ Tensor कोर AI प्रक्रियेसाठी
- ६-कोर Arm Cortex-A78AE v8.2 ६४-बिट CPU, १.५ GHz ची जास्तीत जास्त वारंवारता
- फक्त $२४९ मध्ये उपलब्ध, विकसक, विद्यार्थी आणि निर्मात्यांसाठी सर्वात परवडणारे आणि प्रवेशयोग्य प्लॅटफॉर्म

### अनुप्रयोग
Jetson Orin Nano आधुनिक जनरेटिव्ह AI मॉडेल्स चालवण्यात उत्कृष्ट आहे, ज्यामध्ये व्हिजन ट्रान्सफॉर्मर्स, मोठे भाषा मॉडेल्स आणि व्हिजन-भाषा मॉडेल्स यांचा समावेश आहे. हे विशेषतः GenAI वापर प्रकरणांसाठी डिझाइन केले गेले आहे आणि आता तुम्ही अनेक LLMs एका हाताच्या डिव्हाइसवर चालवू शकता. लोकप्रिय वापर प्रकरणांमध्ये AI-सक्षम रोबोटिक्स, स्मार्ट ड्रोन, बुद्धिमान कॅमेरे आणि स्वायत्त एज डिव्हाइस यांचा समावेश आहे.

**अधिक जाणून घ्या**: [NVIDIA's Jetson Orin Nano SuperComputer: The Next Big Thing in EdgeAI](https://medium.com/data-science-in-your-pocket/nvidias-jetson-orin-nano-supercomputer-the-next-big-thing-in-edgeai-e9eff687ae62)

## २. .NET MAUI आणि ONNX Runtime GenAI सह मोबाइल अनुप्रयोगांमध्ये EdgeAI

हे समाधान .NET MAUI (मल्टी-प्लॅटफॉर्म अॅप UI) आणि ONNX Runtime GenAI वापरून क्रॉस-प्लॅटफॉर्म मोबाइल अनुप्रयोगांमध्ये जनरेटिव्ह AI आणि मोठ्या भाषा मॉडेल्स (LLMs) कसे समाकलित करावे हे दर्शवते. या दृष्टिकोनामुळे .NET विकसकांना Android आणि iOS डिव्हाइसवर स्थानिकपणे चालणारे प्रगत AI-सक्षम मोबाइल अनुप्रयोग तयार करता येतात.

### मुख्य वैशिष्ट्ये
- .NET MAUI फ्रेमवर्कवर आधारित, Android आणि iOS अनुप्रयोगांसाठी एकच कोडबेस प्रदान करते
- ONNX Runtime GenAI समाकलनामुळे जनरेटिव्ह AI मॉडेल्स थेट मोबाइल डिव्हाइसवर चालवता येतात
- CPU, GPU आणि विशेष मोबाइल AI प्रोसेसर यांसारख्या विविध हार्डवेअर प्रवेगकांना समर्थन
- iOS साठी CoreML आणि Android साठी NNAPI सारख्या प्लॅटफॉर्म-विशिष्ट ऑप्टिमायझेशन ONNX Runtime द्वारे समर्थित
- पूर्ण जनरेटिव्ह AI लूप अंमलात आणते ज्यामध्ये पूर्व आणि पोस्ट प्रोसेसिंग, इनफरन्स, लॉजिट्स प्रोसेसिंग, शोध आणि सॅम्पलिंग, आणि KV कॅश व्यवस्थापन यांचा समावेश आहे

### विकासाचे फायदे
.NET MAUI दृष्टिकोन विकसकांना त्यांच्या विद्यमान C# आणि .NET कौशल्यांचा लाभ घेऊन क्रॉस-प्लॅटफॉर्म AI अनुप्रयोग तयार करण्यास सक्षम करते. ONNX Runtime GenAI फ्रेमवर्क लामा, मिस्त्रल, फाई, जेम्मा आणि इतर अनेक मॉडेल आर्किटेक्चरला समर्थन देते. ऑप्टिमाइझ्ड ARM64 कर्नल्स INT4 क्वांटाइझ्ड मॅट्रिक्स गुणाकार वेगवान करतात, मोबाइल हार्डवेअरवर कार्यक्षम कार्यप्रदर्शन सुनिश्चित करतात आणि परिचित .NET विकास अनुभव राखतात.

### वापर प्रकरणे
हे समाधान .NET तंत्रज्ञानाचा वापर करून AI-सक्षम मोबाइल अनुप्रयोग तयार करू इच्छिणाऱ्या विकसकांसाठी आदर्श आहे, ज्यामध्ये बुद्धिमान चॅटबॉट्स, प्रतिमा ओळख अनुप्रयोग, भाषा अनुवाद साधने आणि वैयक्तिकृत शिफारस प्रणाली समाविष्ट आहेत, जे गोपनीयता आणि ऑफलाइन क्षमतेसाठी पूर्णपणे ऑन-डिव्हाइस चालतात.

**अधिक जाणून घ्या**: [.NET MAUI ONNX Runtime GenAI Example](https://github.com/microsoft/onnxruntime-genai/tree/jialli/genny-maui/examples/csharp/GennyMaui)

## ३. Azure मध्ये EdgeAI सह Small Language Models Engine

Microsoft च्या Azure-आधारित EdgeAI समाधानाचा भर Small Language Models (SLMs) कार्यक्षमतेने क्लाउड-एज हायब्रिड वातावरणात तैनात करण्यावर आहे. हा दृष्टिकोन क्लाउड-स्केल AI सेवा आणि एज तैनातीच्या गरजांमधील अंतर भरतो.

### आर्किटेक्चर फायदे
- Azure AI सेवांसह अखंड समाकलन
- ONNX Runtime सह SLMs/LLMs आणि मल्टी-मोडल मॉडेल्स ऑन-डिव्हाइस आणि क्लाउडमध्ये चालवा
- एंटरप्राइझ-स्केल तैनातीसाठी ऑप्टिमाइझ केलेले
- सतत मॉडेल अद्यतने आणि व्यवस्थापनासाठी समर्थन

### वापर प्रकरणे
Azure EdgeAI अंमलबजावणी एंटरप्राइझ-ग्रेड AI तैनातीसाठी उत्कृष्ट आहे ज्यामध्ये क्लाउड व्यवस्थापन क्षमता आहेत. यामध्ये बुद्धिमान दस्तऐवज प्रक्रिया, रिअल-टाइम विश्लेषण आणि क्लाउड आणि एज संगणन संसाधनांचा लाभ घेणारे हायब्रिड AI वर्कफ्लो यांचा समावेश आहे.

**अधिक जाणून घ्या**: [Azure EdgeAI SLM Engine](https://github.com/microsoft/onnxruntime-genai/tree/main/examples/slm_engine)

## [४. Windows ML सह EdgeAI](./windowdeveloper.md)

Windows ML हे Microsoft चे अत्याधुनिक रनटाइम आहे, जे ऑन-डिव्हाइस मॉडेल इनफरन्ससाठी ऑप्टिमाइझ केले आहे आणि तैनाती सुलभ करते, Windows AI Foundry चा पाया म्हणून कार्य करते. हे प्लॅटफॉर्म विकसकांना AI-सक्षम Windows अनुप्रयोग तयार करण्यास सक्षम करते जे PC हार्डवेअरच्या संपूर्ण श्रेणीचा लाभ घेतात.

### प्लॅटफॉर्म क्षमता
- Windows 11 PCs वर चालते, आवृत्ती 24H2 (बिल्ड 26100) किंवा त्याहून अधिक
- x64 आणि ARM64 PC हार्डवेअरवर कार्य करते, जरी PC मध्ये NPU किंवा GPU नसले तरी
- विकसकांना त्यांचे स्वतःचे मॉडेल आणण्यास आणि AMD, Intel, NVIDIA आणि Qualcomm यांसारख्या सिलिकॉन भागीदार परिसंस्थेमध्ये कार्यक्षमतेने तैनात करण्यास सक्षम करते
- पायाभूत सुविधा APIs चा लाभ घेतल्याने, विकसकांना वेगवेगळ्या सिलिकॉनसाठी लक्ष्यित करण्यासाठी त्यांच्या अॅपचे एकाधिक बिल्ड तयार करण्याची आवश्यकता नाही

### विकसक फायदे
Windows ML हार्डवेअर आणि अंमलबजावणी प्रदात्यांचे संक्षेप करते, त्यामुळे तुम्ही तुमचा कोड लिहिण्यावर लक्ष केंद्रित करू शकता. तसेच, Windows ML नवीन NPUs, GPUs आणि CPUs ला समर्थन देण्यासाठी आपोआप अद्यतनित होते. प्लॅटफॉर्म विविध Windows हार्डवेअर परिसंस्थेमध्ये AI विकासासाठी एकसंध फ्रेमवर्क प्रदान करते.

## [५. Foundry Local Applications सह EdgeAI](./foundrylocal.md)

Foundry Local Windows आणि Mac विकसकांना .NET मध्ये स्थानिक संसाधनांचा वापर करून Retrieval Augmented Generation (RAG) अनुप्रयोग तयार करण्यास सक्षम करते, स्थानिक भाषा मॉडेल्सना सेमॅंटिक शोध क्षमतांसह एकत्रित करते. हा दृष्टिकोन गोपनीयता-केंद्रित AI उपाय प्रदान करतो जे पूर्णपणे स्थानिक पायाभूत सुविधांवर कार्य करतात.

### तांत्रिक आर्किटेक्चर
- Phi भाषा मॉडेल, स्थानिक एम्बेडिंग्ज आणि सेमॅंटिक कर्नल एकत्र करून RAG परिस्थिती तयार करते
- सामग्री आणि तिचा सेमॅंटिक अर्थ दर्शविणाऱ्या फ्लोटिंग-पॉइंट मूल्यांच्या अॅरे (व्हेक्टर) म्हणून एम्बेडिंग्ज वापरते
- सेमॅंटिक कर्नल मुख्य समन्वयक म्हणून कार्य करते, Phi आणि स्मार्ट घटकांना एकत्रित करून अखंड RAG पाइपलाइन तयार करते
- SQLite आणि Qdrant यांसारख्या स्थानिक व्हेक्टर डेटाबेससाठी समर्थन

### अंमलबजावणी फायदे
RAG, किंवा Retrieval Augmented Generation, म्हणजे "काही माहिती शोधा आणि ती प्रॉम्प्टमध्ये ठेवा" असे म्हणण्याचा एक सोपा मार्ग आहे. ही स्थानिक अंमलबजावणी डेटा गोपनीयता सुनिश्चित करते आणि सानुकूल ज्ञान तळांवर आधारित बुद्धिमान प्रतिसाद प्रदान करते. हा दृष्टिकोन डेटा सार्वभौमत्व आणि ऑफलाइन ऑपरेशन क्षमतांची आवश्यकता असलेल्या एंटरप्राइझ परिस्थितीसाठी विशेषतः मौल्यवान आहे.

**अधिक जाणून घ्या**: 
- [Foundry Local](./foundrylocal.md)
- [Foundry Local RAG Samples](https://github.com/microsoft/Foundry-Local/tree/main/samples/dotNET/rag)

### Windows Foundry Local

Microsoft Foundry Local ONNX Runtime द्वारे समर्थित OpenAI-सुसंगत REST सर्व्हर प्रदान करते, जे Windows वर स्थानिक मॉडेल चालवण्यासाठी आहे. खाली एक जलद, सत्यापित सारांश दिला आहे; अधिक तपशीलांसाठी अधिकृत दस्तऐवज पहा.

- सुरुवात करा: https://learn.microsoft.com/azure/ai-foundry/foundry-local/get-started
- आर्किटेक्चर: https://learn.microsoft.com/azure/ai-foundry/foundry-local/concepts/foundry-local-architecture
- CLI संदर्भ: https://learn.microsoft.com/azure/ai-foundry/foundry-local/reference/reference-cli
- या रेपोमध्ये पूर्ण Windows मार्गदर्शक: [foundrylocal.md](./foundrylocal.md)

Windows वर स्थापित किंवा अपग्रेड करा (cmd.exe):
```cmd
winget install Microsoft.FoundryLocal
winget upgrade --id Microsoft.FoundryLocal
foundry --version
```

CLI श्रेणी एक्सप्लोर करा:
```cmd
foundry model --help
foundry service --help
foundry cache --help
```

मॉडेल चालवा आणि डायनॅमिक एंडपॉइंट शोधा:
```cmd
foundry model run gpt-oss-20b
foundry service status
```

मॉडेल्स सूचीबद्ध करण्यासाठी जलद REST तपासणी (स्थितीतून PORT बदला):
```cmd
curl -s http://localhost:PORT/v1/models
```

टीप:
- SDK समाकलन: https://learn.microsoft.com/azure/ai-foundry/foundry-local/how-to/how-to-integrate-with-inference-sdks
- तुमचे स्वतःचे मॉडेल आणा (कंपाईल करा): https://learn.microsoft.com/azure/ai-foundry/foundry-local/how-to/how-to-compile-hugging-face-models

## Windows EdgeAI विकास संसाधने

विशेषतः Windows प्लॅटफॉर्मला लक्ष्य करणाऱ्या विकसकांसाठी, आम्ही Windows EdgeAI परिसंस्थेचे संपूर्ण मार्गदर्शन तयार केले आहे. हे संसाधन Windows AI Foundry बद्दल तपशीलवार माहिती प्रदान करते, ज्यामध्ये APIs, साधने आणि Windows वर EdgeAI विकासासाठी सर्वोत्तम पद्धतींचा समावेश आहे.

### Windows AI Foundry प्लॅटफॉर्म
Windows AI Foundry प्लॅटफॉर्म Windows डिव्हाइसवर Edge AI विकासासाठी विशेषतः डिझाइन केलेल्या साधनांचा आणि APIs चा व्यापक संच प्रदान करते. यामध्ये NPU-प्रवर्धित हार्डवेअरसाठी विशेष समर्थन, Windows ML समाकलन आणि प्लॅटफॉर्म-विशिष्ट ऑप्टिमायझेशन तंत्रांचा समावेश आहे.

**सर्वसमावेशक मार्गदर्शक**: [Windows EdgeAI Development Guide](../windowdeveloper.md)

या मार्गदर्शकात समाविष्ट आहे:
- Windows AI Foundry प्लॅटफॉर्मचा आढावा आणि घटक
- NPU हार्डवेअरवर कार्यक्षम इनफरन्ससाठी Phi Silica API
- प्रतिमा प्रक्रिया आणि OCR साठी संगणक दृष्टी APIs
- Windows ML रनटाइम समाकलन आणि ऑप्टिमायझेशन
- स्थानिक विकास आणि चाचणीसाठी Foundry Local CLI
- Windows डिव्हाइससाठी हार्डवेअर ऑप्टिमायझेशन धोरणे
- व्यावहारिक अंमलबजावणी उदाहरणे आणि सर्वोत्तम पद्धती

### [Edge AI विकासासाठी AI Toolkit](./aitoolkit.md)
Visual Studio Code वापरणाऱ्या विकसकांसाठी, AI Toolkit विस्तार Edge AI अनुप्रयोग तयार करणे, चाचणी करणे आणि तैनात करणे यासाठी विशेषतः डिझाइन केलेले सर्वसमावेशक विकास वातावरण प्रदान करते. हे टूलकिट VS Code मध्ये संपूर्ण Edge AI विकास कार्यप्रवाह सुलभ करते.

**विकास मार्गदर्शक**: [AI Toolkit for Edge AI Development](./aitoolkit.md)

AI Toolkit मार्गदर्शकात समाविष्ट आहे:
- एज तैनातीसाठी मॉडेल शोध आणि निवड
- स्थानिक चाचणी आणि ऑप्टिमायझेशन कार्यप्रवाह
- Edge मॉडेल्ससाठी ONNX आणि Ollama समाकलन
- मॉडेल रूपांतरण आणि क्वांटायझेशन तंत्र
- एज परिस्थितीसाठी एजंट विकास
- कार्यप्रदर्शन मूल्यांकन आणि निरीक्षण
- तैनाती तयारी आणि सर्वोत्तम पद्धती

## निष्कर्ष

हे पाच EdgeAI अंमलबजावणी आज उपलब्ध असलेल्या एज AI उपायांची परिपक्वता आणि विविधता दर्शवतात. Jetson Orin Nano सारख्या हार्डवेअर-प्रवर्धित एज डिव्हाइसपासून ONNX Runtime GenAI आणि Windows ML सारख्या सॉफ्टवेअर फ्रेमवर्कपर्यंत, विकसकांना एजवर बुद्धिमान अनुप्रयोग तैनात करण्यासाठी अभूतपूर्व पर्याय उपलब्ध आहेत.

या सर्व प्लॅटफॉर्ममधील समान धागा म्हणजे AI क्षमतांची लोकशाहीकरण, ज्यामुळे विविध कौशल्य पातळ्या आणि वापर प्रकरणांमधील विकसकांसाठी प्रगत मशीन लर्निंग प्रवेशयोग्य बनते. मोबाइल अनुप्रयोग, डेस्कटॉप सॉफ्टवेअर किंवा एम्बेडेड सिस्टम तयार करत असो, हे EdgeAI उपाय एजवर कार्यक्षमतेने आणि खाजगीपणे कार्य करणाऱ्या पुढील पिढीच्या बुद्धिमान अनुप्रयोगांसाठी पाया प्रदान करतात.

प्रत्येक प्लॅटफॉर्म अद्वितीय फायदे प्रदान करतो: हार्डवेअर-प्रवर्धित एज संगणनासाठी Jetson Orin Nano, क्रॉस-प्लॅटफॉर्म मोबाइल विकासासाठी ONNX Runtime GenAI, एंटरप्राइझ क्लाउड-एज एकत्रीकरणासाठी Azure EdgeAI, Windows-नेटिव्ह अनुप्रयोगांसाठी Windows ML, आणि गोपनीयता-केंद्रित RAG अंमलबजावणीसाठी Foundry Local. एकत्रितपणे, ते EdgeAI विकासासाठी एक व्यापक परिसंस्था दर्शवतात.
