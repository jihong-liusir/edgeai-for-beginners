# Sesija 4: Najnoviji modeli – LLM-ovi, SLM-ovi i inferencija na uređaju

## Pregled

Usporedite LLM-ove i SLM-ove, procijenite prednosti i nedostatke lokalne i cloud inferencije te implementirajte demonstracije koje prikazuju EdgeAI scenarije koristeći Phi i ONNX Runtime. Također ćemo istaknuti Chainlit RAG, opcije inferencije putem WebGPU-a i integraciju s Open WebUI.

Reference:
- Foundry Local dokumentacija: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Open WebUI vodič (chat aplikacija s Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Ciljevi učenja
- Razumjeti prednosti i nedostatke LLM-ova i SLM-ova u pogledu troškova, kašnjenja i točnosti
- Odabrati između lokalne i cloud inferencije za specifične poslovne potrebe
- Implementirati mali RAG demo koristeći Chainlit
- Istražiti WebGPU za ubrzanje na strani preglednika
- Povezati Open WebUI s Foundry Local

## Dio 1: LLM vs SLM – Matrica odluka

Razmotrite:
- Kašnjenje: SLM-ovi na uređaju često pružaju odgovore u manje od sekunde
- Trošak: lokalna inferencija smanjuje troškove clouda
- Privatnost: osjetljivi podaci ostaju na uređaju
- Sposobnost: LLM-ovi mogu nadmašiti SLM-ove u složenim zadacima
- Pouzdanost: hibridne strategije smanjuju rizik od prekida rada

## Dio 2: Lokalno vs Cloud – Hibridni obrasci

- Lokalno prvo s cloud podrškom za velike/složene upite
- Cloud prvo s lokalnim za scenarije osjetljive na privatnost ili offline rad
- Usmjeravanje prema vrsti zadatka (generiranje koda prema DeepSeek, opći chat prema Phi/Qwen)

## Dio 3: RAG Chat aplikacija s Chainlit (Minimalno)

Instalirajte ovisnosti:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Pokrenite:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Proširite: dodajte jednostavan retriver (lokalne datoteke) i dodajte dohvaćeni kontekst korisničkom upitu.

## Dio 4: WebGPU inferencija (Upozorenje)

Pokrenite male modele izravno u pregledniku koristeći WebGPU. Ovo je idealno za demonstracije koje daju prednost privatnosti i iskustva bez instalacije. Ispod je minimalan, korak-po-korak primjer koristeći ONNX Runtime Web s WebGPU izvršnim providerom.

1) Provjerite podršku za WebGPU
- Preglednici bazirani na Chromiumu: chrome://gpu → potvrditi da je “WebGPU” omogućeno
- Programska provjera (provjerit ćemo i u kodu): `if (!('gpu' in navigator)) { /* nema WebGPU */ }`

2) Kreirajte minimalan projekt
Kreirajte mapu i dvije datoteke: `index.html` i `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Poslužite lokalno (Windows cmd.exe)
Koristite jednostavni statički server kako bi preglednik mogao dohvatiti model.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Otvorite http://localhost:5173 u svom pregledniku. Trebali biste vidjeti logove inicijalizacije, kreiranje sesije s WebGPU-om i predikciju argmax.

4) Rješavanje problema
- Ako WebGPU nije dostupan: ažurirajte Chrome/Edge i osigurajte da su GPU driveri aktualni, zatim provjerite chrome://flags za “Enable WebGPU”.
- Ako se pojave CORS ili fetch greške: osigurajte da poslužujete datoteke preko http:// (ne file://) i da URL modela dopušta cross-origin zahtjeve.
- Povratak na CPU: promijenite `executionProviders: ['wasm']` kako biste provjerili osnovno ponašanje.

5) Sljedeći koraci
- Zamijenite model specifičnim za domenu (npr. klasifikacija slika ili mali tekstualni model).
- Dodajte logiku za predobradu/postobradu stvarnih ulaza.
- Za veće modele ili produkcijsku latenciju, preferirajte Foundry Local ili ONNX Runtime Server.

## Dio 5: Open WebUI + Foundry Local (Korak-po-korak)

Ovo povezuje Open WebUI s OpenAI-kompatibilnim endpointom Foundry Local za lokalni chat UI.

1) Preduvjeti
- Foundry Local instaliran i funkcionalan (`foundry --version`)
- Jedan model spreman za lokalno pokretanje (npr. `phi-4-mini`)
- Instaliran Docker Desktop (preporučeno za Open WebUI)

2) Pokrenite model s Foundry Local
```powershell
foundry model run phi-4-mini
```
Ovo izlaže OpenAI-kompatibilni API na `http://localhost:8000`.

3) Pokrenite Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Napomene:
- Na Windowsu, `host.docker.internal` omogućuje da kontejner pristupi vašem hostu na `localhost`.
- Postavljamo `OPENAI_API_BASE_URL` na endpoint Foundry Local i lažni `OPENAI_API_KEY`.

4) Konfigurirajte iz Open WebUI sučelja (alternativa)
- Otvorite http://localhost:3000
- Dovršite početnu postavku (admin korisnik)
- Idite na Postavke → Modeli/Provideri
- Postavite osnovni URL: `http://host.docker.internal:8000/v1`
- Postavite API ključ: `local-key` (placeholder)
- Spremite

5) Pokrenite testni upit
- U Open WebUI chatu, odaberite ili unesite naziv modela `phi-4-mini`
- Upit: “Navedite pet prednosti AI inferencije na uređaju.”
- Trebali biste vidjeti odgovor koji se prenosi iz vašeg lokalnog modela

6) Rješavanje problema
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Opcionalno: Trajno spremanje podataka Open WebUI
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
