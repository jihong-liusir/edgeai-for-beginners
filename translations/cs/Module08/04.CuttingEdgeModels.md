# Session 4: Nejmodernější modely – LLMs, SLMs a inference na zařízení

## Přehled

Porovnejte LLMs a SLMs, zhodnoťte kompromisy mezi lokální a cloudovou inferencí a implementujte ukázky, které demonstrují scénáře EdgeAI pomocí Phi a ONNX Runtime. Také se zaměříme na Chainlit RAG, možnosti inference pomocí WebGPU a integraci Open WebUI.

Reference:
- Dokumentace Foundry Local: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Návod Open WebUI (chatovací aplikace s Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Cíle učení
- Porozumět kompromisům mezi LLM a SLM z hlediska nákladů, latence a přesnosti
- Vybrat mezi lokální a cloudovou inferencí podle konkrétních obchodních potřeb
- Implementovat malou ukázku RAG s Chainlit
- Prozkoumat WebGPU pro akceleraci na straně prohlížeče
- Připojit Open WebUI k Foundry Local

## Část 1: LLM vs SLM – Rozhodovací matice

Zvažte:
- Latence: SLMs na zařízení často poskytují odpovědi v řádu milisekund
- Náklady: lokální inference snižuje náklady na cloud
- Soukromí: citlivá data zůstávají na zařízení
- Schopnosti: LLMs mohou překonat SLMs v komplexních úlohách
- Spolehlivost: hybridní strategie snižují riziko výpadků

## Část 2: Lokální vs cloudová inference – Hybridní vzory

- Lokální priorita s cloudovým zálohováním pro velké/komplexní dotazy
- Cloudová priorita s lokální podporou pro citlivé na soukromí nebo offline scénáře
- Směrování podle typu úlohy (generování kódu na DeepSeek, obecný chat na Phi/Qwen)

## Část 3: RAG Chat App s Chainlit (Minimalistická verze)

Instalace závislostí:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Spuštění:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Rozšíření: přidejte jednoduchý retriever (lokální soubory) a připojte získaný kontext k uživatelskému dotazu.

## Část 4: Inference pomocí WebGPU (Upozornění)

Spouštějte malé modely přímo v prohlížeči pomocí WebGPU. To je ideální pro ukázky zaměřené na soukromí a zkušenosti bez nutnosti instalace. Níže je minimalistický, krok za krokem příklad použití ONNX Runtime Web s poskytovatelem WebGPU.

1) Zkontrolujte podporu WebGPU
- Prohlížeče Chromium: chrome://gpu → potvrďte, že je „WebGPU“ povoleno
- Programová kontrola (zkontrolujeme také v kódu): `if (!('gpu' in navigator)) { /* no WebGPU */ }`

2) Vytvořte minimalistický projekt
Vytvořte složku a dva soubory: `index.html` a `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Spusťte lokálně (Windows cmd.exe)
Použijte jednoduchý statický server, aby prohlížeč mohl načíst model.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Otevřete http://localhost:5173 ve svém prohlížeči. Měli byste vidět inicializační logy, vytvoření relace s WebGPU a predikci pomocí argmax.

4) Řešení problémů
- Pokud WebGPU není dostupné: aktualizujte Chrome/Edge a ujistěte se, že ovladače GPU jsou aktuální, poté zkontrolujte chrome://flags pro „Enable WebGPU“.
- Pokud se objeví chyby CORS nebo fetch: ujistěte se, že soubory servírujete přes http:// (ne file://) a že URL modelu umožňuje požadavky z jiných domén.
- Záložní možnost CPU: změňte `executionProviders: ['wasm']` pro ověření základního chování.

5) Další kroky
- Vyměňte za doménově specifický ONNX model (např. klasifikace obrázků nebo malý textový model).
- Přidejte logiku pro předzpracování/postprocessing reálných vstupů.
- Pro větší modely nebo produkční latenci preferujte Foundry Local nebo ONNX Runtime Server.

## Část 5: Open WebUI + Foundry Local (Krok za krokem)

Toto propojuje Open WebUI s OpenAI-kompatibilním endpointem Foundry Local pro lokální chatovací UI.

1) Předpoklady
- Foundry Local nainstalován a funkční (`foundry --version`)
- Jeden model připravený ke spuštění lokálně (např. `phi-4-mini`)
- Nainstalovaný Docker Desktop (doporučeno pro Open WebUI)

2) Spusťte model pomocí Foundry Local
```powershell
foundry model run phi-4-mini
```
Tím se zpřístupní OpenAI-kompatibilní API na `http://localhost:8000`.

3) Spusťte Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Poznámky:
- Na Windows umožňuje `host.docker.internal` kontejneru přístup k vašemu hostiteli na `localhost`.
- Nastavíme `OPENAI_API_BASE_URL` na endpoint Foundry Local a fiktivní `OPENAI_API_KEY`.

4) Konfigurace z UI Open WebUI (alternativa)
- Otevřete http://localhost:3000
- Dokončete počáteční nastavení (admin uživatel)
- Přejděte na Nastavení → Modely/Poskytovatelé
- Nastavte základní URL: `http://host.docker.internal:8000/v1`
- Nastavte API klíč: `local-key` (zástupný klíč)
- Uložte

5) Otestujte dotaz
- V chatu Open WebUI vyberte nebo zadejte název modelu `phi-4-mini`
- Dotaz: „Vyjmenujte pět výhod inference na zařízení.“
- Měli byste vidět odpověď streamovanou z vašeho lokálního modelu

6) Řešení problémů
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Volitelné: Uložení dat Open WebUI
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
