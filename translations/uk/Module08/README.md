# Модуль 08: Практичне використання Microsoft Foundry Local - Повний набір інструментів для розробників

## Огляд

Microsoft Foundry Local представляє нове покоління розробки Edge AI, надаючи розробникам потужні інструменти для створення, розгортання та масштабування AI-додатків локально, зберігаючи безперервну інтеграцію з Azure AI Foundry. Цей модуль охоплює Foundry Local від встановлення до розробки складних агентів.

**Основні технології:**
- Microsoft Foundry Local CLI та SDK
- Інтеграція з Azure AI Foundry
- Локальне виконання моделей
- Кешування та оптимізація моделей на пристрої
- Архітектури на основі агентів

## Цілі навчання модуля

Після завершення цього модуля ви зможете:

- **Опанувати налаштування Foundry Local**: Встановити, налаштувати та оптимізувати Foundry Local для розробки на Windows 11
- **Розгортати різноманітні моделі**: Виконувати моделі phi, qwen, deepseek та GPT-OSS-20B локально за допомогою CLI команд
- **Створювати рішення для виробництва**: Розробляти AI-додатки з використанням просунутого інженерії запитів та інтеграції даних
- **Використовувати екосистему з відкритим кодом**: Інтегрувати моделі Hugging Face та доповнення від спільноти
- **Порівнювати AI-архітектури**: Розуміти компроміси між LLMs та SLMs і стратегії їх розгортання
- **Розробляти AI-агентів**: Створювати інтелектуальних агентів, використовуючи архітектуру Foundry Local та можливості прив'язки
- **Реалізовувати моделі як інструменти**: Створювати модульні, налаштовувані AI-рішення для корпоративних додатків

## Попередні вимоги

### Системні вимоги
- **Операційна система**: Windows 11 (22H2 або новіша)
- **Пам'ять**: 16GB RAM (32GB рекомендовано для більших моделей)
- **Сховище**: 50GB вільного місця для кешування моделей
- **Апаратне забезпечення**: Пристрій з підтримкою NPU рекомендовано (Copilot+ PC), GPU опціонально
- **Мережа**: Високошвидкісний інтернет для початкового завантаження моделей

### Середовище розробки
- Visual Studio Code з розширенням AI Toolkit
- Python 3.10+ та pip
- Git для контролю версій
- PowerShell або Command Prompt
- Azure CLI (опціонально для інтеграції з хмарою)

### Знання
- Базове розуміння концепцій AI/ML
- Знайомство з командним рядком
- Основи програмування на Python
- Концепції REST API
- Базові знання про запити та виконання моделей

## Швидкий старт для зразків

### 1) Налаштування середовища (Windows cmd.exe)
```cmd
cd Module08
py -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Запуск локальної моделі (новий термінал)
```cmd
foundry model list
foundry model run phi-4-mini
```

### 3) Виконання демонстрації Chainlit (Сесія 4)
```cmd
cd Module08
.\.venv\Scripts\activate
chainlit run samples\04\app.py -w
```

### 4) Запуск координатора з кількома агентами (Сесія 5)
```cmd
cd Module08
.\.venv\Scripts\activate
python -m samples.05.agents.coordinator
```

Якщо ви бачите помилки з'єднання, перевірте Foundry Local:
```cmd
curl http://localhost:8000/v1/models
```

### Налаштування маршрутизатора (Сесія 6)
Маршрутизатор виконує швидку перевірку стану та підтримує конфігурацію на основі середовища:
```cmd
set BASE_URL=http://localhost:8000
set GENERAL_MODEL=phi-4-mini
set REASONING_MODEL=deepseek-r1-distill-qwen-7b
set CODE_MODEL=qwen2.5-7b-instruct
REM Or provide a full JSON registry
set TOOL_REGISTRY={"general":{"model":"phi-4-mini"}}
python samples\06\router.py "Pick the best model for code refactoring"
```

Цей модуль представляє передові технології розробки Edge AI, поєднуючи корпоративні інструменти Microsoft з гнучкістю та інноваціями екосистеми з відкритим кодом. Опанувавши Foundry Local, ви будете на передовій розробки AI-додатків.

Для Azure OpenAI (Сесія 2) дивіться README зразка для необхідних змінних середовища та налаштувань версії API.

## Огляд зразків
