# Istunto 4: Huippumallit – LLM:t, SLM:t ja laitepohjainen inferenssi

## Yleiskatsaus

Vertaa LLM:itä ja SLM:itä, arvioi paikallisen ja pilvipohjaisen inferenssin kompromisseja, ja toteuta demoja, jotka esittelevät EdgeAI-skenaarioita Phi:n ja ONNX Runtime:n avulla. Korostamme myös Chainlit RAG:ia, WebGPU-inferenssivaihtoehtoja ja Open WebUI -integraatiota.

Viitteet:
- Foundry Local -dokumentaatio: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Open WebUI -ohjeet (chat-sovellus Open WebUI:lla): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Oppimistavoitteet
- Ymmärrä LLM:n ja SLM:n kompromissit kustannusten, viiveen ja tarkkuuden osalta
- Valitse paikallisen ja pilvipohjaisen inferenssin välillä liiketoiminnan tarpeiden mukaan
- Toteuta pieni RAG-demo Chainlitin avulla
- Tutki WebGPU:ta selaimen puoleiseen kiihdytykseen
- Yhdistä Open WebUI Foundry Localiin

## Osa 1: LLM vs SLM – Päätösmatriisi

Huomioi:
- Viive: SLM:t laitteessa tarjoavat usein alle sekunnin vasteajan
- Kustannukset: paikallinen inferenssi vähentää pilvikustannuksia
- Tietosuoja: arkaluontoiset tiedot pysyvät laitteessa
- Kyvykkyys: LLM:t voivat suoriutua paremmin monimutkaisissa tehtävissä
- Luotettavuus: hybridistrategiat vähentävät käyttökatkojen riskiä

## Osa 2: Paikallinen vs pilvi – Hybridimallit

- Paikallinen ensin, pilvi varajärjestelmänä suurille/monimutkaisille kehotteille
- Pilvi ensin, paikallinen tietosuojaa vaativille tai offline-skenaarioille
- Reititys tehtävätyypin mukaan (koodin generointi DeepSeekille, yleinen chat Phi/Qwenille)

## Osa 3: RAG Chat -sovellus Chainlitillä (Minimaalinen)

Asenna riippuvuudet:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Suorita:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Laajenna: lisää yksinkertainen hakutoiminto (paikalliset tiedostot) ja liitä haettu konteksti käyttäjän kehotteen alkuun.

## Osa 4: WebGPU-inferenssi (Huomio)

Suorita pieniä malleja suoraan selaimessa WebGPU:n avulla. Tämä sopii tietosuojaa korostaviin demoihin ja asennusvapaisiin kokemuksiin. Alla on minimaalinen, vaiheittainen esimerkki ONNX Runtime Webistä WebGPU-suoritusalustalla.

1) Tarkista WebGPU-tuki
- Chromium-selaimet: chrome://gpu → varmista, että “WebGPU” on käytössä
- Ohjelmallinen tarkistus (tarkistamme myös koodissa): `if (!('gpu' in navigator)) { /* ei WebGPU:ta */ }`

2) Luo minimaalinen projekti
Luo kansio ja kaksi tiedostoa: `index.html` ja `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Palvele paikallisesti (Windows cmd.exe)
Käytä yksinkertaista staattista palvelinta, jotta selain voi hakea mallin.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Avaa http://localhost:5173 selaimessasi. Näet alustuksen lokit, istunnon luomisen WebGPU:lla ja argmax-ennusteen.

4) Vianetsintä
- Jos WebGPU ei ole käytettävissä: päivitä Chrome/Edge ja varmista, että GPU-ajurit ovat ajan tasalla, tarkista sitten chrome://flags ja ota “Enable WebGPU” käyttöön.
- Jos CORS- tai fetch-virheitä ilmenee: varmista, että palvelet tiedostoja http://-osoitteella (ei file://) ja että mallin URL sallii cross-origin-pyynnöt.
- Vaihtoehtona CPU: vaihda `executionProviders: ['wasm']` varmistaaksesi peruskäyttäytymisen.

5) Seuraavat askeleet
- Vaihda tilalle toimialakohtainen ONNX-malli (esim. kuvien luokittelu tai pieni tekstimalli).
- Lisää esikäsittely/jälkikäsittely logiikkaa todellisille syötteille.
- Suuremmille malleille tai tuotantoviiveelle suosi Foundry Localia tai ONNX Runtime Serveriä.

## Osa 5: Open WebUI + Foundry Local (Vaiheittain)

Tämä yhdistää Open WebUI:n Foundry Localin OpenAI-yhteensopivaan rajapintaan paikallista chat-käyttöliittymää varten.

1) Esivaatimukset
- Foundry Local asennettuna ja toiminnassa (`foundry --version`)
- Yksi malli valmiina paikalliseen käyttöön (esim. `phi-4-mini`)
- Docker Desktop asennettuna (suositeltu Open WebUI:lle)

2) Käynnistä malli Foundry Localilla
```powershell
foundry model run phi-4-mini
```
Tämä avaa OpenAI-yhteensopivan API:n osoitteessa `http://localhost:8000`.

3) Käynnistä Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Huomautukset:
- Windowsissa `host.docker.internal` mahdollistaa säilön pääsyn isäntään osoitteessa `localhost`.
- Asetamme `OPENAI_API_BASE_URL`:ksi Foundry Localin rajapinnan osoitteen ja näennäisen `OPENAI_API_KEY`:n.

4) Määritä Open WebUI:n käyttöliittymästä (vaihtoehto)
- Siirry osoitteeseen http://localhost:3000
- Täytä alkuasetukset (ylläpitäjän käyttäjä)
- Mene kohtaan Asetukset → Mallit/Palveluntarjoajat
- Aseta Perus-URL: `http://host.docker.internal:8000/v1`
- Aseta API-avain: `local-key` (paikkamerkki)
- Tallenna

5) Suorita testikehote
- Open WebUI -chatissa valitse tai syötä mallin nimi `phi-4-mini`
- Kehote: “Listaa viisi hyötyä laitepohjaisesta AI-inferenssistä.”
- Näet vastauksen, joka suoratoistetaan paikalliselta mallilta

6) Vianetsintä
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Valinnainen: Pysyvä Open WebUI -data
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
