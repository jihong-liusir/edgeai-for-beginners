## Johdanto

Tervetuloa EdgeAI aloittelijoille -opas! Tämä dokumentti on suunniteltu auttamaan sinua kurssimateriaalien läpikäynnissä ja oppimiskokemuksen maksimoimisessa. Se tarjoaa jäsenneltyjä oppimispolkuja, ehdotettuja opintoaikatauluja, keskeisten käsitteiden tiivistelmiä ja lisäresursseja EdgeAI-teknologioiden syvällisempään ymmärtämiseen.

Kyseessä on tiivis 20 tunnin kurssi, joka tarjoaa olennaista tietoa EdgeAI:sta tehokkaassa ajassa. Se sopii erinomaisesti kiireisille ammattilaisille ja opiskelijoille, jotka haluavat nopeasti hankkia käytännön taitoja tässä nousevassa teknologiassa.

## Kurssin yleiskatsaus

Kurssi on jaettu seitsemään kattavaan moduuliin:

1. **EdgeAI perusteet ja muutos** - Ymmärrä keskeiset käsitteet ja teknologian siirtymä
2. **Pienten kielimallien perusteet** - Tutustu eri SLM-perheisiin ja niiden arkkitehtuureihin
3. **Pienten kielimallien käyttöönotto** - Käytännön käyttöönoton strategiat
4. **Mallin muotojen muuntaminen ja kvantisointi** - Edistynyt optimointi kuudella kehysteknologialla, mukaan lukien OpenVINO
5. **SLMOps - Pienten kielimallien operointi** - Tuotantoketjun hallinta ja käyttöönotto
6. **SLM agenttijärjestelmät** - AI-agentit, funktiokutsut ja Model Context Protocol
7. **EdgeAI toteutusesimerkit** - AI Toolkit, Windows-kehitys ja alustakohtaiset toteutukset

## Kuinka käyttää tätä opasta

- **Progressiivinen oppiminen**: Seuraa moduuleja järjestyksessä parhaan oppimiskokemuksen saavuttamiseksi
- **Tietotarkistuspisteet**: Käytä itsearviointikysymyksiä jokaisen osion jälkeen
- **Käytännön harjoittelu**: Suorita ehdotetut harjoitukset teoreettisten käsitteiden vahvistamiseksi
- **Lisäresurssit**: Tutustu lisämateriaaleihin aiheista, jotka kiinnostavat sinua eniten

## Moduuli 5: SLMOps - Pienten kielimallien operointi

### Keskeiset oppimistavoitteet

- Ymmärrä SLMOps:n elinkaaren hallinnan periaatteet
- Hallitse distillointi- ja hienosäätötekniikat reunalaitteiden käyttöönottoa varten
- Toteuta tuotantokäyttöstrategioita seurannalla
- Rakenna yritystason SLM-operaatio- ja ylläpitotyönkulkuja

### Opiskelun painopistealueet

#### Osa 1: Johdatus SLMOps:iin
- **Keskeiset käsitteet**: 
  - SLMOps:n paradigman muutos tekoälyn operoinnissa
  - Kustannustehokkuus ja yksityisyyttä korostava arkkitehtuuri
  - Strateginen liiketoimintavaikutus ja kilpailuedut

#### Osa 2: Mallin distillointi
- **Keskeiset käsitteet**: 
  - Tiedonsiirtotekniikat
  - Kaksivaiheisen distillointiprosessin toteutus
  - Azure ML:n distillointityönkulut

#### Osa 3: Hienosäätöstrategiat
- **Keskeiset käsitteet**: 
  - Parametrien tehokas hienosäätö (PEFT)
  - LoRA- ja QLoRA-tekniikat
  - Monisovittimen koulutus ja hyperparametrien optimointi

#### Osa 4: Tuotantokäyttö
- **Keskeiset käsitteet**: 
  - Mallin muuntaminen ja kvantisointi tuotantokäyttöön
  - Foundry Local -käyttöönottokonfiguraatio
  - Suorituskyvyn vertailu ja laadun validointi

### Itsearviointikysymykset

1. Kuinka SLMOps eroaa perinteisestä MLOps:sta?
2. Selitä mallin distilloinnin hyödyt reunalaitteiden käyttöönotossa.
3. Mitkä ovat keskeiset näkökohdat SLM-mallien hienosäädössä resurssirajoitteisissa ympäristöissä?
4. Kuvaile täydellinen tuotantokäyttöputki reunalaitteiden tekoälysovelluksille.

### Käytännön harjoitukset

1. **Perusdistillointi**: Luo pienempi malli suuremmasta opettajamallista (1 tunti)
2. **Hienosäätökokeilu**: Hienosäädä malli tiettyä alaa varten (1 tunti)
3. **Käyttöputki**: Aseta perus CI/CD-putki mallin käyttöönottoa varten (1 tunti)

## Moduuli 6: SLM-agenttijärjestelmät - Tekoälyagentit ja funktiokutsut

### Keskeiset oppimistavoitteet

- Rakenna älykkäitä tekoälyagentteja reunaympäristöihin pienillä kielimalleilla
- Toteuta funktiokutsutoimintoja systemaattisilla työnkuluilla
- Hallitse Model Context Protocol (MCP) -integraatio standardoitua työkalujen vuorovaikutusta varten
- Luo kehittyneitä agenttijärjestelmiä, jotka vaativat minimaalista ihmisen väliintuloa

### Opiskelun painopistealueet

#### Osa 1: Tekoälyagentit ja SLM:n perusteet
- **Keskeiset käsitteet**: 
  - Agenttien luokittelukehys (refleksi-, mallipohjaiset, tavoitepohjaiset, oppivat agentit)
  - SLM:n ja LLM:n kompromissien analyysi
  - Reunaan soveltuvat agenttisuunnittelumallit
  - Resurssien optimointi agenteille

#### Osa 2: Funktiokutsut pienissä kielimalleissa
- **Keskeiset käsitteet**: 
  - Systemaattinen työnkulun toteutus (tarkoituksen tunnistus, JSON-tuloste, ulkoinen suoritus)
  - Alustakohtaiset toteutukset (Phi-4-mini, valitut Qwen-mallit, Microsoft Foundry Local)
  - Kehittyneet esimerkit (moniagenttiyhteistyö, dynaaminen työkalujen valinta)
  - Tuotantokäytön näkökohdat (nopeusrajoitukset, auditointilokit, turvallisuustoimenpiteet)

#### Osa 3: Model Context Protocol (MCP) -integraatio
- **Keskeiset käsitteet**: 
  - Protokolla-arkkitehtuuri ja kerrostettu järjestelmän suunnittelu
  - Monitaustatuki (Ollama kehitykseen, vLLM tuotantoon)
  - Yhteysprotokollat (STDIO- ja SSE-tilat)
  - Käytännön sovellukset (verkkoprosessointi, datankäsittely, API-integraatio)

### Itsearviointikysymykset

1. Mitkä ovat keskeiset arkkitehtuurin näkökohdat reunalaitteiden tekoälyagenteille?
2. Kuinka funktiokutsut parantavat agenttien kyvykkyyksiä?
3. Selitä Model Context Protocol:n rooli agenttien viestinnässä.

### Käytännön harjoitukset

1. **Yksinkertainen agentti**: Rakenna perus tekoälyagentti funktiokutsuilla (1 tunti)
2. **MCP-integraatio**: Toteuta MCP agenttisovelluksessa (30 minuuttia)

## Moduuli 7: EdgeAI:n toteutusesimerkit

### Keskeiset oppimistavoitteet

- Hallitse AI Toolkit for Visual Studio Code kattavien EdgeAI-kehitystyönkulkujen avulla
- Saavuta asiantuntemus Windows AI Foundry -alustasta ja NPU-optimointistrategioista
- Toteuta EdgeAI useilla laitealustoilla ja käyttöönoton skenaarioilla
- Rakenna tuotantovalmiita EdgeAI-sovelluksia alustakohtaisilla optimoinneilla

### Opiskelun painopistealueet

#### Osa 1: AI Toolkit for Visual Studio Code
- **Keskeiset käsitteet**: 
  - Kattava EdgeAI-kehitysympäristö VS Code:ssa
  - Mallikatalogi ja löytö reunalaitteiden käyttöönottoa varten
  - Paikallinen testaus, optimointi ja agenttien kehitystyönkulut
  - Suorituskyvyn seuranta ja arviointi reunaskenaarioissa

#### Osa 2: Windows EdgeAI:n kehitysopas
- **Keskeiset käsitteet**: 
  - Windows AI Foundry -alustan kattava yleiskatsaus
  - Phi Silica API tehokkaaseen NPU-päättelyyn
  - Tietokonenäkö-API:t kuvankäsittelyyn ja OCR:ään
  - Foundry Local CLI paikalliseen kehitykseen ja testaukseen

#### Osa 3: Alustakohtaiset toteutukset
- **Keskeiset käsitteet**: 
  - NVIDIA Jetson Orin Nano -käyttöönotto (67 TOPS AI-suorituskyky)
  - Mobiilisovellukset .NET MAUI:lla ja ONNX Runtime GenAI:lla
  - Azure EdgeAI-ratkaisut pilvi-reuna-hybridialustalla
  - Windows ML -optimointi universaalilla laitetuella
  - Foundry Local -sovellukset yksityisyyttä korostavalla RAG-toteutuksella

### Itsearviointikysymykset

1. Kuinka AI Toolkit tehostaa EdgeAI-kehitystyönkulkuja?
2. Vertaa käyttöönoton strategioita eri laitealustoilla.
3. Mitkä ovat Windows AI Foundry:n edut reunakehityksessä?
4. Selitä NPU-optimoinnin rooli nykyaikaisissa EdgeAI-sovelluksissa.
5. Kuinka Phi Silica API hyödyntää NPU-laitteistoa suorituskyvyn optimointiin?
6. Vertaa paikallisen ja pilvikäyttöönoton etuja yksityisyyttä korostavissa sovelluksissa.

### Käytännön harjoitukset

1. **AI Toolkit -asennus**: Konfiguroi AI Toolkit ja optimoi malli (1 tunti)
2. **Windows AI Foundry**: Rakenna yksinkertainen Windows AI -sovellus Phi Silica API:lla (1 tunti)
3. **Alustojen välinen käyttöönotto**: Ota sama malli käyttöön kahdella eri alustalla (1 tunti)
4. **NPU-optimointi**: Testaa NPU:n suorituskykyä Windows AI Foundry -työkaluilla (30 minuuttia)
