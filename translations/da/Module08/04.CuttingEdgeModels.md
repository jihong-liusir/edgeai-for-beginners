# Session 4: Avancerede Modeller – LLMs, SLMs og On-Device Inference

## Oversigt

Sammenlign LLMs og SLMs, vurder fordele og ulemper ved lokal vs cloud inference, og implementer demoer, der fremhæver EdgeAI-scenarier ved hjælp af Phi og ONNX Runtime. Vi vil også fremhæve Chainlit RAG, WebGPU inference-muligheder og integration med Open WebUI.

Referencer:
- Foundry Local dokumentation: https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/
- ONNX Runtime GenAI: https://onnxruntime.ai/
- Open WebUI vejledning (chat-app med Open WebUI): https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/how-to/how-to-chat-application-with-open-web-ui

## Læringsmål
- Forstå fordele og ulemper ved LLM vs SLM med hensyn til omkostninger, latenstid og nøjagtighed
- Vælg mellem lokal og cloud inference baseret på specifikke forretningsbehov
- Implementer en lille RAG-demo med Chainlit
- Udforsk WebGPU til acceleration i browseren
- Forbind Open WebUI med Foundry Local

## Del 1: LLM vs SLM – Beslutningsmatrix

Overvej:
- Latenstid: SLMs på enheden leverer ofte svar på under et sekund
- Omkostninger: lokal inference reducerer cloud-omkostninger
- Privatliv: følsomme data forbliver på enheden
- Kapacitet: LLMs kan overgå SLMs i komplekse opgaver
- Pålidelighed: hybride strategier reducerer risikoen for nedetid

## Del 2: Lokal vs Cloud – Hybride Mønstre

- Lokal først med cloud som fallback for store/komplekse forespørgsler
- Cloud først med lokal til privatlivsfølsomme eller offline scenarier
- Rute baseret på opgavetype (kodegenerering til DeepSeek, generel chat til Phi/Qwen)

## Del 3: RAG Chat App med Chainlit (Minimal)

Installer afhængigheder:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install chainlit requests
```

`app.py`:
```python
import chainlit as cl
import requests

BASE_URL = "http://localhost:8000"
MODEL = "phi-4-mini"

@cl.on_chat_start
async def start():
    await cl.Message(content="Welcome to Local RAG Chat!").send()

@cl.on_message
async def main(message: cl.Message):
    # Basic passthrough to Foundry Local (extend with retrieval)
    payload = {
        "model": MODEL,
        "messages": [{"role": "user", "content": message.content}],
        "max_tokens": 300
    }
    r = requests.post(f"{BASE_URL}/v1/chat/completions", json=payload, timeout=60)
    r.raise_for_status()
    content = r.json()["choices"][0]["message"]["content"]
    await cl.Message(content=content).send()
```

Kør:
```powershell
# Ensure a model is running
foundry model run phi-4-mini

# Launch Chainlit
chainlit run app.py -w
```

Udvid: tilføj en simpel retriever (lokale filer) og tilføj den hentede kontekst til brugerens prompt.

## Del 4: WebGPU Inference (Heads-up)

Kør små modeller direkte i browseren ved hjælp af WebGPU. Dette er ideelt til privatlivsfokuserede demoer og oplevelser uden installation. Nedenfor er et minimalt, trin-for-trin eksempel ved brug af ONNX Runtime Web med WebGPU execution provider.

1) Tjek WebGPU-support
- Chromium-browsere: chrome://gpu → bekræft, at “WebGPU” er aktiveret
- Programmatisk tjek (vi tjekker også i kode): `if (!('gpu' in navigator)) { /* no WebGPU */ }`

2) Opret et minimalt projekt
Opret en mappe og to filer: `index.html` og `main.js`.

`index.html`
```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>WebGPU + ONNX Runtime Web (Demo)</title>
    <!-- Load ONNX Runtime Web (WebGPU build) from CDN -->
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.webgpu.min.js"></script>
    <style>
      body { font-family: system-ui, sans-serif; margin: 2rem; }
      pre { background: #f5f5f5; padding: 1rem; overflow:auto; }
    </style>
  </head>
  <body>
    <h1>WebGPU + ONNX Runtime Web</h1>
    <p id="status">Initializing…</p>
    <pre id="output"></pre>
    <script type="module" src="./main.js"></script>
  </body>
</html>
```

`main.js`
```javascript
const statusEl = document.getElementById('status');
const outputEl = document.getElementById('output');

function log(msg) {
  outputEl.textContent += `${msg}\n`;
}

(async () => {
  try {
    if (!('gpu' in navigator)) {
      statusEl.textContent = 'WebGPU not available in this browser.';
      return;
    }
    statusEl.textContent = 'WebGPU detected. Loading model…';

    // Use a tiny identity model (or any small ONNX) hosted remotely.
    // For demo, we synthesize a trivial input and expect the same output shape.
    // Replace with your own model URL if desired.
    const modelUrl = 'https://huggingface.co/onnx/models/resolve/main/vision/classification/mnist-12/mnist-12.onnx?download=true';

    // Create a WebGPU session
    const session = await ort.InferenceSession.create(modelUrl, {
      executionProviders: ['webgpu']
    });
    log('Session created with WebGPU.');

    // Create a dummy input of the correct shape for MNIST (1x1x28x28)
    const inputData = new Float32Array(1 * 1 * 28 * 28);
    // Minimal non-zero to avoid NaNs in some models
    inputData[0] = 1.0;
    const input = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);

    // The MNIST model input name is typically 'Input3' or similar; we can inspect session.inputNames
    const feeds = {};
    const inputName = session.inputNames[0];
    feeds[inputName] = input;

    const results = await session.run(feeds);
    const outputName = session.outputNames[0];
    const output = results[outputName]; // Float32Array logits

    statusEl.textContent = 'Inference complete.';
    log(`Input name: ${inputName}`);
    log(`Output name: ${outputName}`);
    log(`Output length: ${output.data.length}`);

    // Argmax to get predicted digit
    let bestIdx = 0; let bestVal = -Infinity;
    for (let i = 0; i < output.data.length; i++) {
      if (output.data[i] > bestVal) { bestVal = output.data[i]; bestIdx = i; }
    }
    log(`Predicted class (argmax): ${bestIdx}`);
  } catch (e) {
    statusEl.textContent = 'Error during inference.';
    log(e?.stack || String(e));
  }
})();
```

3) Server lokalt (Windows cmd.exe)
Brug en simpel statisk server, så browseren kan hente modellen.

```cmd
REM In cmd.exe
mkdir webgpu-demo
cd webgpu-demo
REM Save index.html and main.js here

REM Option A: Python (if installed)
py -m http.server 5173

REM Option B: Node serve (requires Node.js)
REM npx serve -p 5173 .
```

Åbn http://localhost:5173 i din browser. Du bør se initialiseringslogfiler, sessionoprettelse med WebGPU og en argmax-forudsigelse.

4) Fejlfinding
- Hvis WebGPU ikke er tilgængelig: opdater Chrome/Edge og sørg for, at GPU-drivere er aktuelle, og tjek chrome://flags for “Enable WebGPU”.
- Hvis CORS- eller fetch-fejl opstår: sørg for, at du serverer filer via http:// (ikke file://), og at model-URL'en tillader cross-origin forespørgsler.
- Fallback til CPU: ændr `executionProviders: ['wasm']` for at verificere baseline-adfærd.

5) Næste skridt
- Skift til en domænespecifik ONNX-model (f.eks. billedklassifikation eller en lille tekstmodel).
- Tilføj præ- og postprocesseringslogik til reelle input.
- For større modeller eller produktionslatenstid, foretræk Foundry Local eller ONNX Runtime Server.

## Del 5: Open WebUI + Foundry Local (Trin-for-trin)

Dette forbinder Open WebUI med Foundry Locals OpenAI-kompatible endpoint for en lokal chat-UI.

1) Forudsætninger
- Foundry Local installeret og fungerende (`foundry --version`)
- En model klar til at køre lokalt (f.eks. `phi-4-mini`)
- Docker Desktop installeret (anbefales til Open WebUI)

2) Start en model med Foundry Local
```powershell
foundry model run phi-4-mini
```
Dette eksponerer en OpenAI-kompatibel API på `http://localhost:8000`.

3) Start Open WebUI (Docker)
```powershell
# Pull and run Open WebUI
docker pull ghcr.io/open-webui/open-webui:main

docker run -d --name open-webui -p 3000:8080 ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
Bemærkninger:
- På Windows giver `host.docker.internal` containeren adgang til din host på `localhost`.
- Vi sætter `OPENAI_API_BASE_URL` til Foundry Locals endpoint og en dummy `OPENAI_API_KEY`.

4) Konfigurer fra Open WebUI UI (alternativ)
- Gå til http://localhost:3000
- Fuldfør den indledende opsætning (admin-bruger)
- Gå til Indstillinger → Modeller/Providers
- Sæt Base URL: `http://host.docker.internal:8000/v1`
- Sæt API Key: `local-key` (pladsholder)
- Gem

5) Kør en testprompt
- I Open WebUI chat, vælg eller indtast modelnavn `phi-4-mini`
- Prompt: “List fem fordele ved on-device AI inference.”
- Du bør se et svar streamet fra din lokale model

6) Fejlfinding
```powershell
# Verify Foundry Local is serving and responding
curl http://localhost:8000/v1/models

# Check Docker logs for Open WebUI
docker logs -f open-webui

# If you get ECONNREFUSED, ensure the model is running and port 8000 is open
# If the model name doesn't appear, try typing it manually in Open WebUI
```

7) Valgfrit: Persistér Open WebUI-data
```powershell
# Stop and remove the container
docker stop open-webui
docker rm open-webui

# Re-run with a volume to persist data
mkdir %USERPROFILE%\openwebui-data

docker run -d --name open-webui -p 3000:8080 ^
  -v %USERPROFILE%\openwebui-data:/app/backend/data ^
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1 ^
  -e OPENAI_API_KEY=local-key ^
  ghcr.io/open-webui/open-webui:main
```
