## परिचय

EdgeAI शुरुआती लोगों के लिए अध्ययन गाइड में आपका स्वागत है! यह दस्तावेज़ आपको पाठ्यक्रम सामग्री को प्रभावी ढंग से नेविगेट करने और अपने सीखने के अनुभव को अधिकतम करने में मदद करने के लिए डिज़ाइन किया गया है। इसमें संरचित अध्ययन पथ, सुझाए गए अध्ययन कार्यक्रम, प्रमुख अवधारणाओं का सारांश और EdgeAI तकनीकों की गहरी समझ के लिए पूरक संसाधन शामिल हैं।

यह एक संक्षिप्त 20 घंटे का पाठ्यक्रम है जो EdgeAI के बारे में आवश्यक ज्ञान को समय-कुशल प्रारूप में प्रदान करता है, जो व्यस्त पेशेवरों और छात्रों के लिए आदर्श है जो इस उभरते क्षेत्र में व्यावहारिक कौशल जल्दी से प्राप्त करना चाहते हैं।

## पाठ्यक्रम का अवलोकन

यह पाठ्यक्रम सात व्यापक मॉड्यूल में व्यवस्थित है:

1. **EdgeAI की मूल बातें और परिवर्तन** - मुख्य अवधारणाओं और तकनीकी बदलाव को समझना
2. **छोटे भाषा मॉडल की नींव** - विभिन्न SLM परिवारों और उनकी संरचनाओं का अन्वेषण
3. **छोटे भाषा मॉडल की तैनाती** - व्यावहारिक तैनाती रणनीतियों को लागू करना
4. **मॉडल प्रारूप रूपांतरण और क्वांटाइजेशन** - 6 फ्रेमवर्क के साथ उन्नत अनुकूलन
5. **SLMOps - छोटे भाषा मॉडल संचालन** - उत्पादन जीवनचक्र प्रबंधन और तैनाती
6. **SLM एजेंटिक सिस्टम्स** - AI एजेंट्स, फंक्शन कॉलिंग, और मॉडल संदर्भ प्रोटोकॉल
7. **EdgeAI कार्यान्वयन नमूने** - AI टूलकिट, विंडोज विकास, और प्लेटफ़ॉर्म-विशिष्ट कार्यान्वयन

## इस अध्ययन गाइड का उपयोग कैसे करें

- **प्रगतिशील सीखना**: सबसे सुसंगत सीखने के अनुभव के लिए मॉड्यूल को क्रम में पूरा करें
- **ज्ञान जांच बिंदु**: प्रत्येक अनुभाग के बाद आत्म-मूल्यांकन प्रश्नों का उपयोग करें
- **हैंड्स-ऑन अभ्यास**: सैद्धांतिक अवधारणाओं को मजबूत करने के लिए सुझाए गए अभ्यासों को पूरा करें
- **पूरक संसाधन**: उन विषयों के लिए अतिरिक्त सामग्री का अन्वेषण करें जो आपको सबसे अधिक रुचिकर लगते हैं

## मॉड्यूल 5: SLMOps - छोटे भाषा मॉडल संचालन

### मुख्य सीखने के उद्देश्य

- SLMOps जीवनचक्र प्रबंधन सिद्धांतों को समझें
- एज डिप्लॉयमेंट के लिए डिस्टिलेशन और फाइन-ट्यूनिंग तकनीकों में महारत हासिल करें
- मॉनिटरिंग के साथ उत्पादन डिप्लॉयमेंट रणनीतियाँ लागू करें
- एंटरप्राइज-ग्रेड SLM संचालन और रखरखाव वर्कफ़्लो बनाएं

### अध्ययन के मुख्य क्षेत्र

#### सेक्शन 1: SLMOps का परिचय
- **प्राथमिक अवधारणाएँ**: 
  - AI संचालन में SLMOps का बदलाव
  - लागत दक्षता और प्राइवेसी-फर्स्ट आर्किटेक्चर
  - रणनीतिक व्यापार प्रभाव और प्रतिस्पर्धात्मक लाभ

#### सेक्शन 2: मॉडल डिस्टिलेशन
- **प्राथमिक अवधारणाएँ**: 
  - ज्ञान हस्तांतरण तकनीकें
  - दो-चरणीय डिस्टिलेशन प्रक्रिया का कार्यान्वयन
  - Azure ML डिस्टिलेशन वर्कफ़्लो

#### सेक्शन 3: फाइन-ट्यूनिंग रणनीतियाँ
- **प्राथमिक अवधारणाएँ**: 
  - पैरामीटर-एफिशिएंट फाइन-ट्यूनिंग (PEFT)
  - LoRA और QLoRA उन्नत विधियाँ
  - मल्टी-अडैप्टर ट्रेनिंग और हाइपरपैरामीटर ऑप्टिमाइजेशन

#### सेक्शन 4: उत्पादन डिप्लॉयमेंट
- **प्राथमिक अवधारणाएँ**: 
  - उत्पादन के लिए मॉडल रूपांतरण और क्वांटाइजेशन
  - Foundry Local डिप्लॉयमेंट कॉन्फ़िगरेशन
  - परफॉर्मेंस बेंचमार्किंग और गुणवत्ता मान्यता

### आत्म-मूल्यांकन प्रश्न

1. SLMOps पारंपरिक MLOps से कैसे अलग है?
2. एज डिप्लॉयमेंट के लिए मॉडल डिस्टिलेशन के फायदे समझाएँ।
3. संसाधन-सीमित वातावरण में SLMs को फाइन-ट्यून करने के लिए मुख्य विचार क्या हैं?
4. एज AI एप्लिकेशन के लिए एक पूर्ण उत्पादन डिप्लॉयमेंट पाइपलाइन का वर्णन करें।

### प्रैक्टिकल अभ्यास

1. **बेसिक डिस्टिलेशन**: एक बड़े शिक्षक मॉडल से एक छोटा मॉडल बनाएं (1 घंटा)
2. **फाइन-ट्यूनिंग प्रयोग**: एक विशिष्ट डोमेन के लिए एक मॉडल को फाइन-ट्यून करें (1 घंटा)
3. **डिप्लॉयमेंट पाइपलाइन**: मॉडल डिप्लॉयमेंट के लिए एक बेसिक CI/CD पाइपलाइन सेट करें (1 घंटा)

## मॉड्यूल 6: SLM एजेंटिक सिस्टम - AI एजेंट और फंक्शन कॉलिंग

### मुख्य सीखने के उद्देश्य

- छोटे भाषा मॉडल का उपयोग करके एज वातावरण के लिए बुद्धिमान AI एजेंट बनाएं
- व्यवस्थित वर्कफ़्लो के साथ फंक्शन कॉलिंग क्षमताओं को लागू करें
- मानकीकृत टूल इंटरैक्शन के लिए Model Context Protocol (MCP) इंटीग्रेशन में महारत हासिल करें
- न्यूनतम मानव हस्तक्षेप के साथ परिष्कृत एजेंटिक सिस्टम बनाएं

### अध्ययन के मुख्य क्षेत्र

#### सेक्शन 1: AI एजेंट और SLM की नींव
- **प्राथमिक अवधारणाएँ**: 
  - एजेंट वर्गीकरण फ्रेमवर्क (रिफ्लेक्स, मॉडल-आधारित, लक्ष्य-आधारित, लर्निंग एजेंट)
  - SLM बनाम LLM ट्रेड-ऑफ विश्लेषण
  - एज-विशिष्ट एजेंट डिज़ाइन पैटर्न
  - एजेंटों के लिए संसाधन अनुकूलन

#### सेक्शन 2: छोटे भाषा मॉडल में फंक्शन कॉलिंग
- **प्राथमिक अवधारणाएँ**: 
  - व्यवस्थित वर्कफ़्लो कार्यान्वयन (इरादा पहचान, JSON आउटपुट, बाहरी निष्पादन)
  - प्लेटफ़ॉर्म-विशिष्ट कार्यान्वयन (Phi-4-mini, चयनित Qwen मॉडल, Microsoft Foundry Local)
  - उन्नत उदाहरण (मल्टी-एजेंट सहयोग, डायनामिक टूल चयन)
  - उत्पादन विचार (रेट लिमिटिंग, ऑडिट लॉगिंग, सुरक्षा उपाय)

#### सेक्शन 3: Model Context Protocol (MCP) इंटीग्रेशन
- **प्राथमिक अवधारणाएँ**: 
  - प्रोटोकॉल आर्किटेक्चर और लेयर्ड सिस्टम डिज़ाइन
  - मल्टी-बैकएंड सपोर्ट (डेवलपमेंट के लिए Ollama, उत्पादन के लिए vLLM)
  - कनेक्शन प्रोटोकॉल (STDIO और SSE मोड)
  - वास्तविक दुनिया के अनुप्रयोग (वेब ऑटोमेशन, डेटा प्रोसेसिंग, API इंटीग्रेशन)

3. एजेंट संचार में Model Context Protocol की भूमिका समझाएँ।

### प्रैक्टिकल अभ्यास

## मॉड्यूल 7: EdgeAI कार्यान्वयन नमूने

### मुख्य सीखने के उद्देश्य

- Visual Studio Code के लिए AI Toolkit में व्यापक EdgeAI विकास वर्कफ़्लो में महारत हासिल करें
- Windows AI Foundry प्लेटफ़ॉर्म और NPU ऑप्टिमाइजेशन रणनीतियों में विशेषज्ञता प्राप्त करें
- विभिन्न हार्डवेयर प्लेटफ़ॉर्म और डिप्लॉयमेंट परिदृश्यों में EdgeAI लागू करें
- प्लेटफ़ॉर्म-विशिष्ट ऑप्टिमाइजेशन के साथ उत्पादन-तैयार EdgeAI एप्लिकेशन बनाएं

### अध्ययन के मुख्य क्षेत्र

#### सेक्शन 1: Visual Studio Code के लिए AI Toolkit
- **प्राथमिक अवधारणाएँ**: 
  - VS Code के भीतर व्यापक Edge AI विकास वातावरण
  - एज डिप्लॉयमेंट के लिए मॉडल कैटलॉग और खोज
  - स्थानीय परीक्षण, ऑप्टिमाइजेशन, और एजेंट विकास वर्कफ़्लो
  - एज परिदृश्यों के लिए परफॉर्मेंस मॉनिटरिंग और मूल्यांकन

#### सेक्शन 2: Windows EdgeAI विकास गाइड
- **प्राथमिक अवधारणाएँ**: 
  - Windows AI Foundry प्लेटफ़ॉर्म का व्यापक अवलोकन
  - Phi Silica API के साथ कुशल NPU इन्फरेंस
  - कंप्यूटर विज़न API इमेज प्रोसेसिंग और OCR के लिए
  - स्थानीय विकास और परीक्षण के लिए Foundry Local CLI

#### सेक्शन 3: प्लेटफ़ॉर्म-विशिष्ट कार्यान्वयन
- **प्राथमिक अवधारणाएँ**: 
  - NVIDIA Jetson Orin Nano डिप्लॉयमेंट (67 TOPS AI प्रदर्शन)
  - मोबाइल एप्लिकेशन .NET MAUI और ONNX Runtime GenAI के साथ
  - Azure EdgeAI समाधान क्लाउड-एज हाइब्रिड आर्किटेक्चर के साथ
  - Windows ML ऑप्टिमाइजेशन यूनिवर्सल हार्डवेयर सपोर्ट के साथ
  - Foundry Local एप्लिकेशन प्राइवेसी-केंद्रित RAG कार्यान्वयन के साथ

### आत्म-मूल्यांकन प्रश्न

1. AI Toolkit EdgeAI विकास वर्कफ़्लो को कैसे सरल बनाता है?
2. विभिन्न हार्डवेयर प्लेटफ़ॉर्म पर डिप्लॉयमेंट रणनीतियों की तुलना करें।
3. Edge विकास के लिए Windows AI Foundry के फायदे क्या हैं?
4. आधुनिक Edge AI एप्लिकेशन में NPU ऑप्टिमाइजेशन की भूमिका समझाएँ।
5. Phi Silica API NPU हार्डवेयर का उपयोग करके प्रदर्शन ऑप्टिमाइजेशन कैसे करता है?
6. प्राइवेसी-संवेदनशील एप्लिकेशन के लिए स्थानीय बनाम क्लाउड डिप्लॉयमेंट के फायदे की तुलना करें।

### प्रैक्टिकल अभ्यास

1. **AI Toolkit सेटअप**: AI Toolkit कॉन्फ़िगर करें और एक मॉडल को ऑप्टिमाइज करें (1 घंटा)
2. **Windows AI Foundry**: Phi Silica API का उपयोग करके एक सरल Windows AI एप्लिकेशन बनाएं (1 घंटा)
3. **क्रॉस-प्लेटफ़ॉर्म डिप्लॉयमेंट**: एक ही मॉडल को दो अलग-अलग प्लेटफ़ॉर्म पर डिप्लॉय करें (1 घंटा)
4. **NPU ऑप्टिमाइजेशन**: Windows AI Foundry टूल्स के साथ NPU प्रदर्शन का परीक्षण करें (30 मिनट)

## मिनी प्रोजेक्ट आइडिया

EdgeAI अवधारणाओं का अभ्यास करने के लिए इनमें से एक प्रोजेक्ट पूरा करने पर विचार करें (प्रत्येक को 2-4 घंटे में डिज़ाइन किया गया है):

### शुरुआती प्रोजेक्ट (2-3 घंटे प्रत्येक)
1. **एज टेक्स्ट असिस्टेंट**: एक छोटे भाषा मॉडल का उपयोग करके एक सरल ऑफलाइन टेक्स्ट कंप्लीशन टूल बनाएं
2. **मॉडल तुलना डैशबोर्ड**: विभिन्न SLMs के प्रदर्शन मेट्रिक्स का एक बुनियादी विज़ुअलाइज़ेशन बनाएं
3. **ऑप्टिमाइजेशन प्रयोग**: एक ही बेस मॉडल पर विभिन्न क्वांटाइजेशन स्तरों के प्रभाव को मापें

### मध्यम प्रोजेक्ट (3-4 घंटे प्रत्येक)
4. **AI Toolkit वर्कफ़्लो**: VS Code AI Toolkit का उपयोग करके एक मॉडल को शुरू से अंत तक ऑप्टिमाइज और डिप्लॉय करें
5. **Windows AI Foundry एप्लिकेशन**: Phi Silica API और NPU ऑप्टिमाइजेशन का उपयोग करके एक Windows ऐप बनाएं
6. **क्रॉस-प्लेटफ़ॉर्म डिप्लॉयमेंट**: एक ही ऑप्टिमाइज्ड मॉडल को Windows (OpenVINO) और मोबाइल (.NET MAUI) पर डिप्लॉय करें
7. **फंक्शन कॉलिंग एजेंट**: एज परिदृश्यों के लिए फंक्शन कॉलिंग क्षमताओं के साथ एक AI एजेंट बनाएं

### उन्नत इंटीग्रेशन प्रोजेक्ट (4-5 घंटे प्रत्येक)
8. **OpenVINO ऑप्टिमाइजेशन पाइपलाइन**: NNCF और GenAI टूलकिट का उपयोग करके पूर्ण मॉडल ऑप्टिमाइजेशन लागू करें
9. **SLMOps पाइपलाइन**: प्रशिक्षण से एज डिप्लॉयमेंट तक एक पूर्ण मॉडल जीवनचक्र लागू करें
10. **मल्टी-मॉडल एज सिस्टम**: एज हार्डवेयर पर काम करने वाले कई विशेष मॉडल डिप्लॉय करें
11. **MCP इंटीग्रेशन सिस्टम**: टूल इंटरैक्शन के लिए Model Context Protocol का उपयोग करके एक एजेंटिक सिस्टम बनाएं
